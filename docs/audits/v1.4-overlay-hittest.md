# Overlay Hittest Audit -- v1.4

**Date:** 2026-04-10
**Phase:** 02-overlay-hittest-sweep
**Methodology:** Cross-reference CSS `display: flex|grid` selectors with JS `.hidden` toggling and HTML `[hidden]` attributes across 25 CSS files and 30+ JS modules.

## Global Fix

`:where([hidden]) { display: none !important }` added to `@layer base` in `design-system.css` (after the `*, *::before, *::after` box-sizing reset).

This rule ensures any element with the HTML `[hidden]` attribute computes `display: none` regardless of other CSS rules. `:where()` keeps specificity at (0,0,0); `!important` ensures it wins over author `display: flex/grid/block`.

## Shadow DOM Analysis

Web Components with Shadow DOM have their own style scope. The global `:where([hidden])` rule does NOT penetrate shadow roots. However:

- **Host elements are in light DOM:** `[hidden]` on custom element hosts (e.g., `<ag-modal hidden>`) IS handled by the global rule because host elements live in the light DOM where the global stylesheet applies.
- **No shadow-internal [hidden] usage:** Grep of all 23 Web Components in `public/assets/js/components/` shows zero instances of `[hidden]` toggling inside shadow roots. Components use `aria-hidden` for accessibility state (ag-modal, ag-popover) and CSS class toggling for internal visibility.
- **ag-modal:** Uses `aria-hidden="true|false"` for open/close state, not the `[hidden]` attribute. No conflict.
- **ag-popover:** Uses `aria-hidden` on the popover content panel. No `[hidden]` attribute usage.
- **ag-pdf-viewer:** Uses `.hidden` on download button (light DOM child), which is correctly handled by the global rule.

**Status:** N/A -- no shadow DOM conflicts exist.

## Conclusion

All 16 known conflict sites have been remediated by removing per-selector overrides in favor of the single global `:where([hidden]) { display: none !important }` rule. The complete conflict site inventory (25 selectors across 12 CSS files) confirms all sites are now protected. No new conflict sites were discovered that require additional fixes beyond the global rule. The `:not([hidden])` selectors in design-system.css (lines 5292-5299, used for CSS transition reveal animations) were intentionally preserved -- they are positive selectors for showing elements, not [hidden] overrides.
