# TOKENS-V25-01 — Audit des tokens 1-site `design-system.css`

**Closed:** 2026-05-04 · **Source:** v2.4 P4.3 push 95% → 100% borders/shadows ratio (D-08 amendé verbatim).

## Méthode

Token est *1-site* si son `var(--token)` apparaît **exactement 1 fois** dans `public/assets/css/` hors `design-system.css` lui-même. Détecté par grep itératif sur les 72 tokens `--border-*` / `--shadow-*` du design system, post-merge v2.4.

**Compte final:** 43 tokens 1-site (audit re-couru après merge v2.4 PR #260 — 49 → 43 dérive `because some tokens gained 2nd callers in v2.5 work like Logger error_events page styling).

## Classification

### *keep* — 22 tokens (valeur unique légitime, pas de consolidation)

**Animations / keyframes** (cohésion sémantique forte avec @keyframes spécifique) — 11 tokens :
- `--shadow-keyframe-health-bar-pulse-{from,mid}` — pulse santé séance, pas réutilisable hors `<ag-health-bar>`
- `--shadow-keyframe-hero-pulse-{from,mid}` — pulse hero card, palette success/warning paramétrable
- `--shadow-keyframe-live-dot-{from,mid,end}` — point live indicator (3 stops)
- `--shadow-keyframe-live-glow-{from,to}` — halo brand sur connexion live
- `--shadow-keyframe-sse-pulse-{from,mid}` — pulse SSE notification

**Glass component-specific** (set cohérent thick/thin) — 2 tokens :
- `--border-glass-thick` — bordure haut card glass, modulaire
- `--border-glass-thin` — bordure bas card glass

**Login-specific** (component dédié) — 3 tokens :
- `--shadow-login-btn`
- `--shadow-login-btn-hover`
- `--shadow-login-btn-active`

**Focus rings sémantiques** (palette danger/inset variants) — 6 tokens :
- `--shadow-focus-ring-danger` — distincte de focus standard, pour confirmation destructive
- `--shadow-focus-offset` — offset spécifique buttons primary
- `--shadow-ring-2px-{border,danger,success,surface}` — variants sémantiques uniques par rôle

### *consolidate* — 21 tokens (rapprocher d'un token existant)

**Alpha variants → token canonique avec opacity** (10 tokens) :

| 1-site token | → consolidate vers | Action |
|---|---|---|
| `--border-alpha` | `--border-default` (avec `color-mix`) | remplacer call-site |
| `--border-danger-alpha-25` | `--color-danger` + `border: var(--border-default)` opacity-25 | refactor consumer |
| `--border-primary-alpha-22` | `--color-primary` opacity-22 ou ⇒ alpha-25 (arrondi) | refactor consumer |
| `--border-primary-alpha-25` | conserver, mais consolider avec alpha-22 (1 token au lieu de 2) | unifier alpha → 25 |
| `--border-success-alpha-20` | `--border-success-alpha-25` (arrondi) | unifier alpha → 25 |
| `--border-success-alpha-25` | conserver | unifier alpha → 25 |
| `--border-warning-alpha-25` | conserver | pattern uniforme ` --border-{role}-alpha-25` |

**Recommandation:** unifier toutes les variants alpha à `25` (arrondi visuel non perceptible vs 20/22). Cela donne 4 tokens semantiques (`--border-{primary,danger,success,warning}-alpha-25`) au lieu de 7 actuels.

**Emphasis variants → consolider** (4 tokens) :

| 1-site token | → consolidate vers | Action |
|---|---|---|
| `--border-emphasis-primary` | `--border-emphasis` + `border-color: var(--color-primary)` | refactor |
| `--border-emphasis-warning` | idem avec `--color-warning` | refactor |
| `--border-emphasis-danger-dashed` | `--border-emphasis` + `border-style: dashed` + `border-color: var(--color-danger)` | refactor |
| `--border-emphasis-text-inverse-alpha-30` | utility cas-spécifique, marker à supprimer (single use sur dark mode hero) | refactor consumer |

**Width / generic variants** (4 tokens) :

| 1-site token | → consolidate vers | Action |
|---|---|---|
| `--border-thick-3` | `--border-thick` (assumer 3px) | rename + dedup |
| `--border-thin-1-5` | `--border-thin` ou `--border-default` | refactor |
| `--border-soft` | `--border-subtle` (synonyme) | rename |
| `--border-none` | inline `border: none` (pas un token utile) | remove + inline |

**Shadow ring variants** (3 tokens) :

| 1-site token | → consolidate vers | Action |
|---|---|---|
| `--shadow-ring-1px-primary` | `--shadow-ring-2px-primary` (1 width canonique) | unifier 1px→2px |
| `--shadow-ring-3px-primary-strong` | idem (intentional emphasis = use opacity instead) | refactor |
| `--shadow-ring-inset-{1px-danger,2px-primary}` | unifier inset à 2px | refactor |

## Plan de remédiation

### Phase 7.1 (immediate, atomic) — alpha unification

Remplacer 7 tokens alpha-* (--border-{primary,danger,success,warning}-alpha-{20,22,25}) par 4 tokens canoniques (`--border-{role}-alpha-25`). Migration cas-par-cas dans 7 fichiers CSS.

**Impact:** -3 tokens 1-site (les redondants 20/22 disparaissent, les 25 restent).

### Phase 7.2 (later) — width / soft / none cleanup

- `--border-thick-3` → renommer `--border-thick`
- `--border-thin-1-5` → renommer ou retirer (consumer migre vers `--border-default`)
- `--border-soft` → renommer `--border-subtle` (synonyme déjà présent comme `--border-subtle`)
- `--border-none` → inline `border: none` dans le seul caller

**Impact:** -4 tokens 1-site.

### Phase 7.3 (later) — emphasis flatten

Les 4 `--border-emphasis-*` variantes peuvent être rendues inline (border-color override) plutôt que tokens dédiés. Le `--border-emphasis` base reste, les variants disparaissent.

**Impact:** -4 tokens 1-site.

### Phase 7.4 (later) — ring variants unification

Unifier toutes les rings à 2px width par défaut. Variants 1px et 3px deviennent inline.

**Impact:** -3 tokens 1-site.
