Mapping recommande (typique d'une echelle 4px-base — A VERROUILLER apres lecture reelle des tokens) :

| Valeur brute | Token cible probable          |
|--------------|-------------------------------|
| 2px          | var(--space-px) si dispo, sinon laisser 2px |
| 4px          | var(--space-1)                |
| 8px          | var(--space-2)                |
| 12px         | var(--space-3)                |
| 16px         | var(--space-4)                |
| 20px         | var(--space-5)                |
| 24px         | var(--space-6)                |
| 32px         | var(--space-7)                |
| 40px         | var(--space-8) ou --space-10  |
| 48px         | token correspondant ou conserver |
| 64px+        | token correspondant ou exception documentee |

Regles de migration :
- `padding: 12px;`        -> `padding: var(--space-3);`
- `padding: 0 16px;`      -> `padding: 0 var(--space-4);` (le 0 reste no-token)
- `margin: 16px 8px 12px;` -> `margin: var(--space-4) var(--space-2) var(--space-3);`
- `margin: 16px auto;`    -> `margin: var(--space-4) auto;` (auto reste mot-cle)
- `padding-left: 24px;`   -> `padding-left: var(--space-6);`

Exclusions (ne PAS toucher, ne comptent pas dans les hits) :
- `0` (no-token-needed)
- `auto`, `inherit`, `initial`, `unset`
- valeurs en %, vh, vw, fr, ch, ex
- valeurs fractionnaires (0.5rem, 1.5em) — ces unites n'apparaissent pas dans le grep cible
- bordures 1px, 2px, 3px dans `border:` (couvertes par --border-* tokens v2.6)
- valeurs hors-grille (7px, 13px, 27px etc.) : DECISION par site —
  soit (a) arrondir a la valeur de grille la plus proche et tokeniser (perte mineure
  de fidelite visuelle), soit (b) garder la valeur brute et la documenter comme exception
  dans v2.7-TOKENS-FINAL.md. Preference : option (a) si la difference <=2px et le contexte
  le permet ; option (b) si valeur d'alignement precise (ex : centrage d'icone 13px).

Commandes grep canoniques (utilisees pour verify et pour figer le bilan) :

GREP_SPACING (compteur global) :
   for f in public/assets/css/*.css; do bn=$(basename "$f"); [ "$bn" = "design-system.css" ] && continue; grep -cE "(padding\|margin)(-(top\|right\|bottom\|left\|inline\|block\|inline-start\|inline-end\|block-start\|block-end))?:[[:space:]]*[0-9]+" "$f"; done | paste -sd+ | bc

GREP_SPACING_FILE (par fichier) :
   grep -cE "(padding\|margin)(-(top\|right\|bottom\|left\|inline\|block\|inline-start\|inline-end\|block-start\|block-end))?:[[:space:]]*[0-9]+" <fichier>

GREP_BORDER_RATIO (calcul approximatif borders tokenisees vs brutes) :
   T=$(grep -rcE "border(-(top\|right\|bottom\|left))?:[^;]*var\(--" public/assets/css/ --include="*.css" | grep -v design-system.css | awk -F: "{s+=\$2} END {print s}")
   B=$(grep -rcE "border(-(top\|right\|bottom\|left))?:[^;]*[0-9]+px[^;]*(solid\|dashed\|dotted)" public/assets/css/ --include="*.css" | grep -v design-system.css | awk -F: "{s+=\$2} END {print s}")
   echo "tokenises=$T brutes=$B ratio=$(echo "scale=3; $T/($T+$B)*100" | bc)%"

GREP_SHADOW_RATIO (idem pour shadows) :
   T=$(grep -rcE "box-shadow:[^;]*var\(--" public/assets/css/ --include="*.css" | grep -v design-system.css | awk -F: "{s+=\$2} END {print s}")
   B=$(grep -rcE "box-shadow:[^;]*(rgba?\(\|#[0-9a-f])" public/assets/css/ --include="*.css" | grep -v design-system.css | awk -F: "{s+=\$2} END {print s}")
   echo "tokenises=$T brutes=$B"

(Ces commandes sont des approximations ; ajuster si false-positives detectes — la verite est dans
la lecture humaine des hits restants. v2.6-TOKENS-AUDIT-FINAL.md utilise une methode comparable.)
