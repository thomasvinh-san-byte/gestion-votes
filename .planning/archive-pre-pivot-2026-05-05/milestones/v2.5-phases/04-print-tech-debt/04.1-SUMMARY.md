# Phase 4 Plan 04.1: dompdf header/footer via CSS @page rules — Summary

**One-liner :** Tous les exports PDF générés par dompdf (PV + procurations) émettent désormais un header répété (`[Titre séance] — JJ/MM/YYYY`) et un footer paginé (`Page X sur Y`) via `@page @top-center` / `@bottom-center` + `counter(page|pages)`, validé par 7 nouveaux tests PHPUnit.

## What was built

### Code changes

**`app/Services/ProcurationPdfService.php`** (+49 lignes / -1) :
- Remplacement de `@page { margin: 2cm; }` par bloc `@page` complet avec `@top-center` (header) + `@bottom-center` (footer paginé)
- Pré-calcul header texte en PHP : `$rawTitle . ' — ' . $dateFormatted` (ou fallback sur titre seul si pas de date)
- Helper privé `escapeCssString()` pour échapper `\` et `"` en sécurité CSS, en préservant les accents UTF-8 (rendus par DejaVu Sans)
- Préservation `page-break-inside: avoid` sur `.legal-box, .signature-table, .info-table`

**`app/Services/MeetingReportsService.php`** (+38 lignes / -1) :
- Méthode `buildPdfHtml()` : variable `$css` reconstruite depuis `$pageCss + $pageBreakCss + (CSS body existant)`
- Header texte injecté depuis `$meeting['title']` + `date('d/m/Y', strtotime($scheduled_at))`
- Helper privé `escapeCssString()` (identique à ProcurationPdfService — DRY déclinable v2.5+ vers helper partagé)
- `page-break-inside: avoid` ajouté sur `.result-box, .header-info, h2, h3` (cohérence v2.3 EDITORIAL-07)

### Tests added (7)

- `testRenderHtmlPreservesPageBreakInsideRules` — règle préservée

## Sandbox-deferred items
