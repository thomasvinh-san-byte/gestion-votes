## 3. Classification (3 buckets)

### 3.1 Critical-path — TOUJOURS VISIBLE (D-06)

Ces boutons restent en accès direct quel que soit l'état. **9 sacred clickables** :

1. `#opBtnToggleVote` — Lancer / Fermer le vote (#56)
2. `#opBtnProclaim` — Proclamer (#55)
3. `#execBtnCloseVote` — Clôturer le vote (#57, contextual)
4. `#opBtnCloseSession` (header, rouge) — Clôturer la séance (#42)
5. `#btnPrimary` — Bouton primaire séance (#31)
6. `#btnModeSetup` + `#btnModeExec` — Mode switch (#29, #30)
7. `#meetingSelect` — Sélecteur séance (#26)
8. Sidebar `#operator` link (#9) — page courante, navigation racine
9. `a.nav-brand` (#1) — sortie vers accueil (sécurité racine)

### 3.2 Contextual-only — VISIBLE SELON ÉTAT (D-04)

| Bouton | Règle de visibilité |
|---|---|
| `#opBtnToggleVote` (Lancer) | visible quand `data-vote-state IN [idle, closed]` |
| `#opBtnToggleVote` (Fermer label) | label change quand `data-vote-state == open` |
| `#execBtnCloseVote` | visible **uniquement** quand `data-vote-state == open` |
| `#opBtnProclaim` | visible **uniquement** quand `data-vote-state == closed` |
| `#opBtnNextVote`, `#opBtnEndSession` | visibles **uniquement** post-clôture motion |
| `#execBtnCloseSession`, `#execBtnSwitchResults` (banner) | visibles **uniquement** quand `#execCloseBanner` actif |
| `#opBtnFocusMode` | toujours visible (toggle UX) |

### 3.3 Disclosed — REPLIÉ DERRIÈRE "Plus d'actions" (D-03)

Boutons utiles mais non critical-path → repliés dans `<details class="op-action-disclosure">` au-dessus de `#opActionBar` :

1. `#opBtnRoomDisplay` (Projection — duplicate de `#btnProjector`)
2. `#opBtnFocusMode` (Focus mode — power user)
3. `#opBtnCloseSession` (header rouge — duplique #execCloseBanner)
4. `#btnBarRefresh` (Actualiser)
5. `button.tour-trigger-btn` (Aide)
6. `#btnProjector` (Projection meeting bar)

**6 cliquables migrés en disclosure**.

### 3.4 Sub-tab collapse — TOP 3 + OVERFLOW (D-05)

Audit montre **3 sub-tabs déjà** (Résultat, Avancé, Présences) — **pas de collapse nécessaire** au niveau sub-tabs panel. Décision : laisser intactes les 3 op-tab car D-05 cible "sub-tabs >3". On conserve. **Top 3 = Résultat / Avancé / Présences**, rationale :
- **Résultat** : centerpiece du vote en cours (fréquence d'usage 90%)
- **Avancé** : actions secondaires (unanimité, passerelle, suspend, proxy) → leur container est déjà disclosure naturel via tab inactif
- **Présences** : rappel rapide chiffres présence

→ La règle D-05 est **déjà satisfaite** par la structure existante (3 op-tab, panneaux mutuellement exclusifs).

Cible alternative D-05 : **les top-level tabs nav** (`#tabsNav`) ont **7 boutons** en mode live. On replie les 3 tabs "préparation" derrière un disclosure "Plus d'onglets" en mode exec :
- Top 3 visibles en exec : **Vote** (#38), **Parole** (#37), **Résultats** (#39) — phase live
- Repliés derrière "Plus d'onglets (4)" : Séance, Participants, Ordre du jour, Contrôle (préparation, peu utilisés en direct)

→ Gain net : 7 → 3 + 1 trigger = 4. **Δ = -3**.

### 3.5 Sidebar — collapse par défaut en mode opérateur

Décision discrétion (Claude) : **par défaut en mode operator+exec, replier 3 groupes sur 5** (post / audit / admin) → seulement `prep` + `live` ouverts. Persistance via localStorage `agSidebarGroupsOperator`.

| Groupe | Défaut exec | Cliquables visibles |
|---|---|---|
| Préparation | open | 1 toggle + 4 items = 5 |
| Séance en direct | open | 1 toggle + 3 items = 4 |
| Après la séance | **collapsed** | 1 toggle = 1 |
| Contrôle | **collapsed** | 1 toggle = 1 |
| Système | **collapsed** | 1 toggle = 1 |
| Brand + theme | always | 2 |

**Sidebar exec mode après collapse : 14 cliquables** (vs 25 baseline). Δ = -11.

## 6. Risques identifiés

1. **Compte sidebar dépend de la persistance localStorage** : un utilisateur qui a déplié `admin` voit plus de 25. Le test gardien doit utiliser `page.evaluate(localStorage.clear())` avant de mesurer pour garantir l'état par défaut.
2. **Agenda items** : pour ODJ ≥7 motions le compte dépasse 25 même après réduction. Mitigation : agenda déjà a `overflow:auto` (scroll), donc les items hors viewport ne comptent pas en `:visible` Playwright. À vérifier dans le test.
3. **Sub-tab `Avancé`** déjà caché par défaut (panel inactif). Ses 4 boutons (`#opBtnUnanimity`, `#opBtnPasserelle`, `#opBtnProxy`, `#opBtnSuspend`) ne comptent pas par défaut. Si l'utilisateur active le tab `Avancé`, le compte total monte à 27-28. **Décision** : l'audit fixe la mesure de référence à `sub-tab Résultat actif` (état par défaut au load). Test Playwright assert sur cet état.
