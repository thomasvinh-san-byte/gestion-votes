### 1. `<ag-integrity-modal>` (cible 02.2 â€” debounce REQUIRED)

- Fichier : `public/assets/js/components/ag-integrity-modal.js`
- Burst channel : **`attributeChangedCallback(name, oldValue, newValue)`** (lignes 57-61)

- Pas d'`EventSource` direct. Pas d'`addEventListener('message', ...)`. Le composant est **pilotĂ© par attributs** : le burst SSE arrive via un emetteur externe (futur ou existant) qui setAttribute(`data-events`, ...) sur le custom element apres reception d'un event SSE.
- Sites d'usage actuels :
  - `public/audit.htmx.html:281` â€” `<ag-integrity-modal hidden data-date="" data-events="[]"></ag-integrity-modal>`
  - `public/report.htmx.html:260` â€” idem

- **Etat actuel** : aucun setAttribute('data-events', ...) n'est emis cote JS dans le codebase. Le risque de double-render n'est donc pas demontre en production aujourd'hui. Cependant le plan livre la guard preventive (D-05) pour quand un emetteur SSE pousse des updates via cet attribut.
