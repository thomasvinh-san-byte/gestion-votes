    For each PRIMARY interaction:

    1. **Page mounts without auth + header visible**
       - Navigate:
         ```js
         await page.goto('/public.htmx.html', { waitUntil: 'domcontentloaded' });
         await page.waitForSelector('#badge', { timeout: 10000 });
         ```
       - Assert `#meeting_title` is visible.
       - Assert `#clock` is visible.
       - Assert `#btnThemeToggle` and `#btnFullscreen` are visible.

    2. **Dark theme is forced on mount (DISP-01)**
       - Read `data-theme` from `<html>`:
         ```js
         const theme = await page.evaluate(() => document.documentElement.getAttribute('data-theme'));
         expect(theme).toBe('dark');
         ```

    3. **Theme toggle — click flips data-theme attribute**
       - Click `#btnThemeToggle`.
       - Wait 200ms.
       - Read `data-theme` again — expect it to be `'light'` (flipped from dark):
         ```js
         const themeAfter = await page.evaluate(() => document.documentElement.getAttribute('data-theme'));
         expect(themeAfter).toBe('light');
         ```
       - Click `#btnThemeToggle` again to restore dark.
       - Assert back to `'dark'`.

    4. **Fullscreen button — click is handled (must not throw)**
       - In Playwright, requesting fullscreen outside a user gesture often fails — use `page.evaluate` to detect whether the click listener is wired instead of actually triggering fullscreen:
         ```js
         const hasListener = await page.evaluate(() => {
           const el = document.getElementById('btnFullscreen');
           return !!el && typeof el.onclick === 'function' || el.getAttribute('aria-label') !== null;
         });
         expect(hasListener).toBeTruthy();
         ```
       - Alternatively, click the button inside a `page.evaluate(() => document.getElementById('btnFullscreen').click())` wrapped in try/catch — assert the page did not navigate and no uncaught error was thrown.

    5. **Status badge reflects meeting state**
       - Assert `#badge` has text content that is NOT empty:
         ```js
         const badgeText = (await page.locator('#badge').textContent()).trim();
         expect(badgeText.length).toBeGreaterThan(0);
         ```
       - Valid states: "hors séance", "en direct", "en attente" (depending on backend). Do not assert a specific string — just non-empty.

    6. **SSE connect does not crash the page**
       - Listen to `page.on('pageerror', ...)` for the entire test and accumulate errors. After waiting 2000ms post-mount for SSE subscription, assert `pageerrors.length === 0`:
         ```js
         const errors = [];
         page.on('pageerror', e => errors.push(e.message));
         // ... navigation and interactions ...
         await page.waitForTimeout(2000);
         expect(errors.filter(e => !e.includes('fullscreen'))).toEqual([]);
         ```

    7. **Waiting state / motion state is visible (exclusive)**
       - At least one of the following must be visible: `#waiting_state`, `#motion_title` (with non-empty text), `#meeting_picker` (if multiple meetings).
       - Assert via:
         ```js
         const hasVisibleState = await page.evaluate(() => {
           const waiting = document.getElementById('waiting_state');
           const motion = document.getElementById('motion_title');
           const picker = document.getElementById('meeting_picker');
           return (waiting && !waiting.hidden && waiting.offsetParent !== null)
               || (motion && motion.textContent && motion.textContent.trim() !== '—')
               || (picker && !picker.hidden && picker.offsetParent !== null);
         });
         expect(hasVisibleState).toBe(true);
         ```

    8. **Width verification — no horizontal overflow on 1920×1080**
       - Set viewport: `await page.setViewportSize({ width: 1920, height: 1080 });`
       - Assert `document.documentElement.scrollWidth <= document.documentElement.clientWidth + 1`:
         ```js
         const overflow = await page.evaluate(() => document.documentElement.scrollWidth - document.documentElement.clientWidth);
         expect(overflow).toBeLessThanOrEqual(1);
         ```
