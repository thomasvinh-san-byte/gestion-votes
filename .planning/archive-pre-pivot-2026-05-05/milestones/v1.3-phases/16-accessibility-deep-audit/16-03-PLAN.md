Required tests (each in its own `test.describe` or `test()`):

1. **Skip-link on operator shell (admin/operator layout)**:
   - `await loginAsOperator(page); await page.goto('/operator.htmx.html');`
   - `await page.keyboard.press('Tab');`
   - Assert `await page.locator(':focus').evaluate(el => el.className)` contains `skip-link` (soft: or the href is `#main-content`).
   - Press Enter; assert focus has moved into main via `await page.evaluate(() => document.activeElement.closest('main') !== null)` returns true, OR that `location.hash === '#main-content'`.

2. **Skip-link on login shell**:
   - Same pattern on `/login.html` without login.
   - Accept either a skip-link or the first focus landing on `#email` (if the layout has no skip-link, document as waiver inline).

3. **Skip-link on voter shell**:
   - `await loginAsVoter(page); await page.goto('/vote.htmx.html');`
   - Same skip-link assertion pattern.

4. **Skip-link on public projection shell**:
   - `await page.goto('/public.htmx.html');`
   - Projection may have no skip-link (no interactive nav). Accept as waiver inline with `// A11Y-WAIVER: skip-link — projection has no interactive nav — expires 2026-10-09` and skip the assertion via `test.skip` or conditional.

5. **Tab order coherence (operator shell)**:
   - `await loginAsOperator(page); await page.goto('/operator.htmx.html');`
   - Press Tab up to 5 times. For each stop, capture the active landmark via `page.evaluate(() => { const a = document.activeElement; return a?.closest('header, nav, main, footer')?.tagName || 'NONE'; })`.
   - Soft assertion: the sequence contains only `HEADER`, `NAV`, `MAIN`, `FOOTER` values (no `NONE`). This is a smoke test, not a full order spec.

6. **ag-modal focus trap (via members page)**:
   - `await loginAsOperator(page); await page.goto('/members.htmx.html');`
   - Wait for a row to appear; click the first delete/edit button that triggers a modal: `await page.locator('[data-action="delete"], [data-action="edit"]').first().click();`
   - Wait for `ag-modal[aria-hidden="false"]` to be visible (timeout 5000ms).
   - Press Tab 5 times; assert focus stays inside the modal (check `document.activeElement?.tagName === 'AG-MODAL'` OR the focused element is a descendant of the modal — use evaluate with shadow DOM pattern below).
   - Press Shift+Tab 3 times; same assertion.
   - Press Escape; assert `await page.locator('ag-modal[aria-hidden="false"]').count()` === 0 within 2000ms.
   - Assert focus restored to trigger button: `await page.evaluate(() => document.activeElement?.tagName)` === 'BUTTON' (or the actual trigger element).

   **Shadow DOM pattern for focus assertions** (RESEARCH Pitfall 1):
   ```js
   const focusInfo = await page.evaluate(() => {
     const active = document.activeElement;
     const host = active?.tagName || null;
     const shadowActive = active?.shadowRoot?.activeElement?.tagName || null;
     const insideModal = !!active?.closest('ag-modal');
     return { host, shadowActive, insideModal };
   });
   expect(focusInfo.insideModal || focusInfo.host === 'AG-MODAL').toBe(true);
   ```

Conditional guard for missing modal trigger: if `members.htmx.html` has no delete/edit button available in the seeded test data, fall back to opening a modal from `operator.htmx.html` or use `test.skip('no modal trigger available in fixtures')` with a TODO comment referencing plan 16-03.
