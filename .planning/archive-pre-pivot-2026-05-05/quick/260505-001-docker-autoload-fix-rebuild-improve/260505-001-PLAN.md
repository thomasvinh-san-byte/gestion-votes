# Quick Task 260505-001 : Docker autoload fix + rebuild.sh polish

## Problem

Sur le serveur de dev (`gestion_votes_php`), tous les hits `/api/v1/events.php` retournent **HTTP 500** avec :

```
Uncaught exception: Class "AgVote\SSE\SseAuthGate" not found
```

**Root cause :** Dans le `Dockerfile`, `composer install --optimize-autoloader` (ligne 58) tourne AVANT `COPY . .` (ligne 62). Au moment du `composer install`, seuls `composer.json` + `composer.lock` sont dans le container — `app/` n'existe pas encore. Le classmap optimisé est donc vide pour les classes du projet. Le PSR-4 fallback runtime devrait compenser, mais sur le setup `gestion_votes_php` ça casse (probablement volume mount qui écrase `vendor/` en runtime, ou image jamais rebuildée depuis l'ajout de `SseAuthGate` au commit `044fa1a`).

Effet bloquant : Phase 1 v2.6 — la spec Playwright `sse-heartbeat.spec.js` ne peut pas tourner live tant que le SSE endpoint retourne 500.

## Tasks

### Task 1 — `Dockerfile` : split composer install / dump-autoload

**File:** `Dockerfile`

**Action :** Remplacer le bloc lignes 57-59 :

```dockerfile
# AVANT
COPY composer.json composer.lock ./
RUN composer install --no-dev --optimize-autoloader --no-interaction --no-progress \
    && rm -f /usr/bin/composer
```

Par :

```dockerfile
# APRÈS — dump-autoload après COPY pour que le classmap couvre app/
COPY composer.json composer.lock ./
RUN composer install --no-dev --no-autoloader --no-interaction --no-progress
COPY . .
RUN composer dump-autoload --optimize --classmap-authoritative --no-interaction \
    && rm -f /usr/bin/composer
```

Et **supprimer le `COPY . .` ligne 62** (déjà fait dans la nouvelle séquence).

**Verify :**
- `grep -c "composer install --no-dev --no-autoloader" Dockerfile` = 1
- `grep -c "composer dump-autoload --optimize --classmap-authoritative" Dockerfile` = 1
- `grep -c "^COPY \. \." Dockerfile` = 1 (un seul, après le composer install)

### Task 2 — `bin/rebuild.sh` : ajouter `--quick` pour fast path autoload-only

**File:** `bin/rebuild.sh`

**Action :** Étendre le script pour offrir un fast path qui régénère uniquement l'autoload + restart php-fpm worker, sans rebuild complet.

Ajouts :
- Parser `--quick` ou `-q` → set `QUICK_MODE=1`
- Si `QUICK_MODE` : `docker compose exec agvote-app composer dump-autoload -o` puis `docker compose exec agvote-app pkill -USR2 -f php-fpm` (graceful reload), skip rebuild + healthcheck
- Sinon : flow existant inchangé
- Print "[rebuild] Mode : quick (autoload-only)" ou "[rebuild] Mode : full (build + healthcheck)" pour clarté
- Ajouter section help quand `--help` ou `-h`

**Verify :**
- `grep -c -- "--quick" bin/rebuild.sh` ≥ 2 (parse + help)
- `grep -c "dump-autoload -o" bin/rebuild.sh` = 1
- Default flow (`./bin/rebuild.sh`) toujours présent : `docker compose down --remove-orphans && docker compose build && docker compose up -d`
- `bash -n bin/rebuild.sh` valide la syntaxe
