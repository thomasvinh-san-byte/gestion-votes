### V27-06 — Transitions brut → tokens `--duration-*` / `--ease-*`

#### Mapping figé (vérifié contre `design-system.css:621-660`)

> **Note importante** : la nomenclature réelle est `--duration-*` (pas `--motion-*` comme
> hypothèse planning). Le plan 01-02 doit utiliser `--duration-*`.

| Durée source | Token cible | Equivalent ms |
|--------------|-------------|---------------|
| `0.05s` / `50ms` | `var(--duration-instant)` | 50ms |
| `0.1s` / `100ms` | `var(--duration-fast)` | 100ms |
| `0.15s` / `150ms` | `var(--duration-normal)` | 150ms |
| `0.2s` / `200ms` | `var(--duration-moderate)` | 200ms |
| `0.25s` / `250ms` | `var(--duration-slow)` | 250ms |
| `0.3s` / `300ms` | `var(--duration-deliberate)` | 300ms |
| `0.4s` / `400ms` | `var(--duration-elaborate)` | 400ms |
| `0.5s` / `500ms` | `var(--duration-dramatic)` | 500ms |
| `0.8s` / `800ms` | `var(--duration-dramatic)` (ou conserver `800ms` si critique animation) | 500ms (à arbitrer) |
| `1s` / `1000ms` | conserver littéral OU étendre la palette dans design-system | n/a |

| Easing source | Token cible |
|---------------|-------------|
| `ease`, `ease-in-out` | `var(--ease-standard)` |
| `ease-out` | `var(--ease-out)` |

| `linear` | `var(--ease-linear)` |

| `cubic-bezier(.23, 1, .32, 1)` | conserver littéral (pas de token équivalent) ou étendre design-system |

Pour les transitions composites toutes-propriétés (`transition: ui-color, bg, shadow`),
préférer les tokens composites quand cohérents :
- `var(--transition-color)` (color + background)

- `var(--transition-ui)` (color + bg + border-color combiné)
