**Strategy:** Piggyback on existing SSE infrastructure. Use Redis SET with TTL to track active operator connections. Emit a `operator.presence` event on connect/disconnect. Client shows a badge when >1 operator.

**Backend — events.php:**
- After the existing consumer registration block (line ~100-108), add operator presence tracking:
  ```php
  // Track operator presence in Redis (for multi-operator badge)
  $actorRole = $_SESSION['auth_user']['role'] ?? '';
  if (in_array($actorRole, ['operator', 'admin', 'president'], true) && RedisProvider::isAvailable()) {
      try {
          $redis = RedisProvider::connection();
          $presenceKey = "sse:operators:{$meetingId}";
          $redis->sAdd($presenceKey, $consumerId);
          $redis->expire($presenceKey, 90); // 90s TTL — renewed each reconnect
          $operatorCount = (int) $redis->sCard($presenceKey);
          // Emit presence event to all consumers of this meeting
          if ($operatorCount >= 1) {
              sendEvent('operator.presence', ['count' => $operatorCount, 'meeting_id' => $meetingId]);
          }
      } catch (Throwable $e) {
          // Graceful degradation
      }
  }
  ```
- At end-of-script cleanup (after the event loop exits), remove this consumer from the presence set:
  ```php
  // Cleanup presence on graceful exit
  if (isset($redis, $presenceKey)) {
      try { $redis->sRem($presenceKey, $consumerId); } catch (Throwable $e) {}
  }
  ```
  Wrap in `register_shutdown_function` to handle ungraceful exits:
  ```php
  if (isset($redis, $presenceKey, $consumerId)) {
      register_shutdown_function(function() use ($redis, $presenceKey, $consumerId) {
          try { $redis->sRem($presenceKey, $consumerId); } catch (Throwable $e) {}
      });
  }
  ```
