# Quick Task 260505-001 — Summary

## Outcome

Bug HTTP 500 sur `/api/v1/events.php` (cause : `Class "AgVote\SSE\SseAuthGate" not found`) — fix livré côté image Docker. Bonus : `bin/rebuild.sh --quick` pour éviter le rebuild full quand seul du code PHP a bougé.

## Changes

### `Dockerfile` (commit `2d30be7`)

Avant : `composer install --optimize-autoloader` au build (avant `COPY . .`) → classmap généré sur arbre vide.

Après : split en 2 étapes :
1. `composer install --no-dev --no-autoloader` (cache stable tant que `composer.lock` ne bouge pas)
2. `COPY . .`
3. `composer dump-autoload --optimize --classmap-authoritative`

Mode `--classmap-authoritative` désactive le PSR-4 fallback runtime — toute classe doit être dans le map, ce qui rend les "class not found" immédiats au build plutôt que silencieux en prod.

### `bin/rebuild.sh` (commit `1a1dfb7`)

Ajout du flag `--quick` (alias `-q`) :
- Régénère uniquement l'autoload composer dans le container running
- Fallback si composer absent (Dockerfile le retire post-install) : reload `USR2` php-fpm
- Pas de rebuild d'image, pas de healthcheck wait
- Use case : modif code PHP en dev, le volume mount apporte les nouveaux fichiers, suffit de rafraîchir le classmap

Default flow (`./bin/rebuild.sh` sans flag) préservé identique.

Ajout `--help` / `-h` inline. Validation `--quick + --no-cache` incompatibles (erreur explicite).
