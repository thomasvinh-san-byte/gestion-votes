**Finding 1 — CSV injection in ExportService::writeCsvRow:**
Add a private `sanitizeCsvCell(mixed $value): string` method that prefixes any cell value starting with `=`, `+`, `-`, or `@` with a tab character `\t`. This prevents formula injection when the CSV is opened in Excel/LibreOffice. Update `writeCsvRow()` to apply `sanitizeCsvCell()` to each element before calling `fputcsv()`.

```php
private function sanitizeCsvCell(mixed $value): string {
    $str = (string) $value;
    if ($str !== '' && in_array($str[0], ['=', '+', '-', '@'], true)) {
        return "\t" . $str;
    }
    return $str;
}

public function writeCsvRow($handle, array $row, string $separator = ';'): void {
    fputcsv($handle, array_map([$this, 'sanitizeCsvCell'], $row), $separator);
}
```

**Finding 2 — XSS in EmailTemplateService::render for HTML emails:**
The `render()` method does a raw `str_replace()` of variables into template bodies. When the template is HTML (all three default templates are HTML), user-controlled values like `{{member_name}}`, `{{meeting_title}}`, `{{meeting_location}}` are injected verbatim. Fix by adding a `renderHtml()` method that escapes values before substitution:

```php
public function renderHtml(string $templateBody, array $variables): string {
    $escapedVars = array_map(
        fn($v) => htmlspecialchars((string) $v, ENT_QUOTES | ENT_SUBSTITUTE, 'UTF-8'),
        $variables,
    );
    return str_replace(array_keys($escapedVars), array_values($escapedVars), $templateBody);
}
```

In `renderTemplate()`, replace `$this->render($template['body_html'], $variables)` with `$this->renderHtml($template['body_html'], $variables)`. Keep `render()` unchanged for plain-text bodies and subject lines (escaping HTML entities in subjects would corrupt them).

**Finding 3 — No lockout recovery command:**
Create `app/Command/RateLimitResetCommand.php`. Model it after the existing `RateLimitCleanupCommand`. Command name: `ratelimit:reset`. Required options: `--context` (string, e.g. `login`) and `--identifier` (string, the IP or email). Call `RateLimiter::reset($context, $identifier)`.

```php
#[AsCommand(name: 'ratelimit:reset', description: 'Reset rate-limit counter for a specific context and identifier')]
final class RateLimitResetCommand extends Command {
    protected function configure(): void {
        $this->addOption('context', null, InputOption::VALUE_REQUIRED, 'Rate-limit context (e.g. login)');
        $this->addOption('identifier', null, InputOption::VALUE_REQUIRED, 'Identifier to reset (IP, email, etc.)');
    }
    protected function execute(InputInterface $input, OutputInterface $output): int {
        $context = $input->getOption('context');
        $identifier = $input->getOption('identifier');
        if (!$context || !$identifier) {
            $output->writeln('<error>Both --context and --identifier are required.</error>');
            return Command::FAILURE;
        }
        RateLimiter::reset($context, $identifier);
        $output->writeln(sprintf('<info>Rate-limit reset for context="%s" identifier="%s".</info>', $context, $identifier));
        return Command::SUCCESS;
    }
}
```

Add `use AgVote\Command\RateLimitResetCommand;` and `$app->addCommand(new RateLimitResetCommand());` to `bin/console`. Also update the usage comment at the top of `bin/console` to document `ratelimit:reset`.

This silently overwrites the first row's data. Fix by collecting emails seen in this batch:

In `processMemberRows()`, add a `$seenEmails = []` array before the `foreach`. Inside the loop, after extracting `$data['email']`, add:
```php
if (!empty($data['email'])) {
```

This is distinct from the existing `checkDuplicateEmails()` pre-scan which also catches it — the pre-scan aborts early, but `processMemberRows` is the place to also handle it gracefully during row processing when called from `membersXlsx()` (which does not call the pre-scan). **Finding 9 — ExportController: CSV exports buffer all rows in memory via array iteration.**

However, `attendanceCsv()`, `votesCsv()`, `membersCsv()`, `motionResultsCsv()`, and `ballotsAuditCsv()` call `fclose($out)` before output ends normally. Add `ob_end_clean()` before `$export->initCsvOutput()` in each CSV method to prevent any accidental output buffering from interfering with the stream.
