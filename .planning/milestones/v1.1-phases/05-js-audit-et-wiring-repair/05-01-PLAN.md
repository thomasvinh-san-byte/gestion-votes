  <action>
Create a comprehensive ID contract inventory in `.planning/phases/05-js-audit-et-wiring-repair/05-ID-CONTRACTS.md`.

Methodology:
1. For each JS file in `public/assets/js/pages/` and `public/assets/js/core/`:
   - Extract every `getElementById('X')` call -> ID `X`
   - Extract every `querySelector('#X')` call -> ID `X`
   - Extract every `querySelector('[data-X]')` call -> data attribute `X`
2. For each matching HTML file:
   - Extract all `id="X"` attributes
   - Extract all `data-*` attributes used by JS selectors
3. Cross-reference: mark each selector as OK (found in HTML), ORPHAN (JS references ID not in HTML), MISMATCH (naming convention broken, e.g. camelCase vs kebab-case)

Priority order (highest risk first):
1. `vote.js` / `vote.htmx.html` — 1 confirmed broken selector (`getElementById('voteButtons')` vs `id="vote-buttons"`)
2. `operator-exec.js` / `operator.htmx.html` — 80+ getElementById calls
3. `members.js` / `members.htmx.html` — 54 getElementById calls
4. `meetings.js` / `meetings.htmx.html` — 40 getElementById calls
5. `shell.js` / `partials/sidebar.html` — expects `#sidebarPin`, `#sidebarScroll`, `#sidebarFade`
6. `auth-ui.js` — dynamically creates `#auth-banner` (no HTML counterpart, self-contained)
7. All remaining 15+ page JS files

Output format — markdown table per file group:

```markdown
## vote.js -> vote.htmx.html

| JS Selector | Type | HTML Match | Status |
|-------------|------|------------|--------|
| getElementById('voteButtons') | ID | id="vote-buttons" | MISMATCH — fix to 'vote-buttons' |
| getElementById('voteTimer') | ID | id="voteTimer" | OK |
```

At the end, add a Summary section listing:
- Total selectors audited
- Total OK
- Total ORPHAN (with explanation if dynamic/JS-generated)
- Total MISMATCH (with fix recommendations)

Convention documentation: JS-targeted IDs use camelCase (standard), CSS-targeted classes use kebab-case. Known exceptions: `id="vote-buttons"`, `id="main-content"`, `id="auth-banner"`.

IMPORTANT: Do NOT modify any JS or HTML file in this task. This is audit-only.
