**1. SecurityProvider::headers() — add report-only CSP with nonce:**

In `app/Core/Providers/SecurityProvider.php`, modify the `headers()` method:

Keep the EXISTING enforcing CSP header as-is (it provides baseline defense):
```php
header("Content-Security-Policy: default-src 'self'; "
    . "script-src 'self'; "
    . "style-src 'self' 'unsafe-inline' https://fonts.googleapis.com; "
    . "img-src 'self' data: blob:; font-src 'self' https://fonts.gstatic.com; "
    . "connect-src 'self' ws: wss:; frame-ancestors 'self'; form-action 'self'");
```

ADD a NEW report-only header AFTER the existing one:
```php
$nonce = self::nonce();
header("Content-Security-Policy-Report-Only: default-src 'self'; "
    . "script-src 'nonce-{$nonce}' 'strict-dynamic'; "
    . "style-src 'self' 'nonce-{$nonce}' 'unsafe-inline' https://fonts.googleapis.com; "
    . "img-src 'self' data: blob:; font-src 'self' https://fonts.gstatic.com; "
    . "connect-src 'self' ws: wss:; frame-ancestors 'self'; form-action 'self'");
```

IMPORTANT notes on the report-only header:
- `script-src` uses `'nonce-{$nonce}' 'strict-dynamic'` — NO `'self'` (strict-dynamic ignores it in supporting browsers anyway)
- `style-src` keeps `'unsafe-inline'` alongside the nonce because there are 50+ dynamic inline styles from JS innerHTML (Web Components set inline styles via JS). The nonce covers `<style>` elements; `'unsafe-inline'` covers `element.style` attributes. This is the correct pattern per CSP Level 3 spec: when a nonce is present, `'unsafe-inline'` is ignored for `<style>` elements in supporting browsers, but we need it for older browsers and for style attributes.
- Do NOT add `'unsafe-hashes'` — not needed since all inline event handlers were migrated to `hx-on:*` in Phase 4

**2. Nginx — remove CSP for PHP-served page locations:**

The problem: nginx's server-level `add_header Content-Security-Policy` applies to ALL responses including PHP-served pages. PHP also sets its own CSP. Two CSP headers = browser enforces the most restrictive combination, breaking nonce-based CSP.

Solution: Move the nginx server-level CSP to only apply to non-PHP responses.

In `deploy/nginx.conf` and `deploy/nginx.conf.template`:

a) Remove the server-level `add_header Content-Security-Policy` line (line 48 in nginx.conf). PHP handles CSP for all PHP-served responses.

b) Keep the CSP in the static assets location (`~* \.(css|js|svg|...)$`) — static assets don't go through PHP and need their own CSP.

c) Keep the CSP in the SSE endpoint location (it has its own `add_header` block).

d) For the HTML fallback location (`location /`), which serves `index.html` and other static HTML, add the CSP there too since those don't go through PHP.

e) For the login location (`location = /login`), which serves static `login.html`, add a CSP header since login.html is NOT served through PHP:
```nginx
location = /login {
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header Referrer-Policy "strict-origin-when-cross-origin" always;
    add_header Content-Security-Policy "default-src 'self'; script-src 'self'; style-src 'self' 'unsafe-inline' https://fonts.googleapis.com; img-src 'self' data: blob:; font-src 'self' https://fonts.gstatic.com; connect-src 'self' ws: wss:; frame-ancestors 'self'; form-action 'self';" always;
    add_header Strict-Transport-Security $hsts always;
    try_files /login.html =404;
}
```

IMPORTANT about nginx header inheritance: When a child location uses ANY `add_header`, it replaces ALL parent `add_header` directives. By removing the server-level CSP, page locations that route to PHP will get their CSP from PHP only. Locations that need CSP (static assets, SSE, login) must declare it explicitly.

However, the other server-level security headers (X-Frame-Options, X-Content-Type-Options, Referrer-Policy, Permissions-Policy, COOP, CORP) are also set by PHP via SecurityProvider::headers(). For PHP-served pages, nginx headers and PHP headers coexist (nginx adds them to the response, PHP adds them too). This is fine for non-CSP headers (duplicate same-value headers are harmless). But to be clean, consider keeping the non-CSP security headers at server level (they apply to all responses uniformly).

The cleanest approach:
- Keep ALL server-level `add_header` EXCEPT `Content-Security-Policy` at server level
- Move `Content-Security-Policy` to only the locations that need it (static assets, SSE, login, fallback)
- PHP-served pages get CSP from PHP only

**Create `tests/e2e/specs/csp-enforcement.spec.js`:**

This Playwright spec listens for CSP violation console messages and page errors across all pages, asserting zero violations.

Structure:
```javascript
// @ts-check
const { test, expect } = require('@playwright/test');
const { loginAsOperator } = require('../helpers/login');

// All pages to check for CSP violations
const AUTHED_PAGES = [
    '/dashboard',
    '/wizard',
    '/hub',
    '/operator',
    '/postsession',
    '/validate',
    '/archives',
    '/meetings',
    '/audit',
    '/members',
    '/users',
    '/analytics',
    '/settings',
    '/admin',
    '/help',
    '/email-templates',
    '/report',
    '/trust',
    '/docs',
];

const PUBLIC_PAGES = [
    '/public',
    '/login',
];

test.describe('CSP Nonce Enforcement', () => {

    test('zero CSP violations on authenticated pages', async ({ page }) => {
        const violations = [];

        page.on('console', msg => {
            const text = msg.text();
            if (text.includes('[Report Only]') ||
                text.includes('Content Security Policy') ||
                text.includes('content-security-policy')) {
                violations.push({ url: page.url(), type: 'console', message: text });
            }
        });

        page.on('pageerror', error => {
            // Only capture CSP-related page errors
            if (error.message.includes('Content Security Policy') ||
                error.message.includes('CSP')) {
                violations.push({ url: page.url(), type: 'pageerror', message: error.message });
            }
        });

        // Login first
        await loginAsOperator(page);

        for (const pageUrl of AUTHED_PAGES) {
            await page.goto(pageUrl, { waitUntil: 'networkidle' });
            // Wait for any deferred script loading
            await page.waitForTimeout(500);
        }

        if (violations.length > 0) {
            console.log('CSP Violations found:');
            violations.forEach(v => console.log(`  ${v.url}: [${v.type}] ${v.message}`));
        }

        expect(violations).toEqual([]);
    });

    test('zero CSP violations on public pages', async ({ page }) => {
        const violations = [];

        page.on('console', msg => {
            const text = msg.text();
            if (text.includes('[Report Only]') ||
                text.includes('Content Security Policy') ||
                text.includes('content-security-policy')) {
                violations.push({ url: page.url(), type: 'console', message: text });
            }
        });

        page.on('pageerror', error => {
            if (error.message.includes('Content Security Policy') ||
                error.message.includes('CSP')) {
                violations.push({ url: page.url(), type: 'pageerror', message: error.message });
            }
        });

        for (const pageUrl of PUBLIC_PAGES) {
            await page.goto(pageUrl, { waitUntil: 'networkidle' });
            await page.waitForTimeout(500);
        }

        if (violations.length > 0) {
            console.log('CSP Violations found:');
            violations.forEach(v => console.log(`  ${v.url}: [${v.type}] ${v.message}`));
        }

        expect(violations).toEqual([]);
    });

    test('CSP report-only header is present on PHP-served pages', async ({ page }) => {
        await loginAsOperator(page);

        const response = await page.goto('/dashboard', { waitUntil: 'networkidle' });
        const headers = response.headers();

        // Report-only header must be present
        const cspRO = headers['content-security-policy-report-only'];
        expect(cspRO).toBeDefined();
        expect(cspRO).toContain('strict-dynamic');
        expect(cspRO).toContain('nonce-');

        // script-src in report-only must NOT contain unsafe-inline
        const scriptSrc = cspRO.match(/script-src\s+([^;]+)/);
        expect(scriptSrc).toBeTruthy();
        expect(scriptSrc[1]).not.toContain("'unsafe-inline'");
    });
});
```
