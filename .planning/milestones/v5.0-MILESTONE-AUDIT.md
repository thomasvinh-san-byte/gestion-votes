## Integration Findings

### Finding 1: E2E Seed Data Gap (Minor)
`workflow-meeting.spec.js` depends on `04_e2e.sql` seed data loaded manually into local Docker. CI e2e job starts fresh with default seeds (01-03). Tests may fail in CI if they depend on "Conseil Municipal" meeting data.
**Fix:** Add `04_e2e.sql` to CI e2e job stack startup.

### Finding 2: Migration Idempotency Not in CI (Minor)
`validate-migrations.sh --syntax-only` runs in CI but the full idempotency double-pass requires live PostgreSQL. This is a documented design decision — full validation is for local/pre-deploy use.
**Fix (optional):** Add idempotency pass to CI integration job.
