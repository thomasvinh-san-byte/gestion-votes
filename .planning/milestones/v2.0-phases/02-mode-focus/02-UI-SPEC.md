# Phase 2 — Mode Focus: UI Design Contract

> Visual and interaction contract for Phase 2. Generated by gsd-ui-researcher.
> Stack: PHP 8.4 + HTMX + custom CSS design system (no Tailwind, no shadcn).
> Reuses 100% of Phase 1 tokens. No new design tokens introduced.

---

## Phase Goal

L'opérateur peut basculer vers une vue épurée à 5 zones qui masque les informations secondaires et conserve uniquement les contrôles essentiels pour conduire le scrutin.

Requirements covered: FOCUS-01, FOCUS-02, FOCUS-03.

## The 5 Zones (FOCUS-01)

In focus mode, exactly 5 zones are visible. Everything else is hidden.

| # | Zone                | Source element(s)                       | Width    | Height | Role               |
|---|---------------------|-----------------------------------------|----------|--------|--------------------|
| 1 | Titre motion        | `.op-resolution-title` (existing)       | 100%     | 56px   | Context anchor     |
| 2 | Résultat vote       | `.op-vote-card` (Pour/Contre/Abstention) | 60% (left col) | fluid  | **Focal point**    |
| 3 | Quorum status       | Quorum tag/value reused from KPI strip   | 40% (right col, top half) | 80px   | Status confirmation |
| 4 | Chronomètre         | `.op-live-chrono` enlarged              | 40% (right col, bottom half) | 80px | Time pressure       |
| 5 | Actions             | `.op-action-bar` (Proclamer / Fermer)   | 100%     | 80px   | Primary CTAs       |

**Single focal point:** Zone 2 (Résultat vote — vote counters). The user's eye is anchored on Pour/Contre/Abstention numbers. Quorum and chronomètre are supporting context — visible but visually recessive.

### Hidden in focus mode

These are present in normal mode but hidden when focus mode is active:

- `.op-sidebar` (left agenda — 280px column hidden, main expands to full width)
- `.op-exec-status-bar` (séance title + connectés + elapsed)
- `.op-kpi-strip` (PRÉSENTS / QUORUM / ONT VOTÉ / RÉSOLUTION / DURÉE / EN LIGNE / INACTIFS — 7 KPI items)
- `.op-tags` (correspondance, procurations, quorum tags)
- `.op-resolution-progress` (segmented motion progress bar)
- `.op-guidance` / `exec-close-banner` (guidance bar)
- `.op-tabs` (sub-tabs inside resolution card)
- `.op-checklist-panel` (Phase 1 right-side checklist — hidden in focus mode; checklist is monitoring, not action)
- `.op-exec-header-right` Projection + Clôturer secondary buttons (kept as small icon-only buttons in zone 5 actions)

### Focus mode horizontal layout (1080p, 1920×1080)

```
┌────────────────────────────────────────────────────────────────────┐
│  Zone 1 — TITRE MOTION (full width, 56px)                          │ Toggle btn (top-right corner)
├──────────────────────────────────────────┬─────────────────────────┤
│                                          │  Zone 3 — QUORUM        │
│  Zone 2 — RÉSULTAT VOTE                  │  Atteint 42/60 (70%)    │
│  (FOCAL POINT — Pour/Contre/Abstention,  ├─────────────────────────┤
│   bars, large counters, live dot)        │  Zone 4 — CHRONOMÈTRE   │
│                                          │  00:02:34               │
│  60% width                               │                         │
│                                          │  40% width              │
├──────────────────────────────────────────┴─────────────────────────┤
│  Zone 5 — ACTIONS (Proclamer • Fermer le vote • Passer motion)     │
│  full width, 80px, sticky bottom                                   │
└────────────────────────────────────────────────────────────────────┘
```

CSS grid template (applied to `#viewExecContent` when `body.focus-mode` is set):

```css
.view-exec.focus-mode #viewExecContent {
  display: grid;
  grid-template-columns: 3fr 2fr;
  grid-template-rows: auto 1fr auto;
  grid-template-areas:
    "title  title"
    "result side"
    "actions actions";
  gap: var(--space-4);
  height: 100%;
  padding: var(--space-4);
}
.view-exec.focus-mode .op-focus-title    { grid-area: title;   }
.view-exec.focus-mode .op-vote-card      { grid-area: result;  }
.view-exec.focus-mode .op-focus-side     { grid-area: side; display: grid; grid-template-rows: 1fr 1fr; gap: var(--space-4); }
.view-exec.focus-mode .op-action-bar     { grid-area: actions; }
```

The 5 zones consume the full viewport. No scrolling required at 1080p (FOCUS-02).

## Typography

Two font weights only — same as Phase 1: **500** (`--font-medium`) for labels, **600** (`--font-semibold`) for titles, values, and CTAs. Numbers use `--font-mono` for tabular alignment.

| Role                       | CSS Token        | Size          | Weight          | Line height | Justification |
|----------------------------|------------------|---------------|-----------------|-------------|---------------|
| Motion title (Zone 1)      | `--text-2xl`     | 24px          | 600 semibold    | 1.2         | Anchors context; not the focal point so smaller than vote counters |
| Vote counters (Zone 2)     | custom           | **4rem (64px)** | 700 mono       | 1.0         | **FOCAL POINT** — readable across the room; reuses `.op-vote-count` enlarged via override |
| Vote counter labels        | `--text-sm`      | 13px          | 600 semibold    | 1.2         | "POUR / CONTRE / ABSTENTION" uppercase, recessive |
| Quorum value (Zone 3)      | `--text-3xl`     | 30px          | 600 semibold mono | 1.1       | Glanceable, second tier under vote counters |
| Quorum label               | `--text-sm`      | 13px          | 500 medium      | 1.2         | "Quorum" |
| Chronomètre (Zone 4)       | `--text-3xl`     | 30px          | 600 semibold mono | 1.1       | Tabular numerals, same tier as quorum |
| Chronomètre label          | `--text-sm`      | 13px          | 500 medium      | 1.2         | "Temps écoulé" |
| Action button label        | `--text-base`    | 16px          | 600 semibold    | 1.2         | Primary CTAs visible at distance |
| Toggle button label        | `--text-sm`      | 13px          | 500 medium      | 1.2         | "Mode focus" / "Vue complète" |

**Visual hierarchy (size descending):**
1. Vote counters — 64px (focal point)
2. Quorum value + Chronomètre — 30px (supporting context, equal weight)
3. Motion title — 24px (anchor, recessive)
4. Action button labels — 16px (always-on CTAs)
5. Counter labels / chronomètre label / toggle — 13px

This scale exists already in design-system.css. The 64px vote counter override is added inline as a focus-mode specific rule — no new token.

## Color (60/30/10 split — reuses Phase 1 tokens)

| Role             | CSS Token                  | % of canvas | Usage in focus mode                                          |
|------------------|----------------------------|-------------|--------------------------------------------------------------|
| Dominant (60%)   | `--color-surface`          | 60%         | Page background, vote card background, quorum/chrono cards   |
| Secondary (30%)  | `--color-bg-subtle`        | 30%         | Action bar background, toggle button background              |
| Accent (10%)     | `--color-primary`          | 10%         | Live dot, active vote card border, Proclamer button background, toggle "active" state |

**Single accent reservation:** The `--color-primary` accent is reserved for:
1. The pulsing live dot inside the vote card (when vote is open)
2. The 2px primary border around `.op-vote-card--active`
3. The `Proclamer` primary CTA button background
4. The toggle button `aria-pressed="true"` indicator (focus mode active)

**Status colors (informational, not chromatic accents):**
- `--color-success` — Pour count number color, quorum atteint icon, "Quorum atteint" text
- `--color-danger` — Contre count number color, "Fermer le vote" button (warning treatment), quorum non atteint
- `--color-warning` — Abstention count number color, vote card warning state if needed

**Focal point color treatment:**
The vote counters (Zone 2) carry the only saturated colors (success/danger/warning on the numbers). Zones 3 and 4 use neutral text on subtle backgrounds. This forces the eye to the vote result first.

**Borders and shadows:**
- Each zone card: `border: 1px solid var(--color-border)` + `border-radius: var(--radius-lg)` (8px)
- Focal vote card (Zone 2) when vote is open: `border: 2px solid var(--color-primary)` + `box-shadow: 0 0 0 2px var(--color-primary-subtle)` (reuses existing `.op-vote-card--active`)
- No drop shadows on Zones 3, 4, 5 — keep them visually recessive

## Toggle Button Specification (FOCUS-03)

The toggle button is the single new control introduced by this phase.

| Property        | Value                                                              |
|-----------------|--------------------------------------------------------------------|
| Element         | `<button id="opBtnFocusToggle" class="op-focus-toggle" type="button">` |
| Position        | Top-right of `.op-exec-header`, before the Projection button       |
| Size            | 44px height, auto width with `padding: var(--space-2) var(--space-4)` |
| Variant         | `btn btn-sm btn-secondary` reusing existing button system          |
| Icon            | `icon-maximize` (focus inactive) / `icon-minimize` (focus active)  |
| State indicator | `aria-pressed="true|false"` |
| Text            | "Mode focus" → "Vue complète" (toggles)                            |
| Persistence     | `sessionStorage.setItem('opFocusMode', '1' | '0')` — survives reloads within session, resets on logout |
| Restore on load | On `viewExec` show, JS reads `sessionStorage.opFocusMode` and applies `body.focus-mode` class accordingly |

The toggle is **always visible** — both in normal exec mode (offering "Mode focus") and in focus mode itself (offering "Vue complète"). It never disappears.

In focus mode, the toggle moves to the top-right corner of the viewport (absolute positioned within Zone 1's area, 16px inset) so it remains discoverable without competing with content.

## Layout Integration (Reuses Phase 1 structure)

Phase 1 introduced `.op-exec-body` as a flex row wrapping `.op-exec-main` + `.op-checklist-panel`. Phase 2 does NOT restructure this; it overlays a CSS state class.

```
body.focus-mode  (toggle on)
  → .view-exec inherits a transformed grid
  → .op-sidebar { display: none }
  → .op-exec-status-bar { display: none }
  → .op-kpi-strip { display: none }
  → .op-tags { display: none }
  → .op-resolution-progress { display: none }
  → .op-guidance { display: none }
  → .op-tabs { display: none }
  → .op-checklist-panel { display: none }
  → .op-exec-body becomes the 5-zone grid container
  → .op-resolution-card hides everything except the .op-vote-card child
  → .op-action-bar becomes sticky / grid-area: actions
```

The state class is applied to `<body>` so the sidebar (rendered outside `.op-main`) can be hidden via CSS selector. Toggling is a pure CSS state change — no DOM rebuild, no HTMX swap.

## Interaction Contract

### Toggle behavior (FOCUS-03)

| Trigger                          | Action                                                            |
|----------------------------------|-------------------------------------------------------------------|
| User clicks `#opBtnFocusToggle`  | JS toggles `body.focus-mode` class + writes sessionStorage        |
| User reloads page during session | JS reads sessionStorage on `viewExec` show and re-applies state   |
| User logs out / closes tab       | sessionStorage cleared (browser default) — focus mode resets      |
| User switches to non-exec view (`viewSetup`, `viewPostsession`) | `body.focus-mode` class persists but visually inert (rules scoped to `.view-exec`) |
| User returns to `viewExec`       | Focus mode rules reactivate automatically                         |

### Vote state inside focus mode (Zone 2)

- Vote not started: card shows "Aucun vote en cours" placeholder, counters at 0, faded.
- Vote in progress: `.op-vote-card--active` class applied — primary border + live dot pulses.
- Vote closed: counters lock at final values, live dot stops, "Proclamer" button becomes primary.

This logic is **unchanged from existing operator.js** — focus mode only affects visibility/layout, never vote state.

### Quorum card (Zone 3)

- Pulls value from existing `opKpiQuorum` and `opKpiPresent` data.
- Updates via existing SSE handlers (`attendance.updated`, `quorum.updated`) — no new wiring.
- Color of value: `--color-success` if atteint, `--color-danger` if non atteint.

### Chronomètre (Zone 4)

- Reuses existing `opExecTimer` + `execSessionTimer` data source.
- Single timer in focus mode (whichever is the live session timer).
- Updates every second via existing `setInterval` in operator.js — no new timer.

### Actions (Zone 5)

- Three buttons in fixed order, all from existing markup:
  1. **Proclamer** (`#opBtnProclaim`) — primary, success treatment when enabled
  2. **Fermer le vote** (`#opBtnToggleVote`) — warning treatment
  3. **Passer à la motion suivante** — needs adding (or reusing `#opBtnNextMotion` if it exists in the agenda sidebar; verify in plan phase)
- All buttons keep their existing `disabled` state logic from operator.js. Focus mode does NOT change when buttons enable/disable.

### Reduced motion

- Live dot pulse disabled under `prefers-reduced-motion: reduce` (already handled in Phase 1 for the existing `.op-live-dot`).
- Toggle transition: instant if reduced motion, 200ms otherwise.

## Accessibility Contract

- Toggle button: `aria-pressed="true|false"`, `aria-label` reflects current intent (activate vs. deactivate).
- Hidden zones use `display: none` (not `[hidden]`) to avoid the v1.4 `[hidden]+display:flex` overlay class — and remain `aria-hidden` consistent with display state.
- Focus order in focus mode: Toggle → Motion title (read-only) → Vote counters (read-only) → Quorum (read-only) → Chronomètre (read-only) → Proclamer → Fermer le vote → Passer.
- `role="region"` + `aria-label` on each of the 5 zones for screen reader navigation.
- Live regions: vote counters keep `aria-live="polite"` (existing). No additional announcements added by focus mode itself — focus mode is a visual reorganization, not a content change.
- The `body.focus-mode` state itself is announced once when toggled via the toggle button's `aria-pressed` change — no redundant alert.

## Animation Contract

| Animation               | Duration  | Easing            | Trigger                     | Reduced-motion |
|-------------------------|-----------|-------------------|-----------------------------|----------------|
| Focus mode enter        | 200ms     | `--ease-standard` | Toggle click → focus on     | Disabled (instant) |
| Focus mode exit         | 200ms     | `--ease-standard` | Toggle click → focus off    | Disabled (instant) |
| Live dot pulse          | 1.5s loop | ease-in-out       | `.op-vote-card--active`     | Disabled (existing) |
| Vote counter changes    | none in this phase | —        | —                           | n/a (Phase 3)  |

The grid layout transition uses CSS `transition: grid-template-columns 200ms` + `transition: opacity 200ms` on hidden elements. Hidden zones fade out, then `display: none` applies after transition end. No JavaScript animation library used.

## Single Focal Point Verification

In focus mode the eye lands first on **Zone 2 — Vote counters** (Pour/Contre/Abstention). Verification:
- Largest type on screen: 64px counters (next largest is 30px — half the size)
- Saturated colors only on counter numbers (success green / danger red / warning amber)
- 60% width, top-left dominant grid area
- Live dot pulse animates inside this zone only when a vote is open
- Border treatment (2px primary border) applied only to this card during active vote

Quorum and chronomètre are deliberately styled as recessive supporting context: smaller type, neutral colors, no animation, smaller cards. Actions (Zone 5) are full-width but visually neutral until enabled, at which point Proclamer becomes the secondary attention magnet.

### What NOT to break (integration safety)

The following existing systems must continue to work identically with focus mode toggled on or off:

| Existing system                          | Risk                                                | Safeguard                                         |
|------------------------------------------|-----------------------------------------------------|---------------------------------------------------|
| Phase 1 checklist panel                  | Focus mode hides it — must reappear cleanly on toggle off | Use `display: none` in CSS, not DOM removal. State preserved. |
| Phase 1 SSE disconnect banner            | Banner sits inside checklist panel (now hidden)     | When focus mode active AND SSE offline, banner becomes a top-of-screen toast variant — out of scope here, accept that banner is invisible in focus mode and document this constraint in plan |
| Existing chronomètre `setInterval`       | Two timers (`opExecTimer` + `execSessionTimer`) update via JS | Focus mode reads from same DOM IDs, just hidden in normal positions and shown enlarged in Zone 4 — no setInterval changes |
| Ballot cache / `opKpiVoted` data         | Vote counters in Zone 2 are reused `.op-vote-card`  | Focus mode reuses the same DOM nodes — counters remain wired to existing `opKpiVoted` / `op-vote-count` updaters |
| Idempotency (v1.7) on Proclamer / Fermer | Action buttons in Zone 5 are the same DOM nodes     | Buttons keep their existing `id`, `data-*` attributes, and HTMX wiring — focus mode only restyles, never re-binds |
| Mode state (preparation vs. exécution)   | Focus mode only valid inside `viewExec`             | CSS rules scoped to `body.focus-mode .view-exec` — preparation views unaffected |
| Quorum overlay modal                     | Modal uses fixed inset:0 z-index 1000               | Focus mode has no z-index changes; modal still appears over focus mode UI |
| Transition card overlay                  | Same — fixed positioning unaffected                 | No conflict |
| Keyboard shortcuts                       | Out of scope per project memory                     | Not added in this phase |
| Sidebar agenda persistence               | Sidebar hidden in focus mode                        | `display: none` keeps DOM intact; clicking a motion in agenda is impossible during focus mode but that's acceptable — operator must exit focus mode to navigate motions |

The JS toggle implementation must be **idempotent**: clicking the toggle twice in rapid succession results in the same final state as one click (no race condition with sessionStorage write).
