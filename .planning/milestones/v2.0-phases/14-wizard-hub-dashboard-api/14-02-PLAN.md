Fix the loadData() function in hub.js (around line 387-423):

1. **Line 393 — Change API endpoint:**
   Current (WRONG): `var res = await window.api('/api/v1/meetings.php?id=' + encodeURIComponent(sessionId));`
   The `/api/v1/meetings` endpoint is a list endpoint (MeetingsController::index) that ignores `?id=`.
   The correct single-meeting endpoint is `/api/v1/wizard_status` which accepts `?meeting_id=X`.
   Fix to: `var res = await window.api('/api/v1/wizard_status?meeting_id=' + encodeURIComponent(sessionId));`

2. **Update mapApiDataToSession to handle wizard_status response shape:**
   The wizard_status endpoint returns flat fields: `members_count`, `motions_total`, `meeting_title`, `meeting_status`.
   The existing mapApiDataToSession already handles `member_count` (line 342-343) but the field is `members_count` (plural).

   In the mapApiDataToSession function (line 336+), add support for the wizard_status field names:
   - Add `typeof data.members_count === 'number'` check before the existing `data.member_count` check
   - Add `typeof data.motions_total === 'number'` as a resolution count source
   - Map `data.meeting_title` to title (in addition to existing `data.title`)
   - Map `data.meeting_status` to status (in addition to existing `data.status`)

   Specifically, update mapApiDataToSession to handle these fields at the top:
   ```javascript
   // Normalize wizard_status field names to hub conventions
   var normalized = Object.assign({}, data);
   if (data.meeting_title && !data.title) normalized.title = data.meeting_title;
   if (data.meeting_status && !data.status) normalized.status = data.meeting_status;
   if (typeof data.members_count === 'number' && !data.member_count) normalized.member_count = data.members_count;
   if (typeof data.motions_total === 'number' && !data.resolution_count) normalized.resolution_count = data.motions_total;
   data = normalized;
   ```
   Place this at the very beginning of mapApiDataToSession, before the existing member count logic.

3. **Ensure the data path is correct:**
   The current code at line 394 does: `if (res && res.body && res.body.ok && res.body.data)` — this is correct for the global api() function which returns `{ status, body: { ok, data } }`. No change needed here.

Fix the loadDashboard() function in dashboard.js (around line 68-122):

**Problem:** dashboard.js uses `Utils.apiGet` which returns `response.json()` directly — that means the `.then(function(data))` receives the full `{ ok: true, data: { meetings: [...], meeting: {...}, ... } }` envelope. But the code at line 80 reads `data.upcoming_count` which doesn't exist in the response.

**The actual /api/v1/dashboard response** (from DashboardController::index) contains:
- `data.meetings` — array of all meetings (with status field)
- `data.meeting` — single meeting object (if meeting_id provided) or null
- `data.attendance` — attendance stats
- No `upcoming_count`, `live_count`, `pending_invitations`, `pending_pv` fields

**Fix approach — compute KPIs from meetings array:**

1. **Unwrap the response envelope (after line 75):**
   The `.then(function(data))` receives `{ ok, data: {...} }`. Add at the top of the callback:
   ```javascript
   if (!data || !data.ok) { showFallback(); return; }
   var d = data.data; // unwrap envelope
   var meetings = Array.isArray(d.meetings) ? d.meetings : [];
   ```

2. **Compute KPI values from meetings array (replace lines 79-86):**
   ```javascript
   var now = new Date();
   var upcoming = meetings.filter(function(m) {
     return m.status === 'draft' || m.status === 'planned' || (m.status !== 'live' && m.status !== 'paused' && m.status !== 'ended' && m.status !== 'archived' && m.scheduled_at && new Date(m.scheduled_at) > now);
   });
   var live = meetings.filter(function(m) { return m.status === 'live' || m.status === 'paused'; });

   el = document.getElementById('kpiSeances');
   if (el) el.textContent = upcoming.length;
   el = document.getElementById('kpiEnCours');
   if (el) el.textContent = live.length;
   el = document.getElementById('kpiConvoc');
   if (el) el.textContent = 0; // convocation data not in dashboard API — show 0
   el = document.getElementById('kpiPV');
   if (el) el.textContent = meetings.filter(function(m) { return m.status === 'ended'; }).length;
   ```

3. **Fix urgent action section (lines 89-98):**
   Replace `data.urgent_action` with logic derived from meetings:
   ```javascript
   // Show urgent card if there's a live meeting
   var liveMeeting = live.length > 0 ? live[0] : null;
   if (liveMeeting) {
     var uTitle = document.getElementById('urgentTitle');
     var uSub = document.getElementById('urgentSub');
     if (uTitle) uTitle.textContent = 'Seance en cours';
     if (uSub) uSub.textContent = liveMeeting.title || '';
   } else {
     var urgentCard = document.getElementById('actionUrgente');
     if (urgentCard) urgentCard.hidden = true;
   }
   ```

   if (prochaines && upcoming.length > 0) {
