  <name>Task 1: Rewrite meetings.js core — data fetch, filter pills, search, sort, list rendering</name>
  <files>public/assets/js/pages/meetings.js</files>
  <action>
  Rewrite meetings.js as a clean IIFE module. Preserve the `var` keyword convention and IIFE pattern.

  STRUCTURE:
  ```
  (function() {
    'use strict';
    // === STATE ===
    // === CONSTANTS ===
    // === DOM REFS ===
    // === DATA LOADING ===
    // === FILTER PILLS ===
    // === SEARCH & SORT ===
    // === SESSION LIST RENDERING ===
    // === EMPTY STATES ===
    // === POPOVER MENUS ===
    // === EDIT/DELETE MODALS ===
    // === CALENDAR VIEW ===
    // === ONBOARDING BANNER ===
    // === VIEW TOGGLE ===
    // === PAGINATION ===
    // === INIT ===
  })();
  ```

  1. STATE: `var allMeetings = []; var currentFilter = 'all'; var searchText = ''; var sortMode = 'date_desc'; var currentView = 'list'; var calendarDate = new Date(); var currentPage = 1; var PER_PAGE = 12;`

  2. DATA LOADING: `function loadMeetings()` — call `Utils.apiGet('/api/v1/meetings')`, store in allMeetings, call updateFilterCounts(), renderCurrentView(). On error, show empty state with error message.

  3. FILTER PILLS: `function initFilterPills()` — add click handlers on each `.filter-pill` button. On click: set currentFilter, toggle `.active` class (remove from siblings, add to clicked), set aria-selected, reset currentPage to 1, call renderCurrentView(). `function updateFilterCounts()` — count meetings by status category and update #countAll, #countUpcoming, #countLive, #countCompleted textContent.

  Filter mapping:
  - `all` = all meetings
  - `upcoming` = status in ['scheduled', 'frozen', 'draft', 'convocations']
  - `live` = status in ['live', 'paused']
  - `completed` = status in ['closed', 'validated', 'archived', 'pv_sent']

  4. SEARCH: `function initSearch()` — on #meetingsSearch input event (debounced 250ms via Shared.debounce), set searchText, reset page, renderCurrentView(). Filter: `meeting.title.toLowerCase().includes(searchText.toLowerCase())`.

  5. SORT: `function initSort()` — on #meetingsSort change, set sortMode, renderCurrentView(). Sort functions for date_desc (newest first), date_asc, title_az, title_za.

  6. SESSION LIST RENDERING: `function renderSessionList()`:
     - Apply filter -> search -> sort pipeline to allMeetings
     - Paginate result
     - For each meeting, call renderSessionItem(meeting) to build HTML
     - Set innerHTML of #meetingsList
     - Update #meetingsCount text
     - If empty after filtering, show contextual empty state

  7. `function renderSessionItem(m)` — returns HTML string:
     ```html
     <div class="session-item" data-id="${m.id}" onclick="location.href='/hub.htmx.html?meeting_id=${m.id}'">
       <span class="session-dot ${dotClass}"></span>
       <div class="session-info">
         <div class="session-title">${Shared.escapeHtml(m.title)}</div>
         <div class="session-meta">
           <span class="session-meta-item date">${Shared.formatDate(m.scheduled_at)}</span>
           <span class="session-meta-item participants">${m.participant_count || 0} participants</span>
           <span class="session-meta-item resolutions">${m.motions_count || 0} resolutions</span>
           <span class="session-meta-item quorum">${formatQuorum(m)}</span>
         </div>
       </div>
       <span class="tag tag-${tagVariant}">${statusLabel}</span>
       <div class="session-actions">
         <button class="btn btn-ghost btn-icon btn-sm session-menu-btn" data-meeting-id="${m.id}" aria-label="Actions">
           <svg width="16" height="16" viewBox="0 0 24 24" fill="none" stroke="currentColor" stroke-width="2"><circle cx="12" cy="5" r="1"/><circle cx="12" cy="12" r="1"/><circle cx="12" cy="19" r="1"/></svg>
         </button>
       </div>
     </div>
     ```
     Status dot class mapping: draft -> .draft, scheduled/frozen/convocations -> .upcoming, live/paused -> .live, closed/validated/archived/pv_sent -> .completed.
     Tag variant mapping: draft -> muted, scheduled -> accent, live -> danger, closed -> success, archived -> muted.
     Status labels in French: Brouillon, Planifiee, En cours, Terminee, Archivee, PV envoye, Convocations.

  8. POPOVER MENUS: Use event delegation on #meetingsList for click on `.session-menu-btn`. Create an ag-popover with contextual items based on meeting status per user decision:
     - Draft: Modifier (opens edit modal), Supprimer (opens delete modal)
     - Scheduled: Voir (navigate to hub), Annuler (future — show toast "Non implemente")
     - Completed/Archived: Voir (navigate to hub), Archiver (future)
     Position popover near the clicked button.

  9. EMPTY STATES: `function renderEmptyState(filter)` — return Shared.emptyState() with contextual messaging:
     - filter 'all' and 0 total: icon 'calendar', title "Aucune seance", description "Creez votre premiere seance pour commencer.", actionHtml: button linking to /wizard.htmx.html
     - filter 'upcoming': title "Aucune seance a venir", description "Toutes vos seances sont terminees ou en cours."
     - filter 'live': title "Aucune seance en cours", description "Lancez une seance depuis la page operateur."
     - filter 'completed': title "Aucune seance terminee", description "Les seances terminees apparaitront ici."
     Use HTML entities for French accents in string literals.

  10. EDIT/DELETE MODALS: Preserve the existing edit/delete modal open/close logic (event listeners on #editMeetingSaveBtn, #deleteMeetingConfirmBtn, data-close-modal buttons). Keep the API calls for PUT/DELETE. After successful edit/delete, call loadMeetings() to refresh the list.

  11. PAGINATION: Reuse the existing pagination pattern — render page buttons in #meetingsPagination, each click sets currentPage and re-renders.

  12. VIEW TOGGLE: `function initViewToggle()` — on `.view-toggle-btn` click, set currentView, toggle active class, show/hide #meetingsList vs #calendarContainer, call renderCurrentView(). `function renderCurrentView()` — if list view, call renderSessionList(); if calendar, call renderCalendar().

  13. ONBOARDING BANNER: Check `data-session-driven="true"` on banner. If set, after loadMeetings() completes, check if allMeetings.length === 0 — show banner. If allMeetings.length > 0 — hide banner and set localStorage flag. Keep existing dismiss button handlers.

  14. INIT: On DOMContentLoaded: initFilterPills(), initSearch(), initSort(), initViewToggle(), loadMeetings(), init edit/delete modal listeners.
