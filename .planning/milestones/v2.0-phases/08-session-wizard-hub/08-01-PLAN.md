<task type="auto">
  <name>Task 2: Add localStorage draft, drag-drop reorder, API wiring, and validation gating to wizard.js</name>
  <files>public/assets/js/pages/wizard.js</files>
  <action>
Read the existing wizard.js to understand current structure (IIFE, var keyword, step navigation, members/resolutions arrays, button handlers).

Add the following features to wizard.js while preserving existing code structure and `var` keyword convention:

1. **localStorage draft auto-save/restore** (per 08-RESEARCH.md Pattern 2):
   - `var DRAFT_KEY = 'ag-vote-wizard-draft';`
   - `saveDraft()` — serializes currentStep, Step 1 field values (wizTitle, wizType, wizDate, wizTimeHH, wizTimeMM, wizPlace, wizAddr, wizQuorum), members array, resolutions array to localStorage
   - `restoreDraft()` — on page load (inside init()), restores all field values, members, resolutions from localStorage; calls showStep(draft.step) to resume at saved step
   - `clearDraft()` — removes draft key; called after successful API creation
   - Call `saveDraft()` on every 'Suivant' click and on Step 1 field blur events
   - Call `restoreDraft()` in init() before first showStep()

2. **Strict validation gating** (per 08-RESEARCH.md Pattern 5, locked decision):
   - `validateStep(n)` function:
     - Step 0: title not empty, date not empty, HH is 0-23, MM is 0-59
     - Step 1: members.length > 0
     - Step 2: resolutions.length > 0
     - Step 3: always true (recap)
   - Each 'Suivant' button handler calls validateStep(n) first; if false, highlight required fields with `.field-error` class and show inline error message; do NOT advance step
   - Use a `showFieldErrors(n)` helper to add/remove error styling

3. **HTML5 drag-and-drop resolution reorder** (per 08-RESEARCH.md Pattern 3, locked decision):
   - In Step 3 resolution list rendering: set `row.draggable = true`, add drag handle visual (grip icon `⠿`)
   - `onDragStart` — store source index, add `.dragging` class
   - `onDragOver` — preventDefault, add `.drag-over` class
   - `onDragLeave` — remove `.drag-over` class
   - `onDrop` — splice resolutions array to reorder, call saveDraft(), re-render
   - `onDragEnd` — re-render (clears drag states)

4. **API wiring for btnCreate** (per 08-RESEARCH.md code example):
   - Wire the existing btnCreate handler to call `api('POST', '/api/v1/meetings', payload)`
   - `buildPayload()` — constructs object from wizard state (title, type, date/time, place, address, quorum, members, resolutions)
   - On success: call `clearDraft()`, queue success toast via `sessionStorage.setItem('ag-vote-toast', JSON.stringify({msg: 'Seance creee avec succes', type: 'success'}))`, redirect to `/hub.htmx.html?id=` + res.id
   - On failure: re-enable button, show error toast via `Shared.showToast()`
   - Disable button + show "Creation..." text while request is in flight

5. **Global voting rule default** (locked decision — Step 1 sets default, Step 3 inherits):
   - Add a voting rule select/input in Step 1 (or use existing wizQuorum field logic)
   - When adding a resolution in Step 3, pre-fill the voting rule from the Step 1 default
   - Allow per-resolution override

Use `var` keyword everywhere. Follow existing IIFE pattern. Use `api()` global function (not fetch). Use `escapeHtml()` for all user content in innerHTML.
