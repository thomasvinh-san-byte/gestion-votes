  <files>public/assets/js/pages/hub.js</files>
  <action>
    Gap: hub.js loadData() uses hardcoded demo data. Must call api() to load real session data from `/api/v1/meetings.php?id={id}`, falling back to demo data when no ID or API fails.

    Rewrite `loadData()` in `public/assets/js/pages/hub.js` to:

    1. Extract session ID from URL: `var params = new URLSearchParams(window.location.search); var sessionId = params.get('id');`

    2. If sessionId exists, call `window.api('/api/v1/meetings.php?id=' + encodeURIComponent(sessionId))` to fetch real data.

    3. Map the API response to the sessionData shape used by renderKpis/renderChecklist. The API response follows the pattern `res.body.ok && res.body.data` (see postsession.js, trust.js, validate.js, report.js for established pattern). Map fields:
       - `sessionData.title` = `data.title`
       - `sessionData.date` = `data.date`
       - `sessionData.memberCount` = `data.participants` or `data.members` length or a count field (use what the API returns)
       - `sessionData.resolutionCount` = `data.resolutions` length or count
       - `sessionData.convocationsSent` = derive from data (e.g., `data.convocation_status === 'sent'` or boolean)
       - `sessionData.documentCount` = `data.documents` length or count
       - Map KPI display values from real data

    4. Update identity fields (hubTitle, hubDate, hubPlace, hubParticipants) from real data.

    5. On API failure or missing sessionId, fall back to current demo data (keep existing hardcoded values as the fallback). Log a console.warn when falling back so developers know.

    6. For documents, if API returns documents array, use it; otherwise fall back to demo files array.

    7. Since `api()` is async and hub.js currently uses synchronous patterns with `var`, make loadData an async function. The IIFE pattern supports this. Change `loadData()` to `async function loadData()` and use `await` for the api call, wrapped in try/catch.

    8. In `init()`, since loadData is now async, call it with: `loadData().catch(function(e) { console.warn('Hub loadData error:', e); });` — this ensures errors don't break the page.

    IMPORTANT: Use `window.api` (not just `api`) to match the postsession.js/shell.js pattern — the api function is a global on window from utils.js. Use `var` keyword. Keep the IIFE pattern.

    The fallback ensures the hub still works in development without a running backend — this was the reason it was deferred, but with fallback it satisfies both needs.
