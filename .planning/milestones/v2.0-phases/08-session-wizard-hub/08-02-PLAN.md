Refactor hub.js with the following changes while preserving `var` keyword convention and IIFE pattern:

1. **Horizontal colorful status bar** (HUB-01, per 08-RESEARCH.md Pattern 4):
   - Add `renderStatusBar()` function:
     - Gets `#hubStatusBar` element
     - Iterates HUB_STEPS array; for each step:
       - If index < currentStep: `.done` class, background = `var(--color-success)`
       - If index === currentStep: `.active` class, background = step's color (from HUB_STEPS[i].color)
       - If index > currentStep: no extra class, background = `var(--color-border)`
     - Each segment gets `title` attribute with step name for tooltip
   - Call `renderStatusBar()` inside the existing render cycle (wherever renderStepper is called)

2. **Standalone preparation checklist** (HUB-04, per 08-RESEARCH.md code example, locked decision):
   - Define `CHECKLIST_ITEMS` array with 6 items per CONTEXT.md decision:
     - { key: 'title', label: 'Titre defini', autoCheck: function(d) { return !!d.title; } }
     - { key: 'date', label: 'Date fixee', autoCheck: function(d) { return !!d.date; } }
     - { key: 'members', label: 'Membres ajoutes', autoCheck: function(d) { return d.memberCount > 0; } }
     - { key: 'resolutions', label: 'Resolutions creees', autoCheck: function(d) { return d.resolutionCount > 0; } }
     - { key: 'convocations', label: 'Convocations envoyees', autoCheck: function(d) { return d.convocationsSent; } }
     - { key: 'documents', label: 'Documents attaches', autoCheck: function(d) { return d.documentCount > 0; } }
   - Add `renderChecklist(sessionData)` function:
     - Gets `#hubChecklist` element
     - Iterates CHECKLIST_ITEMS, calls autoCheck(sessionData) for each
     - Renders progress header (done count / total), progress bar (percentage width), and item rows
     - Done items get `.done` class with check icon; pending items show "A faire" label
   - Call `renderChecklist()` after loadData completes with session data

3. **Dynamic main action card** (HUB-02, locked decision):
   - Verify existing `renderAction()` is already dynamic (CTA text, icon, color change by stage). If so, ensure it:
     - Uses CSS classes from hub.css (not inline styles) for the card container, icon, title, description
     - Keeps ONLY dynamic values (background color per stage) as inline styles from JS
   - If renderAction is missing or insufficient, implement: render a `.hub-action` card with icon, title, description, and a `.btn` CTA button. The CTA text, icon, and button color change based on `HUB_STEPS[currentStep]` data.

4. **KPI cards** (HUB-03):
