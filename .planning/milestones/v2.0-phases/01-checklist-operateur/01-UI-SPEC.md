| Token         | CSS Variable   | Value | Usage in this phase                        |

| sm            | `--space-2`    | 8px   | Row gap between checklist items            |

| lg            | `--space-6`    | 24px  | Panel internal padding (vertical top/bot)  |
| xl            | `--space-8`    | 32px  | Not used in checklist panel                |
| 2xl           | `--space-12`   | 48px  | Not used in checklist panel                |

Exceptions:
- Checklist row line-height is 44px minimum (touch target requirement — matches existing sidebar 44px standard from v1.9).
- Collapsed panel toggle button: 44px × 44px touch target.
- SSE disconnection banner: 8px top/bottom padding (`--space-2`), full-width, no side margin.

## Typography

All sizes map to existing `--text-*` scale from design-system.css.

Two font weights only: 500 (`--font-medium`) for labels, 600 (`--font-semibold`) for titles, alert text, and values. Values use `--font-mono` typeface which provides sufficient visual differentiation from labels without requiring a third weight.

| Role              | CSS Token      | Size | Weight                      | Line Height |
|-------------------|----------------|------|-----------------------------|-------------|
| Panel title       | `--text-xs`    | 12px | 600 (`--font-semibold`)     | 1.2         |
| Checklist label   | `--text-sm`    | 13px | 500 (`--font-medium`)       | 1.4         |
| Checklist value   | `--text-sm`    | 13px | 600 (`--font-semibold`) mono | 1.4        |
| Alert banner text | `--text-sm`    | 13px | 600 (`--font-semibold`)     | 1.4         |

Typography rationale (from CONTEXT.md decisions):
- Checklist rows are compact: icon + label + value on one line — 13px is the established operator chrome size.
- Values (e.g. `42/60`) use `--font-mono` + weight 600; the monospace typeface alone differentiates values from labels visually, consistent with `.op-kpi-value` pattern.
- Panel title follows `.op-sidebar-title` / `.hub-checklist-title` uppercase pattern at 12px 600.

## Component Inventory

New components introduced by this phase:

### 1. `.op-checklist-panel` — Right-side monitoring panel

- **Placement:** Inside `.op-exec-main`, as a second column, or appended after `.op-kpi-strip` as a collapsible side strip. Exact position: right of `.op-exec-main` flex container, width 240px fixed.
- **Layout:** Flex column, gap `--space-2` (8px), padding `--space-4` (16px) horizontal, `--space-6` (24px) vertical.
- **Border:** `border-left: 1px solid var(--color-border)` + `background: var(--color-surface)`.
- **Visible:** Only when `#viewExec` is active (mode execution). Hidden via `[hidden]` attribute in preparation mode.
- **Collapse state:** Panel collapses to a 32px-wide strip showing only the toggle icon. Collapse state stored in `sessionStorage` key `opChecklistCollapsed`.

### 2. `.op-checklist-title` — Panel header

- Text: "Contrôle séance" in uppercase, 12px, weight 600, `--color-text-muted`.
- Layout: flex row, space-between, with collapse toggle `<button>` on the right.

### 3. `.op-checklist-row` — Single indicator row

- Layout: `display: flex; align-items: center; gap: var(--space-2)` (8px).
- Min-height: 44px (touch target).
- Structure: `[status-icon 16px] [label text] [value text right-aligned]`
- Status icon: SVG from sprite, 16×16, colored via `color` property.
- Value: `--font-mono`, weight 600 (`--font-semibold`), 13px, right-aligned via `margin-left: auto`.

### 4. `.op-checklist-row--alert` — Row in alert state

- Applied when indicator is red (quorum non atteint, SSE offline).
- Background: `var(--color-danger-subtle)`.
- Border-left: `3px solid var(--color-danger)`.
- Icon animation: `animation: checklistPulse 1s ease-in-out 3` (3 cycles = 3 seconds then stops).
- `checklistPulse` keyframes: `0%,100% { opacity: 1 } 50% { opacity: 0.3 }` — icon only, not the row background.

### 5. `.op-checklist-sse-banner` — SSE disconnection persistent banner

- **Trigger:** SSE state transitions to `offline`.
- **Placement:** Top of `.op-checklist-panel`, above all rows, full panel width.
- **Height:** auto (wraps at 2 lines max).
- **Style:** `background: var(--color-danger-subtle); border-bottom: 1px solid var(--color-danger); padding: var(--space-2) var(--space-4); font-size: var(--text-sm); color: var(--color-danger-text); font-weight: 600`.
- **Icon:** `icon-wifi-off` from sprite, 14px, inline before text.
- **Text:** "Connexion perdue — les données peuvent être obsolètes"
- **Hidden by default:** `[hidden]` attribute. JavaScript removes `[hidden]` on SSE offline.
- **Clears:** When SSE reconnects, re-add `[hidden]`.

### 6. `.op-checklist-toggle` — Panel collapse/expand button

- Size: 32×32px, `border-radius: var(--radius-md)`.
- Icon: `icon-chevron-right` (collapses) / `icon-chevron-left` (expands).
- `aria-expanded="true|false"` attribute maintained by JS.
- On click: toggles `.op-checklist-panel--collapsed` class on panel + stores in `sessionStorage`.

## Interaction Contract

### SSE State → Checklist State Mapping

| SSE State      | SSE Row Icon Color        | Banner Visibility | Row Class            |
|----------------|---------------------------|-------------------|----------------------|
| `live`         | `--color-success` (green) | hidden            | default              |
| `reconnecting` | `--color-warning` (amber) | hidden            | default              |
| `offline`      | `--color-danger` (red)    | visible           | `--alert`            |

### Quorum State → Quorum Row

| Condition              | Icon           | Icon Color          | Row Class  |
|------------------------|----------------|---------------------|------------|
| presents >= requis     | `icon-check-circle` | `--color-success` | default    |
| presents < requis      | `icon-alert-circle` | `--color-danger`  | `--alert`  |

### Votes Recus Row

- Format: `12/45` (votes reçus / total éligibles). No percentage.
- Icon: `icon-ballot` or `icon-check-square`, `--color-text-muted` (neutral, never red/green).
- No alert state — informational only.

### Votants Connectes Row

- Format: `7 en ligne`. Icon: `icon-users`, `--color-text-muted`.
- No alert state — informational only.

### Alert flash behavior (CHECK-05)

- When any row transitions to `--alert` state: icon pulses 3 times over 3 seconds (`checklistPulse` animation, `iteration-count: 3`).
- No modal, no toast, no sound. Flash is inline, on the icon only.
- `prefers-reduced-motion`: disable `checklistPulse` animation; row still gets `--alert` class and background color.

### No active motion state (checklist before session is live)

- Before mode execution: panel has `[hidden]` attribute. No empty state needed — panel is simply not rendered.

## Checklist Row Order

Top to bottom within `.op-checklist-panel`:

1. SSE banner (visible only on offline)
2. Réseau / SSE — connection status
3. Quorum — presents/requis ratio
4. Votes reçus — fraction of eligible voters
5. Votants connectés — online count

Rationale (Claude's discretion — from CONTEXT.md): Most critical status first (network connection affects all others), then quorum (binary pass/fail), then vote progress, then presence count.

## Accessibility Contract

- Panel has `role="complementary"` + `aria-label="Contrôle séance"`.
- SSE banner has `role="alert"` + `aria-live="assertive"` — fires once on state change.
- Quorum row value has `aria-label="Quorum : 42 présents sur 60 requis"` (full text, not fraction only).
- Vote counter row has `aria-live="polite"` on the value `<span>` — updates silently.
- Votants connectés row has `aria-live="polite"` on the value `<span>`.
- Icon SVGs: `aria-hidden="true"` on all decorative icons (label text carries meaning).
- Collapse toggle: `aria-expanded="true|false"` + `aria-controls="opChecklistPanel"`.
- `prefers-reduced-motion`: `checklistPulse` animation wrapped in `@media (prefers-reduced-motion: no-preference)`.

## Layout Integration

The checklist panel integrates into the existing operator exec view without restructuring `.op-body` (280px sidebar + fluid main):

```
.op-body
  .op-sidebar          (280px, unchanged)
  .op-main             (fluid, unchanged)
    .view-exec
      .op-exec-header  (unchanged)
      #viewExecContent
        .op-exec-status-bar  (unchanged)
        .op-kpi-strip        (unchanged)
        .op-tags             (unchanged)
        .op-exec-body        [NEW wrapper: flex row]
          .op-exec-main      (existing, flex 1)
          .op-checklist-panel [NEW, 240px fixed, collapsible]
```

The `.op-exec-body` wrapper is a new `display: flex` element that wraps the existing `.op-exec-main` and the new `.op-checklist-panel`. This avoids touching the grid structure above.

At `max-width: 1024px` breakpoint: checklist panel collapses automatically (auto-collapsed, same `sessionStorage` mechanism, panel not shown in horizontal strip mode to preserve vertical space).

## Animation Contract

| Animation          | Duration  | Easing              | Trigger                        | Reduced-motion |
|--------------------|-----------|---------------------|--------------------------------|----------------|
| `checklistPulse`   | 1s × 3    | ease-in-out         | Row enters `--alert` state     | Disabled       |
| Panel collapse     | 200ms     | `--ease-standard`   | Toggle button click            | Disabled       |
| SSE banner show    | none (instant) | —              | SSE state → offline            | n/a            |
| Value update       | none (instant) | —              | SSE event pushes new value     | n/a (no anim)  |

No Anime.js usage in this phase — animations are pure CSS. Anime.js is reserved for the existing KPI strip count-up pattern (Phase 3 scope).
