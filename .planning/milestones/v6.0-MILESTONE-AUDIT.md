# v6.0 Production & Email — Milestone Audit

## Summary

All 8 requirements have code implementations but **5 have integration gaps**. The critical gap: `buildMailerConfig()` (Phase 62) is only called for the test_smtp button, not for actual email sending paths (Phase 63). This means DB-configured SMTP works for testing but not for real sends. 3 notification requirements are fully wired.

## Critical Gap: buildMailerConfig Bypass

`EmailController::schedule()`, `EmailController::sendBulk()`, and `MeetingWorkflowController::transition()` all instantiate `EmailQueueService($config)` or `MailerService($config)` with raw env config — bypassing the DB-over-env merge from Phase 62. Fix: call `MailerService::buildMailerConfig()` before instantiation in all 3 locations.

## Additional Gaps

1. `EmailTemplateService::preview()` sample data missing `{{results_url}}` and `{{hub_url}}` — previews show unreplaced variables
2. `createDefaultTemplates()` does not seed a `results` template type
3. `settings.js` btnTestEmail sends `body` instead of `body_html`
