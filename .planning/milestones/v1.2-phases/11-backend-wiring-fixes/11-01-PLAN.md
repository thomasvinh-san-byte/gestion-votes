<objective>
Wire the three DEAD vote settings (`settVoteMode`, `settQuorumThreshold`, `settMajority`) so they are actually read by the calculation engines. Currently `SettingsRepository::get()` is never called for these keys from VoteEngine/QuorumEngine — the audit found the UPSERT succeeds but the calculation ignores the value.

Purpose: Close the worst wiring gap in v1.2-PAGES-AUDIT.md. Without this, an administrator can change "Mode de vote" and the app behaves identically.

Output: Two PHPUnit tests that change a setting and assert the calculation result changes. Code path: `SettingsRepository::get($tid, 'settVoteMode')` appears in VoteEngine source.
</objective>

<setting_semantics>
Setting keys (stored as strings in tenant_settings):
- `settVoteMode`: values `'public'` or `'secret'`. When `secret`, motion decision must flag `secret=true`.
- `settQuorumThreshold`: integer percent 0-100 stored as string. Must be divided by 100 to produce a float 0..1.
- `settMajority`: values `'simple'` (0.5), `'absolute'` (0.5), `'two_thirds'` (0.6667), `'three_quarters'` (0.75). Read settings.htmx.html line 118-124 for exact select options.
</setting_semantics>
