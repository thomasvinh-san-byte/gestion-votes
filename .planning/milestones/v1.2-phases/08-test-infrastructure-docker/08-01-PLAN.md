    Add a new service `tests` at the end of the `services:` block in docker-compose.yml (before the top-level `networks:` key). Insert EXACTLY this service definition (indentation = 2 spaces for service name, 4 for keys):

    ```yaml
      # ---------------------------------------------------------------------------
      # Playwright E2E test runner (profile: test)
      # ---------------------------------------------------------------------------
      # Usage: docker compose --profile test run --rm tests npx playwright test
      # Or via wrapper: ./bin/test-e2e.sh
      # ---------------------------------------------------------------------------
      tests:
        image: mcr.microsoft.com/playwright:v1.59.1-jammy
        container_name: agvote-tests
        profiles:
          - test
        working_dir: /work/tests/e2e
        user: "${UID:-1000}:${GID:-1000}"
        environment:
          IN_DOCKER: "true"
          CI: "${CI:-}"
          BASE_URL: "http://app:8080"
        volumes:
          - ./:/work
          - tests-node-modules:/work/tests/e2e/node_modules
        depends_on:
          app:
            condition: service_healthy
        networks:
          - backend
        entrypoint: ["/bin/bash", "-lc"]
        command: ["npm install --no-audit --no-fund --silent && npx playwright test"]
    ```

    Then add `tests-node-modules:` to the top-level `volumes:` block so it reads:

    ```yaml
    volumes:
      pgdata:
        driver: local
      app-storage:
        driver: local
      uploads:
        driver: local
      tests-node-modules:
        driver: local
    ```

    Why this exact shape:
    - `profiles: [test]` — prevents service starting on plain `docker compose up` (locked decision)
    - `image: mcr.microsoft.com/playwright:v1.59.1-jammy` — matches @playwright/test 1.59.1 in package.json (locked decision)
    - `user: "${UID:-1000}:${GID:-1000}"` — non-root, avoids root-owned files on host (CONTEXT discretion)
    - `working_dir: /work/tests/e2e` — mount repo at /work, cwd to e2e folder
    - `IN_DOCKER: "true"` — consumed by playwright.config.js conditional
    - `BASE_URL: "http://app:8080"` — service name resolution within backend network
    - `depends_on: app (healthy)` — waits until app healthcheck passes
    - `tests-node-modules` named volume — prevents `npm install` on every run hitting the host bind mount
    - `entrypoint + command` — runs npm install (idempotent, cached in volume) then playwright

    Validate YAML after edit: `docker compose --profile test config > /dev/null`
  </action>
