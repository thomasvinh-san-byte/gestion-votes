# Phase 20.3 Plan 02: Settings Sidebar Nav + Wizard Step-Nav Summary

Settings page restructured to vertical sidebar nav (200px) per wireframe v3.19.2. Wizard step-nav pinned at card bottom via flex column layout with margin-top:auto.

## Tasks Completed

### Task 1: Restructure settings page to sidebar nav layout

The settings page structure (`.settings-layout`, `.settings-sidenav`, `.settings-sidenav-item`) was already implemented in HTML and JS from a prior execution. This task verified the implementation was correct and added the missing responsive breakpoint.

**What was done:**
- Verified `settings.htmx.html` already has `.settings-layout` (flex) + `<nav class="settings-sidenav">` with 4 `.settings-sidenav-item` buttons
- Verified `settings.js` already uses `.settings-sidenav-item` selector (not old `.settings-tab`)
- Added `@media (max-width: 768px)` rules to `settings.css` for sidebar layout to collapse to horizontal tab row on mobile
- Old `.settings-tab` CSS remains (inactive — elements no longer in HTML) for safety

**Commit:** `1e92b12`

### Task 2: Fix wizard step-nav to pin at card bottom via flex

Applied flex column layout to wizard step cards and wrapped step content in `.wiz-step-body` for all 4 steps.

**What was done:**
- `wizard.css`: Updated `.wiz-step` to `display:flex; flex-direction:column; min-height:0`
- `wizard.css`: Added `.wiz-step-body { flex:1; overflow-y:auto; min-height:0 }`
- `wizard.css`: Updated `.step-nav` to use `margin-top:auto; flex-shrink:0` instead of `margin-top:16px`
- `wizard.htmx.html`: Added `.wiz-step-body` wrapper to steps 2, 3, and 4 (step 1 already had it)

**Commit:** `6ef7703`

## Deviations from Plan

None — plan executed exactly as written. Settings HTML/JS was already correct from prior work; task scope reduced to CSS verification and responsive addition.

## Decisions Made

1. **Settings sidebar already implemented** — Both `settings.htmx.html` (sidebar nav HTML) and `settings.js` (`.settings-sidenav-item` selectors) were already updated before this plan ran. Task 1 became a verify-and-extend task, adding the responsive breakpoint that was missing.

2. **margin-top:auto over sticky/fixed** — Used `margin-top:auto` on `.step-nav` within a flex column context per research recommendation (avoids Pitfall 6: visual gap on short steps with sticky positioning).

3. **Kept old .settings-tab CSS** — Not removed since it's harmless (no elements use it) and removing it could cause issues if any code references it. Out of scope to clean up pre-existing CSS.
