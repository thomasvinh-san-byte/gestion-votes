## 5.5 Écrire un nouveau spec

### Squelette minimal

```javascript
// tests/e2e/specs/mon-feature.spec.js
// @ts-check
const { test, expect } = require('@playwright/test');
const { loginAsOperator } = require('../helpers');

test.describe('Mon feature — contrat fonctionnel', () => {
  test('le bouton X déclenche l\'action Y', async ({ page }) => {
    await loginAsOperator(page);
    await page.goto('/operator.htmx.html');
    await page.click('#btnX');
    await expect(page.locator('#resultY')).toBeVisible();
  });
});
```

### Conventions du projet

- **Nom de fichier** : `{feature-or-flow}.spec.js` — kebab-case, sous `tests/e2e/specs/`
- **Helpers** : login via `loginAs{Role}` depuis `helpers.js` (cookies cachés, évite le rate-limit auth)
- **Selectors** : préférer `id` (`#opBtnLaunchVote`) ou `data-testid`. Éviter les sélecteurs BEM substring (cf. [`.planning/codebase/EXPLORE-PATTERNS.md`](../../.planning/codebase/EXPLORE-PATTERNS.md))
- **Tags** :
  - `@a11y-vX.Y` pour les contrats accessibilité
  - `@integration` pour les tests qui dépendent d'une fixture seed (helper `seedMeeting`)
  - Combiner via `--grep` : `npx playwright test --grep "@integration"`
- **Fixtures backend** : utiliser `seedMeeting`/`seedRunningMeeting` depuis `helpers/seed-meeting.js` quand un meeting réel est nécessaire (endpoint `/api/v1/test/seed-meeting` dev-only)
- **Budget timeout** : 60s max par test (CLAUDE.md). Si plus, splitter en plusieurs tests.

### Lancement local
