**Iteration loop — repeat until all pass:**

1. Run `cd tests/e2e && BASE_URL=http://localhost:8080 npx playwright test --project=chromium --reporter=list 2>&1 | tail -100` to see failures.

2. For each failing spec:
   a. Read the error message (timeout? element not found? assertion failed?)
   b. If selector not found: load the actual HTML page via `curl http://localhost:8080/{page}` and search for the expected element. Find the correct selector.
   c. If timeout: increase timeout or add proper wait (waitForLoadState, waitForSelector).
   d. If assertion failed: check what the actual value is and update the assertion if the v4.4 behavior changed (e.g., different text, different structure).
   e. If auth failure: ensure the E2E seed data is loaded in the Docker DB and credentials match helpers.js.
   f. If API 404/500: check that the API endpoint path is correct by listing `public/api/v1/` directory.

3. Fix the spec file and re-run that specific spec to verify: `npx playwright test --project=chromium specs/{file}.spec.js`

4. After all individual fixes, run the full suite again to confirm no regressions.

**Specific areas likely to need runtime fixes:**

- **Auth flow**: The Docker stack may require specific seed data. If login fails, check if E2E seed SQL needs to be run. The helpers.js credentials (operator@ag-vote.local / Operator2026!) must match what's in the database.

- **API endpoint paths**: The specs reference paths like `/api/v1/meetings.php`, `/api/v1/ballots_cast.php`. Check if v4.4 changed any endpoint paths (e.g., removed `.php` extension, changed routing).

- **audit-regression.spec.js**: The P1/P2/P3 audit tests reference very specific DOM elements (#kpiTotal, #statLive, #pvEmptyState, .tour-launch, .tour-card). These may have changed names in v4.4 rebuilds. Check each page's actual DOM.

- **mobile-viewport.spec.js**: The hamburger/bottom-nav test may fail if v4.4 uses a different mobile navigation pattern. Check the actual mobile layout.

- **workflow-meeting.spec.js**: The API workflow tests hit real endpoints. If the E2E meeting doesn't exist in the DB, these will fail. Check seed data.

- **api-security.spec.js**: Has 40+ endpoint tests. Some endpoints may have been renamed. Cross-check against `ls public/api/v1/`.

**Do NOT skip or delete tests.** If a test verifies behavior that genuinely no longer exists (e.g., a removed feature), comment it out with a clear explanation and mark `.skip` with a TODO.

**Do NOT change test intent.** Only update selectors, timeouts, and specific assertions to match the current v4.4 state.
