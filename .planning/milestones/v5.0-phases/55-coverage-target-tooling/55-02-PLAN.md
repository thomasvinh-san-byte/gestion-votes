<task type="auto">
  <name>Task 2: Add coverage threshold enforcement to phpunit.xml</name>
  <files>phpunit.xml</files>
  <action>
1. Run coverage one more time to confirm we're above 90%:
   ```bash
   vendor/bin/phpunit --testsuite Unit --coverage-text 2>&1 | grep -E "Services|Controller"
   ```

2. Add coverage threshold enforcement to phpunit.xml. PHPUnit 10.5 supports this via the `--coverage-filter` and exit codes. The approach:

   Add a `<coverage>` block attribute or use the `failOnLowCoverage` approach. In PHPUnit 10.5, coverage thresholds are enforced via:
   ```xml
   <coverage>
     <report>
       <html outputDirectory="coverage-report"/>
       <text outputFile="php://stdout" showUncoveredFiles="true"/>
     </report>
   </coverage>
   ```

   AND adding to the root `<phpunit>` element:
   ```
   requireCoverageMetadata="false"
   ```

   Actually, PHPUnit 10.x does NOT have built-in per-directory threshold enforcement. The standard approach is:

   a) Create a small shell script `scripts/coverage-check.sh` that:
      - Runs phpunit with --coverage-clover coverage.xml
      - Parses the clover XML to extract line coverage for Services/ and Controller/
      - Exits non-zero if either is below 90%

   OR

   b) Add `--coverage-clover coverage.xml` to phpunit.xml report config, then use a PHP script to parse.

   Use approach (a) — create `scripts/coverage-check.sh`:
   ```bash
   #!/usr/bin/env bash
   set -euo pipefail
   THRESHOLD=90

   vendor/bin/phpunit --testsuite Unit --coverage-clover coverage.xml

   # Parse clover.xml for Services/ coverage
   SERVICES_COV=$(php -r '
     $xml = simplexml_load_file("coverage.xml");
     $stmts = 0; $covered = 0;
     foreach ($xml->project->package as $pkg) {
       foreach ($pkg->file as $file) {
         if (str_contains((string)$file["name"], "app/Services/")) {
           foreach ($file->line as $line) {
             if ((string)$line["type"] === "stmt") {
               $stmts++;
               if ((int)$line["count"] > 0) $covered++;
             }
           }
         }
       }
     }
     echo $stmts > 0 ? round($covered / $stmts * 100, 1) : 0;
   ')

   # Parse for Controller/ coverage
   CTRL_COV=$(php -r '
     $xml = simplexml_load_file("coverage.xml");
     $stmts = 0; $covered = 0;
     foreach ($xml->project->package as $pkg) {
       foreach ($pkg->file as $file) {
         if (str_contains((string)$file["name"], "app/Controller/")) {
           foreach ($file->line as $line) {
             if ((string)$line["type"] === "stmt") {
               $stmts++;
               if ((int)$line["count"] > 0) $covered++;
             }
           }
         }
       }
     }
     echo $stmts > 0 ? round($covered / $stmts * 100, 1) : 0;
   ')

   echo "Services coverage: ${SERVICES_COV}%"
   echo "Controller coverage: ${CTRL_COV}%"

   FAIL=0
   if (( $(echo "$SERVICES_COV < $THRESHOLD" | bc -l) )); then
     echo "FAIL: Services coverage ${SERVICES_COV}% < ${THRESHOLD}%"
     FAIL=1
   fi
   if (( $(echo "$CTRL_COV < $THRESHOLD" | bc -l) )); then
     echo "FAIL: Controller coverage ${CTRL_COV}% < ${THRESHOLD}%"
     FAIL=1
   fi
   exit $FAIL
   ```

3. Update phpunit.xml to also generate clover output:
   Add inside `<coverage><report>`:
   ```xml
   <clover outputFile="coverage.xml"/>
   ```

4. Make the script executable: chmod +x scripts/coverage-check.sh

5. Verify: running `bash scripts/coverage-check.sh` passes (exit 0) with both directories at 90%+.
