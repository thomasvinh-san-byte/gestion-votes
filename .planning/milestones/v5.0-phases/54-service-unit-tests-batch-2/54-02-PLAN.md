Create tests/Unit/MonitoringServiceTest.php with namespace Tests\Unit, extending TestCase.

The challenge: MonitoringService takes a RepositoryFactory (not individual repos), and calls `$this->repo->system()`, `$this->repo->meeting()`, `$this->repo->motion()`, `$this->repo->voteToken()`. Must mock RepositoryFactory and its sub-repo returns.

Also: `check()` calls `disk_free_space()`, `disk_total_space()`, `memory_get_usage()`, `phpversion()`, and `date()` -- these are real PHP functions that will work in test env. The env vars (MONITOR_*) can be set via `putenv()` in setUp.

Also: `sendAlertEmails()` reads `MONITOR_ALERT_EMAILS` env var and creates a `new MailerService($this->config)`. `sendWebhook()` reads `MONITOR_WEBHOOK_URL`. Set these env vars to empty/false in setUp to disable notification dispatch (so we only test metric collection and alert creation).

setUp pattern:
```php
private MonitoringService $service;
private $repoFactory;
private $sysRepo;

protected function setUp(): void {
    // Disable notification dispatch
    putenv('MONITOR_ALERT_EMAILS=');
    putenv('MONITOR_WEBHOOK_URL=');
    // Set known thresholds
    putenv('MONITOR_AUTH_FAILURES_THRESHOLD=5');
    putenv('MONITOR_DB_LATENCY_MS=2000');
    putenv('MONITOR_DISK_FREE_PCT=10');
    putenv('MONITOR_EMAIL_BACKLOG=100');

    $this->sysRepo = $this->createMock(\AgVote\Repository\SystemRepository::class);
    $meetingRepo = $this->createMock(\AgVote\Repository\MeetingRepository::class);
    $motionRepo = $this->createMock(\AgVote\Repository\MotionRepository::class);
    $voteTokenRepo = $this->createMock(\AgVote\Repository\VoteTokenRepository::class);

    $this->repoFactory = $this->createMock(\AgVote\Core\Providers\RepositoryFactory::class);
    $this->repoFactory->method('system')->willReturn($this->sysRepo);
    $this->repoFactory->method('meeting')->willReturn($meetingRepo);
    $this->repoFactory->method('motion')->willReturn($motionRepo);
    $this->repoFactory->method('voteToken')->willReturn($voteTokenRepo);

    // Default repo returns
    $meetingRepo->method('countForTenant')->willReturn(10);
    $motionRepo->method('countAll')->willReturn(25);
    $voteTokenRepo->method('countAll')->willReturn(50);
    $this->sysRepo->method('countAuditEvents')->willReturn(100);
    $this->sysRepo->method('countPendingEmails')->willReturn(0);

    $this->service = new MonitoringService(
        ['default_tenant_id' => 'tenant-001', 'app_url' => 'https://votes.test'],
        $this->repoFactory,
    );
}

protected function tearDown(): void {
    putenv('MONITOR_ALERT_EMAILS');
    putenv('MONITOR_WEBHOOK_URL');
    putenv('MONITOR_AUTH_FAILURES_THRESHOLD');
    putenv('MONITOR_DB_LATENCY_MS');
    putenv('MONITOR_DISK_FREE_PCT');
    putenv('MONITOR_EMAIL_BACKLOG');
}
```
