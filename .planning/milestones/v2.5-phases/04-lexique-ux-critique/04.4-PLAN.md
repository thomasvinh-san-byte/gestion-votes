<objective>
Enrichir les 50 messages d'erreur les plus émis dans `app/Services/ErrorDictionary.php` avec un "next-step" actionnable (virgule + verbe d'impératif/subjonctif). Exemple :
- Avant : `'already_voted' => 'Vous avez déjà voté sur cette résolution.'`
- Après : `'already_voted' => 'Vous avez déjà voté sur cette résolution, demandez à l'opérateur d'annuler le précédent pour modifier.'`

Purpose: ERR-01 — un message d'erreur sans next-step laisse l'utilisateur bloqué. Une virgule + un verbe d'action transforme "ce qui ne va pas" en "ce qu'il faut faire".
Output: `ErrorDictionary.php` modifié, ≥50 messages enrichis ; aucun code ajouté/supprimé.
</objective>

<task type="auto">
  <name>Task 2: Enrichir les 50 messages avec next-step actionnable</name>

    - `app/Services/ErrorDictionary.php` complet

  <action>
1. Pour chacun des 50 codes ciblés, réécrire le message :
   - Conserver le diagnostic (ce qui ne va pas).
   - Ajouter une virgule, puis un verbe à l'impératif (`demandez`, `contactez`, `vérifiez`, `essayez`, `recommencez`, `revenez`, `reconnectez-vous`, `attendez`, `réessayez après`, `consultez`, `accédez`, `sélectionnez`, `cliquez`, `corrigez`, `complétez`, `fermez`) ou au subjonctif (`que l'opérateur`, `qu'un administrateur`).
   - Le next-step doit être **concret et exécutable par l'utilisateur**. Pas de "réessayez plus tard" tout court (banni ERR-03).

2. **Phrases interdites (ERR-03) — NE PAS utiliser** :
   - `réessayer.` ou `réessayez.` en fin de phrase tout seul
   - `contactez l'admin` ou `contactez le admin`
   - `erreur survenue`
   - `une erreur est survenue`
   - `veuillez réessayer plus tard`

3. **Lexique (LEX-01/LEX-02)** : utiliser `votant` (éligible scrutin) vs `participant` (présent séance) vs `membre` (inscrit générique) selon contexte. Bannir `Approuver` au profit de `Confirmer`/`Valider`.

4. **Exemples de bonne forme** :
   - `'unauthorized' => 'Vous devez être connecté pour effectuer cette action, reconnectez-vous depuis la page de connexion.'`
   - `'forbidden' => 'Cette action requiert un rôle supérieur, demandez à un opérateur ou administrateur de l'effectuer.'`
   - `'meeting_not_found' => 'Cette séance est introuvable, vérifiez le lien d'accès ou retournez à la liste des séances.'`
   - `'invalid_meeting_id' => 'Identifiant de séance invalide, sélectionnez une séance dans la liste.'`
   - `'missing_meeting_id' => 'Identifiant de séance manquant, sélectionnez une séance dans la liste.'`
   - `'motion_not_found' => 'Cette résolution est introuvable, vérifiez l'ordre du jour de la séance.'`
   - `'already_voted' => 'Vous avez déjà voté sur cette résolution, demandez à l'opérateur d'annuler votre vote précédent pour le modifier.'`
   - `'quorum_not_met' => 'Le quorum n'est pas atteint, attendez davantage de votants ou demandez au président de reporter le scrutin.'`
   - `'rate_limited' => 'Trop de tentatives en peu de temps, attendez {N} secondes avant de réessayer.'`
   - `'invalid_csrf' => 'Jeton de sécurité expiré, rechargez la page et recommencez l'action.'`

5. **Préserver les codes "techniques" qui ne sont pas user-facing** : si un code est uniquement loggué côté serveur (pas affiché à l'utilisateur), pas besoin d'enrichir. Mais si un doute, enrichir par sécurité (pas de coût).

6. **Modification atomique** : modifier le fichier en place, en gardant la même structure (tableau associatif). Aucune ligne `=>` ajoutée ou supprimée.

7. **Vérifier syntaxe PHP** :
   ```bash
   php -l app/Services/ErrorDictionary.php
   ```

8. **Vérifier baseline inchangée** :
   ```bash
   grep -c '=>' app/Services/ErrorDictionary.php
   # doit retourner le même nombre qu'en Task 1
   ```

9. **Compter les messages enrichis** : un message enrichi contient au moins une virgule ET un verbe d'action. Heuristique :
   ```bash
   grep -E "=> *'[^']+, *(demandez|contactez|vérifiez|essayez|recommencez|revenez|reconnectez|attendez|consultez|accédez|sélectionnez|cliquez|corrigez|complétez|fermez|rechargez|que )" app/Services/ErrorDictionary.php | wc -l
   # doit retourner ≥ 50
   ```
   (Cette heuristique sert juste de garde-fou ; le vrai test arrive en plan 04.5 — `UxConventionsTest`.)
