## Cas spéciaux préservés (non remplacés)

- `0` (sans unité) — resets standards CSS
- `auto` — centrage horizontal (margin auto)
- Valeurs en `em` quand utilisées pour `font-size`, `line-height`, `letter-spacing` (non couvertes par la regex padding/margin)
- Valeurs `%` (non rencontrées sur padding/margin dans ces fichiers)
- `position`, `width`, `height`, `min-height`, `gap` (hors scope du plan : la regex cible uniquement `padding|margin(-[a-z]+)?`)

**À noter — hors scope :** `gap: 14px` dans `.dashboard-urgent` (ligne 941) est conservé tel quel : `gap` n'entre pas dans la portée de Plan 03.5 (PLAN ciblait padding/margin uniquement). Followup possible dans une dette future si normalisation `gap` souhaitée.

## Arrondis cosmétiques documentés

5 cas où la valeur source n'avait pas de token strictement équivalent :

1. **`14px` / `0.875rem` (14px) → `var(--space-3)` (12px)** : -2px sur 6 occurrences (`.dashboard-urgent` padding, `.section-header` margin/padding, `.tab-toolbar` margin, `.meeting-card-body` × 2, `.meeting-card-header`). Décision alignée sur Plan 02.5 (audit-chip trust.css). La densité de la wireframe v3.19.2 (commentaire ligne 1347 "wireframe density 14px") perd 2px → cohérence > densité pixel-perfect.
2. **`0.4rem` (≈6.4px) → `var(--space-1-5)` (6px)** : -0.4px sur `.file-item` padding-vertical. Imperceptible.
3. **`0.35rem` (≈5.6px) → `var(--space-1-5)` (6px)** : +0.4px sur `.type-chip-label` padding-vertical. Imperceptible.
4. **`0.15rem` (≈2.4px) → `var(--space-0-5)` (2px)** : -0.4px sur `.file-item-remove` padding. Imperceptible (bouton icône close).
5. **`22px → var(--space-5)` (20px)** et **`52px → var(--space-14)` (56px)** dans login.css (déjà documenté Task 1).
