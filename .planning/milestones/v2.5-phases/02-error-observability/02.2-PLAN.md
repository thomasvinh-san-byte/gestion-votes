# Plan 02.2 — Debounce SSE empty-state handlers

## Goal

Prévenir les double-render de `<ag-integrity-modal>` et de la dashboard hero card live lors d'une rafale d'événements SSE, en appliquant un debounce ≥ 250ms sur leurs handlers SSE. Test reproductible : 5 events injectés en 100ms → 1 seul render visible.

Source : ERR-V24-02 (REQUIREMENTS.md), CONTEXT.md décisions D-05..D-07.

## Acceptance Criteria

1. **`<ag-integrity-modal>` SSE handler** debounced à ≥250ms (pattern `setTimeout`/`clearTimeout` avec timer privé par instance).
2. **Dashboard hero card live** SSE handler (à identifier dans `dashboard.htmx.html` ou `pages/dashboard.js`) debounced à ≥250ms avec le même pattern.
3. **Configurabilité** : attribut `data-sse-debounce-ms` (défaut 250) sur le custom element / le container hero card. Testé pour 250 et 500.
4. **Test E2E** : `tests/e2e/specs/sse-burst-idempotency.spec.js` créé. Injection synthétique 5 events SSE en 100ms via `eventSource.dispatchEvent` ou `MockEventSource`. Assertion : `data-render-count` (ajouté par les composants) = 1 après 100ms + débounce window.
5. **`prefers-reduced-motion`** respecté : pas d'animation introduite, debounce silencieux.
6. **Régression nulle** sur tests existants : cockpit-health-bar, critical-path-operator restent verts.
7. **Pattern documenté** dans `01.2-SUMMARY.md` (ou plan SUMMARY) pour réutilisation future autres composants SSE.

### Task 2 — Implémenter debounce dans `<ag-integrity-modal>`

Modifier `public/assets/js/components/ag-integrity-modal.js` :

```js
// Add private debounce timer
class AgIntegrityModal extends HTMLElement {
  #sseDebounceTimer = null;

  get _debounceMs() {
    return parseInt(this.getAttribute('data-sse-debounce-ms') || '250', 10);
  }

  _onSseEvent(event) {
    if (this.#sseDebounceTimer) clearTimeout(this.#sseDebounceTimer);
    this.#sseDebounceTimer = setTimeout(() => {
      this._render(event); // existing render path
      this.#sseDebounceTimer = null;
    }, this._debounceMs);
  }

  // Add for testability:
  _trackRender() {
    const current = parseInt(this.getAttribute('data-render-count') || '0', 10);
    this.setAttribute('data-render-count', String(current + 1));
  }
}
```

Le compteur `data-render-count` permet l'assertion E2E sans introspecter les internals.

### Task 3 — Implémenter debounce hero card live

Pattern identique sur le handler hero card dashboard. Code partagé idéalement extrait dans `public/assets/js/utils/sse-debounce.js` :

```js
// utils/sse-debounce.js
export function createDebouncedSseHandler(target, renderFn, getMs = () => 250) {
  let timer = null;
  return (event) => {
    if (timer) clearTimeout(timer);
    timer = setTimeout(() => {
      renderFn(event);
      const current = parseInt(target.getAttribute('data-render-count') || '0', 10);
      target.setAttribute('data-render-count', String(current + 1));
      timer = null;
    }, getMs());
  };
}
```

Réutiliser sur les 2 composants pour DRY.

### Task 4 — Test E2E

Créer `tests/e2e/specs/sse-burst-idempotency.spec.js` :

```js
import { test, expect } from '@playwright/test';
import { loginAsOperator, gotoDashboard } from '../helpers.js';

test.describe('@regression SSE burst idempotency', () => {
  test('ag-integrity-modal: 5 events in 100ms → 1 render', async ({ page }) => {
    await loginAsOperator(page);
    // open modal
    await page.click('[data-action="open-integrity-modal"]');
    const modal = page.locator('ag-integrity-modal');
    await expect(modal).toBeVisible();
    
    // inject 5 events in 100ms
    await page.evaluate(() => {
      const modal = document.querySelector('ag-integrity-modal');
      for (let i = 0; i < 5; i++) {
        setTimeout(() => modal.dispatchEvent(new MessageEvent('message', { data: '{}' })), i * 20);
      }
    });
    
    // wait for debounce window
    await page.waitForTimeout(400);
    
    const renderCount = await modal.getAttribute('data-render-count');
    expect(parseInt(renderCount || '0', 10)).toBe(1);
  });

  test('hero card live: 5 events in 100ms → 1 render', async ({ page }) => {
    // similar pattern for hero card
  });

  test('debounce respects data-sse-debounce-ms attribute', async ({ page }) => {
    // override to 500ms, test that 400ms wait does NOT yet trigger render
  });
});
```

## Risks & Mitigations

| Risk | P | I | Mitigation |
|---|---|---|---|
| Debounce masque update légitime urgent | M | M | Default 250ms est court ; documenter dans pattern doc + permettre override `data-sse-debounce-ms` |
| Ne couvre pas tous les empty-states | M | L | Scope explicit : seulement modal intégrité + hero card live (les 2 cas du req). Autres composants → backlog v2.5+ |
| Test E2E flaky (timing-dependent) | M | M | Wait `400ms` window > 250ms debounce + 100ms burst = 350ms ; marge 50ms. Si flaky, augmenter à 500ms wait. |
| `data-render-count` interfère avec a11y/CSS | L | L | Attribut data-* est neutre, aucun impact |
