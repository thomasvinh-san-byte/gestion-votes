# Plan 03.1 — seedMeeting helper + dual-install fix

## Goal

Activer le test F-4 (`modal-focus-trap.spec.js @integration`) précédemment skippé en livrant un helper `seedMeeting()` côté Playwright qui appelle un endpoint API dev `/api/test/seed-meeting` (désactivé en production). Résoudre le dual-install Playwright en faisant de `tests/e2e/` la SOT, root `package.json` ne contient plus `@playwright/test`.

Source : TEST-V24-01 + TEST-V24-03 (REQUIREMENTS.md), CONTEXT.md décisions D-01..D-07.

## Acceptance Criteria

1. **Endpoint `/api/test/seed-meeting`** créé : POST, JSON body `{tenantId, status, motionsCount}`, retourne `{meeting_id: "..."}`. Désactivé hors `APP_ENV in [development, test, demo]` → retourne 404.
2. **Middleware d'env guard** : nouveau `EnvGuardMiddleware` (ou pattern existant) appliqué sur la route. Test PHPUnit couvre les 4 cas (`APP_ENV=development|test|demo|production`) — production seul retourne 404.
3. **Audit log à chaque appel** : `audit_log('test_seed_meeting', ['tenant_id' => ..., 'meeting_id' => ...])` pour traçabilité.
4. **Helper JS** `tests/e2e/helpers/seed-meeting.js` créé. Signature `seedMeeting({tenantId, status, motionsCount}) → Promise<meetingId>`. Réutilise `request.post()` Playwright fixture.
5. **F-4 test activé** : `tests/e2e/specs/modal-focus-trap.spec.js` test marqué `@integration` ne porte plus `test.skip` ou flag conditionnel — utilise `seedMeeting()` au setup. Run-deferred dev-machine, mais le code est en place + spec rend le test exécutable.
6. **Root package.json** : `@playwright/test` retiré des `devDependencies`. ESLint + chart.js + htmx.org conservés (frontend deps légitimes).
7. **Scripts `test:e2e*`** root : proxy vers `cd tests/e2e && npx playwright test ...`. Comportement utilisateur identique (`npm run test:e2e:chromium` from root continue à marcher).
8. **Test gardien** : `bin/check-deps.sh` créé, exit 1 si `@playwright/test` apparaît dans root `package.json` `devDependencies`.
9. **Rationale documenté** dans tests/e2e/README.md (voir 03.2 pour la rédaction complète, mais 03.1 ajoute un commentaire dans root package.json + 1 ligne dans le README).
10. **Régression nulle** sur tests existants (PHPUnit + Playwright specs intactes restent compatibles).

## Task Breakdown

### Task 1 — EnvGuardMiddleware

Créer `app/Core/Middleware/EnvGuardMiddleware.php` :

```php
<?php
declare(strict_types=1);
namespace AgVote\Core\Middleware;

final class EnvGuardMiddleware
{
    public function __construct(private array $allowedEnvs = ['development', 'test', 'demo']) {}

    public function handle(): void
    {
        $env = $_ENV['APP_ENV'] ?? getenv('APP_ENV') ?? 'production';
        if (!in_array($env, $this->allowedEnvs, true)) {
            http_response_code(404);
            header('Content-Type: application/json');
            echo json_encode(['ok' => false, 'error' => 'not_found']);
            exit;
        }
    }
}
```

Pattern compatible avec router middleware system existant.

### Task 2 — TestSeedController + endpoint

Créer `app/Controller/TestSeedController.php` :

```php
<?php
declare(strict_types=1);
namespace AgVote\Controller;

use AgVote\Repository\MeetingRepository;
use AgVote\Repository\MotionRepository;

final class TestSeedController
{
    public function __construct(
        private ?MeetingRepository $meetingRepo = null,
        private ?MotionRepository $motionRepo = null,
    ) {
        $this->meetingRepo ??= new MeetingRepository();
        $this->motionRepo ??= new MotionRepository();
    }

    public function seedMeeting(): void
    {
        $body = json_decode(file_get_contents('php://input'), true) ?? [];
        $tenantId = (string)($body['tenantId'] ?? '');
        $status = (string)($body['status'] ?? 'setup');
        $motionsCount = (int)($body['motionsCount'] ?? 0);

        if ($tenantId === '') {
            api_fail('invalid_request', 422);
            return;
        }

        $meetingId = $this->meetingRepo->createForTest($tenantId, $status);
        for ($i = 0; $i < $motionsCount; $i++) {
            $this->motionRepo->createForTest($meetingId, 'Motion test #' . ($i + 1));
        }

        audit_log('test_seed_meeting', ['tenant_id' => $tenantId, 'meeting_id' => $meetingId]);

        api_ok(['meeting_id' => $meetingId]);
    }
}
```

Si `MeetingRepository::createForTest()` n'existe pas, créer la méthode minimale (insertion directe dans transaction) ou réutiliser l'API existante.

### Task 4 — Helper Playwright JS

Créer `tests/e2e/helpers/seed-meeting.js` :

```js
export async function seedMeeting(request, { tenantId, status = 'setup', motionsCount = 0 }) {
  const response = await request.post('/api/test/seed-meeting', {
    data: { tenantId, status, motionsCount },
  });
  if (!response.ok()) {
    throw new Error(`seedMeeting failed: ${response.status()} ${await response.text()}`);
  }
  const json = await response.json();
  return json.meeting_id;
}
```

ESM export, compatible Playwright fixture.
