## Deviations from Plan

### None

The plan was executed exactly as written, with two scope clarifications documented in the audit (not deviations, but explicit interpretations) :

1. **`sidebar*.css` glob is empty.** The plan referenced `public/assets/css/sidebar*.css` but no such file exists in the codebase. Sidebar styles live in `app.css` (selectors: `.nav-item`, `.nav-group`, `.sidebar-*`). I cross-validated AC #3 by grepping the actual sidebar selectors in `app.css` (count: 0). The `CockpitPaletteTest::testSidebarHasZeroDangerColor` defends both the literal glob and the real selector locations.

2. **Baseline count is 21 line-matches, not 21 token occurrences.** Line 666 (`.op-tag-no-quorum`) holds 3 distinct `--color-danger*` references on one line; the audit table lists them as 3 logical refs (rows 6, 7, 8) but `grep -c` counts the line once. The plan's "21" matches `grep -c` semantics; we report both numbers in the audit for clarity.
