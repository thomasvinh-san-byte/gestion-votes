**Step 1: Find all hardcoded color values.**

Run: `grep -rn '#[0-9a-fA-F]\{3,8\}\|rgb(\|rgba(\|hsl(' public/assets/css/*.css --include='*.css' --exclude='design-system.css'`

For each match, categorize:
- **Inside @media print** — LEAVE AS-IS (print needs explicit colors)
- **Inside CSS comments** — LEAVE AS-IS
- **Inside var() fallback** like `var(--color-x, #hex)` — LEAVE AS-IS (fallback pattern is acceptable)
- **Standalone hardcoded value** — MUST REPLACE

**Step 2: Replace each hardcoded value with the correct token.**

Use this mapping for common values found in the audit:

| Hardcoded Value | Replace With |
|-----------------|-------------|
| `#fff`, `#ffffff`, `white` (as color value) | `var(--color-surface-raised)` or `var(--color-text-inverse)` depending on context |
| `#000`, `#000000`, `black` (as color value) | `var(--color-text-dark)` |
| `#333` | `var(--color-text-dark)` |
| `#666` | `var(--color-text-muted)` |
| `#888` | `var(--color-text-muted)` |
| `#ccc` | `var(--color-border)` (dark mode context) |
| `#e5e5e5` | `var(--color-border-subtle)` |
| `#1650E0` | `var(--color-primary)` |
| `#1140C0` | `var(--color-primary-hover)` |
| `#0B7A40`, `#0b7a40` | `var(--color-success)` |
| `#C42828`, `#c42828` | `var(--color-danger)` |
| `#B56700`, `#b56700` | `var(--color-warning)` |
| `#b8860b` | `var(--color-warning-text)` |

**Specific files with known issues (from audit):**

   - If inside `@media print` — leave as-is
   - If inside `[data-theme="dark"]` scoped to admin — REMOVE (design-system.css dark tokens handle this)
   - If inside a non-print context — replace hex values with references to other tokens or remove the override entirely

2. **hub.css** (lines ~42, 51): `color: #fff` and `stroke: #fff` — replace with `color: var(--color-text-inverse)` and `stroke: var(--color-text-inverse)`

   ```css
   background: linear-gradient(170deg, var(--color-bg), var(--color-bg-subtle));
   ```
   For the dark variant, use:
   ```css
   [data-theme="dark"] .public-page { background: linear-gradient(170deg, var(--color-bg), var(--color-bg-subtle)); }
   ```
   OR if these are for fixed always-light/always-dark contexts (public landing), keep them but move the values to design-system.css as named tokens.

