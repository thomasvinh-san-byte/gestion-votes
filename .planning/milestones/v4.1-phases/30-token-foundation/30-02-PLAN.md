**Step 1: Audit every occurrence of `var(--text-base)` and `font-size: var(--text-base)` across ALL CSS files.**

Run: `grep -rn 'text-base' public/assets/css/*.css`

For each match, categorize:
- **UI chrome** (form labels, table headers, nav items, sidebar items, button text, badge text, metadata, card chrome) — these SHOULD shrink to 14px. Leave as `var(--text-base)`.
- **Reading text** (paragraph body text, help descriptions, instructions, form field help text, modal body descriptions, long-form content) — these MUST stay 16px. Change to `var(--text-md)`.

Also find any explicit `font-size: 1rem` or `font-size: 16px` that is used for body/reading contexts and change to `var(--text-md)`.

The `body` element in design-system.css uses `font-size: var(--text-base)` — this is CORRECT. The body default is 14px for the app. Reading-text contexts that need 16px override with `var(--text-md)`.

**Step 2: Update reading-text references.**

For each reading-text context found in Step 1, change `var(--text-base)` to `var(--text-md)`. Examples of contexts that MUST use --text-md:
- `.help-content p`, `.help-answer`, any help/FAQ body text
- `.modal-body p`, modal description paragraphs
- `.empty-state-description` or similar long descriptions
- `.wizard-help-text`, `.form-help` descriptions
- Landing page body text
- Any `.prose` or `.content` class used for reading

**Step 3: Flip the token.**

In design-system.css `:root`, change:
```css
--text-base: 0.875rem;  /* 14px — primary UI chrome size */
```

Verify that `--text-md` is already defined as `1rem` (16px) from Plan 01. If not, ensure it is:
```css
--text-md: 1rem;  /* 16px — body reading text */
```

**Step 4: Update the body font-size line-height.**

In the `body` rule, update line-height to match the new 14px base:
```css
body {
  font-size: var(--text-base);     /* now 14px */
  line-height: var(--leading-base); /* 1.571 — 14px * 1.571 = 22px */
}
```

**CRITICAL RULES:**
- Do NOT change any font-size that already uses --text-sm (13px), --text-xs (12px), --text-lg (18px), etc. — only --text-base references need auditing.
- The voter view (vote.css) body text for ballot instructions MUST use --text-md (16px) — voters on mobile need larger text.
- Print styles (`@media print` blocks) should use explicit rem values, not variables.
