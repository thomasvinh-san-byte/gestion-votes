Create `tests/Unit/PageControllerTest.php`:

1. Use `PHPUnit\Framework\TestCase` as base (NOT ControllerTestCase -- PageController is static, not an AbstractController child).

2. In setUp(): set `$_SERVER['REQUEST_URI']` to a known value. Store original `$_SERVER` to restore in tearDown().

3. Test nonce injection:
   - Set `$_SERVER['REQUEST_URI'] = '/dashboard'`
   - Use output buffering: `ob_start()`, call `PageController::serveFromUri()`, `$output = ob_get_clean()`
   - Assert output does NOT contain `%%CSP_NONCE%%` (placeholder was replaced)
   - Assert output contains `nonce=` (nonce was injected) OR assert the nonce value from `SecurityProvider::nonce()` appears in output
   - Assert `http_response_code()` returns 200 (note: may need `@runInSeparateProcess` or check if PHPUnit allows this)

4. Test 404 for invalid page:
   - Set `$_SERVER['REQUEST_URI'] = '/totally-invalid-page'`
   - Use output buffering, call `serveFromUri()`
   - Assert output is empty
   - Verify 404 was set

5. Test serve() directly with a valid page name:
   - Call `PageController::serve('dashboard')` with output buffering
   - Assert `%%CSP_NONCE%%` is not in output
   - Assert SecurityProvider::nonce() value IS in output

6. Test serve() with nonexistent file:
   - Call `PageController::serve('zzz-does-not-exist')` -- a page NOT in PAGES constant is caught by serveFromUri, but serve() directly checks file_exists
   - Actually serve() receives any string and checks file_exists. Pass a page name whose .htmx.html does not exist.
   - Assert 404

IMPORTANT: PageController uses `http_response_code()` and `header()` which require either `@runInSeparateProcess` annotation or testing in a way that doesn't trigger "headers already sent" warnings. Use `@runInSeparateProcess` and `@preserveGlobalState disabled` annotations on test methods that call header().
