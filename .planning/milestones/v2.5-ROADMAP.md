**Key Decisions:**

- **`/admin/error-stats` v2.5 supersede v2.4** : la version v2.4 (P2.3) filtrait `audit_events` avec banner "limitation 6 actions audit-flavored". v2.5 va directement à `error_events` (vraie source) — meilleur signal, page plus utile, pas de banner d'apologie. Pas de fallback : la table v2.5 est la source de vérité.
- **`Logger::error()` au lieu de `Logger::errorContext()`** : le helper v2.4 `errorContext($message, $code, $context)` est redondant avec `error($message, $context)` (le code va naturellement dans le context array). Migration directe vers `Logger::error()` simplifie l'API.
- **Capture `api_fail()` try/catch isolé** : si `error_events` est down (DB hors-service au démarrage), la response utilisateur doit toujours être envoyée. Outer try/catch swallow + `Logger::warning` trace.
- **SEC-V2-01 extended au-delà du scope initial** : la requirement listait "8 méthodes MotionRepository" mais le pattern existait dans 11 repos (27 méthodes). Tout migré en un seul passage pour cohérence (commit séparé `bef552f` pour le scope initial + `ec1ee49` pour l'extension).
- **Stop-tests directive accepted** : tests Phase 5 (HEARTBEAT-V25-03/04) et Phase 7 entière différés explicite suite à directive utilisateur "concentre toi sur le code". Documenté comme tech debt v2.6+.
