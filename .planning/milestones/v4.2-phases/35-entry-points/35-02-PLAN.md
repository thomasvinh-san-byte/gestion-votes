**JS changes in login.js — add field-level error class toggling:**

Find the error handling code in the login form submission handler. Currently it shows errors via `#errorBox`. Add field-level error indication:

1. When server returns a validation error mentioning "email" (or email validation fails), add `field-error` class to the email field's parent `<div>`:
```javascript
// Find the email field's parent div
var emailField = document.getElementById('email');
var passwordField = document.getElementById('password');

// Helper: add field-error class to parent div
function setFieldError(field, hasError) {
  if (field && field.parentElement) {
    // For password, parent is .field-wrap, go one level up
    var wrapper = field.closest('.field-wrap') ? field.closest('.field-wrap').parentElement : field.parentElement;
    if (hasError) {
      wrapper.classList.add('field-error');
    } else {
      wrapper.classList.remove('field-error');
    }
  }
}
```

2. On form submit, clear previous field errors:
```javascript
setFieldError(emailField, false);
setFieldError(passwordField, false);
```

3. On error response, apply field-error based on error content:
```javascript
// If error mentions email/identifiant → highlight email field
if (errorMessage && (errorMessage.toLowerCase().includes('email') || errorMessage.toLowerCase().includes('identifiant'))) {
  setFieldError(emailField, true);
}
// If error mentions password/mot de passe → highlight password field
if (errorMessage && (errorMessage.toLowerCase().includes('password') || errorMessage.toLowerCase().includes('mot de passe'))) {
  setFieldError(passwordField, true);
}
// For generic auth errors (wrong credentials), highlight both
if (errorMessage && (errorMessage.toLowerCase().includes('incorrect') || errorMessage.toLowerCase().includes('invalide'))) {
  setFieldError(emailField, true);
  setFieldError(passwordField, true);
}
```

4. On input change, clear the field error for that field:
```javascript
emailField.addEventListener('input', function() { setFieldError(emailField, false); });
passwordField.addEventListener('input', function() { setFieldError(passwordField, false); });
```

Adapt all of the above to work within the existing code structure. Do NOT restructure the file — add the setFieldError helper and calls at the right spots within the existing flow. Keep `#errorBox` functioning as-is for the general error message.
