## Critical Fix Applied During Audit

**Nginx routing for /reset-password and /setup** — commit `c40e7cf5`

Both routes were missing from nginx.conf location blocks. Without these, requests to `/reset-password` and `/setup` fell through to `location /` which serves `index.html` (SPA shell) instead of `index.php` (PHP front controller). Fixed by adding exact-match location blocks in both `deploy/nginx.conf` and `deploy/nginx.conf.template`.

This affected: RESET-01, RESET-02, RESET-03, SETUP-01, SETUP-02, SETUP-03 (in production nginx only — Apache .htaccess handled it correctly).
