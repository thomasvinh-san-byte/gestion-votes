1. `emit(string $meetingId, string $severity, string $code, string $message, array $audience, array $data, string $tenantId): void`
   - Call `$this->meetingRepo->findByIdForTenant($meetingId, $tenantId)` — if null, return silently
   - Call `$this->notifRepo->countRecentDuplicates($meetingId, $code, $message, $tenantId)` — if > 0, return (dedup)
   - Normalize audience: `array_values(array_unique(array_filter($audience)))` — if empty, default to `['operator', 'trust']`
   - Build audience literal: `'{' . implode(',', array_map(fn($a) => '"' . $a . '"', $audience)) . '}'`
   - Call `$this->notifRepo->insert($tenantId, $meetingId, $severity, $code, $message, $audLiteral, json_encode($data) ?: '{}')`

2. `emitReadinessTransitions(string $meetingId, array $validation, string $tenantId): void`
   - Call `findByIdForTenant` — if null, return silently
   - Normalize `$validation['codes']`: if not array, set to `[]`
   - Get `$prevState = $this->notifRepo->findValidationState($meetingId, $tenantId)`
   - If `$prevState === null` (first pass): call `upsertValidationState` with current state and return (silent init, NO insert, NO countRecentDuplicates)
   - Extract `$prevReady = (bool)$prevState['ready']` and `$prevCodes = json_decode($prevState['codes'], true) ?: []`
   - Extract `$nowReady = (bool)$validation['can']` and `$nowCodes = $validation['codes']`
   - **Global transitions:** If `$prevReady !== $nowReady`:
     - If `$nowReady`: emit with code `readiness_ready`, severity `info`, message "La seance est prete pour validation"
     - If `!$nowReady`: emit with code `readiness_not_ready`, severity `warn`, message "La seance n'est plus prete pour validation"
   - **Code diff (only if no global transition):** If `$prevReady === $nowReady`:
     - `$added = array_diff($nowCodes, $prevCodes)` — for each, call `$tpl = $this->readinessTemplate($code, true)` then `$this->emit(...)` with `readiness_{$code}` code
     - `$removed = array_diff($prevCodes, $nowCodes)` — for each, call `$tpl = $this->readinessTemplate($code, false)` then `$this->emit(...)` with `readiness_{$code}_resolved` code
   - Always call `upsertValidationState($meetingId, $tenantId, $nowReady, json_encode($nowCodes))`

3. `private readinessTemplate(string $code, bool $added): array` — returns `['severity' => ..., 'message' => ...]`
   - Switch on `$code`:
     - `open_motions`: added → `['severity' => 'blocking', 'message' => 'Des votes sont encore ouverts']`; removed → `['severity' => 'info', 'message' => 'Tous les votes sont clos']`
     - `missing_president`: added → `['severity' => 'blocking', 'message' => 'Le president n\'est pas designe']`; removed → `['severity' => 'info', 'message' => 'Le president est designe']`
     - `bad_closed_results`: added → `['severity' => 'blocking', ...]`; removed → `['severity' => 'info', ...]`
     - `consolidation_missing`: added → `['severity' => 'blocking', ...]`; removed → `['severity' => 'info', ...]`
     - default: added → `['severity' => 'warn', 'message' => "Alerte : {$code}"]`; removed → `['severity' => 'info', 'message' => "Resolu : {$code}"]`

4. Pass-through methods (delegate directly to repository):
   - `list(string $meetingId, string $audience, int $sinceId, int $limit, string $tenantId): array` → `$this->notifRepo->listSinceId(...)`
   - `recent(string $meetingId, string $audience, int $limit, string $tenantId): array` → `$this->notifRepo->listRecent(...)`
