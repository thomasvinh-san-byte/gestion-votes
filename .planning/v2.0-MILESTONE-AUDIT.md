## Critical Integration Bugs (from Integration Checker)

### P0: Wizard Session Creation Completely Broken

**File:** `public/assets/js/pages/wizard.js` line 703
**Bug:** `api('POST', '/api/v1/meetings', payload)` — arguments inverted. The `api()` signature is `api(url, data, method)`, so `'POST'` is passed as the URL (→ 404), `/api/v1/meetings` as data body, and `payload` as method.
**Fix:** Change to `api('/api/v1/meetings', payload, 'POST')` or `api('/api/v1/meetings', payload)` (POST inferred from non-null data).

**File:** `public/assets/js/pages/wizard.js` line 712
**Bug:** `res.id || res._id` — but `api()` returns `{status, body}`, so correct extraction is `res.body?.id || res.body?._id`.

**Impact:** No session can be created through the wizard UI. Hub redirect always lands at `?id=undefined`. Affects WIZ-05, and indirectly HUB-01 to HUB-05 (hub only receives demo data).

### P1: Toast Integration Missing on Wizard and Hub

**Files:** `wizard.js` line 719-720, `hub.js` lines 433-434
**Bug:** Both call `Shared.showToast(...)` which does not exist on the `Shared` object. The correct API is `AgToast.show(type, message)` from `ag-toast.js`, but neither `wizard.htmx.html` nor `hub.htmx.html` load the component script.
**Fix:** Either add `<script src="/assets/js/components/ag-toast.js">` to both pages and use `AgToast.show()`, or add `showToast` method to `Shared` object in `shared.js` as a wrapper.

### P1: Operator Execution Header Timer Never Starts

**File:** `public/assets/js/pages/operator-tabs.js` line 2107
**Bug:** `setMode('exec')` calls `startSessionTimer()` (which updates `barClock` in the meeting bar) but never calls `O.fn.startExecTimer()` (which updates `opExecTimer` in the exec header).
**Fix:** Add `O.fn.startExecTimer()` call inside `setMode('exec')` and `O.fn.stopExecTimer()` when leaving exec mode.
