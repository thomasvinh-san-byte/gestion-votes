- **Niveau AAA non visé** (cf. §6 — déclaration de conformité)

### Pages auditées (22)

Les 21 pages applicatives HTMX + `login.html`. Liste exhaustive en §2.

### Runners dédiés

Trois specs Playwright couvrent les trois axes de l'audit :

1. **Runner structurel** — `tests/e2e/specs/accessibility.spec.js`
   - Matrice paramétrée par un tableau `PAGES = [{ path, loginFn, requiredLocator }]` (plan 16-01).
   - Règle `color-contrast` **désactivée** par `disableRules: ['color-contrast']` dans `tests/e2e/helpers/axeAudit.js`.
   - Raison : les tokens de couleur sont partagés sur l'ensemble de l'app et dépendent du thème (clair/sombre). Les auditer avec la matrice structurelle produirait des faux bloquants en CI — l'audit contraste est isolé dans un runner dédié (voir ci-dessous).
   - **Ceci n'est pas un waiver** mais une décision méthodologique (D-04, cf. `.planning/phases/16-accessibility-deep-audit/16-CONTEXT.md`).

2. **Runner contraste** — `tests/e2e/specs/contrast-audit.spec.js`
   - Gated par `process.env.CONTRAST_AUDIT` (default-skip). Exécuté manuellement, hors CI.
   - `AxeBuilder.withTags(['wcag2aa']).withRules(['color-contrast'])` — whitelist sur la seule règle contraste.
   - Produit `.planning/v1.3-CONTRAST-AUDIT.json` (22 pages, agrégation par page + détail par nœud).

3. **Runner keyboard/focus** — `tests/e2e/specs/keyboard-nav.spec.js`
   - 6 tests couvrant 4 shells (operator/login/voter/public projection) : skip-link, ordre Tab, focus-trap `ag-modal`.
   - Assertions Shadow DOM via `page.evaluate()` avec branche slotted-descendant + `host.shadowRoot.activeElement`.

### Environnement d'exécution

- Navigateur : chromium headless
- Orchestration : `bin/test-e2e.sh` → `docker compose run` sur image Playwright Jammy
- HTML servi par le conteneur `agvote-app` avec bind-mount `public/` (override dev via `docker-compose.override.yml`, plan 16-02)
- Contraintes environnement hôte : libs système manquantes (`libatk-1.0.so.0`, `libcups`, etc.) — obligent l'exécution en Docker, documenté dans 16-03 et 16-04.

## 4. Keyboard navigation & focus management

**Source:** `tests/e2e/specs/keyboard-nav.spec.js` (214 lignes, **6 tests**). Commit `f227f1d8`.

### Couverture

| Shell | Page | Login helper | Cible skip-link | Test |
| --- | --- | --- | --- | --- |
| app-shell + sidebar | `/operator.htmx.html` | `loginAsOperator` | `#main-content` | Tab → skip-link → Enter → focus sur main |
| Login 2-panneaux | `/login.html` | — | `#loginForm` (ou `#main-content`) | Tab → skip-link visible → activation |
| Voter tablet | `/vote.htmx.html` | `loginAsVoter` | `#vote-buttons` | Tab → skip-link sur page vote |
| Projection publique | `/public.htmx.html` | — | `#main-content` (structural) | Présence structurelle uniquement (waiver inline, cf. §5) |

### Tests additionnels

- **Ordre Tab (operator shell):** 6 Tab consécutifs restent dans `HEADER` / `NAV` / `MAIN` / `ASIDE` / `FOOTER` (assertion souple).
- **Focus-trap `ag-modal`:** injection déterministe d'un trigger + `ag-modal` via `page.evaluate()` avec 4 éléments focusables ; Tab × 5 + Shift-Tab × 3 → focus reste dans la modale ; Escape → `aria-hidden="true"` sous 2s ; focus restauré sur le trigger `#e2e-trap-trigger`.

### Pattern Shadow DOM

Playwright `page.locator(':focus')` ne traverse pas les shadow roots. Le test utilise :

```js
const trapped = await page.evaluate(() => {
  const active = document.activeElement;
  if (!active) return false;
  if (active.closest && active.closest('ag-modal#e2e-trap-modal')) return true; // slotted
  if (active.tagName === 'AG-MODAL' && active.shadowRoot?.activeElement) return true; // shadow
  return false;
});
```

Cette double branche couvre l'implémentation hybride d'`ag-modal` (contenu slotted en Light DOM + bouton `modal-close` en Shadow DOM).

### État de validation

- **Validation statique (`--list`):** 6 tests correctement enregistrés, imports résolus, describe/test blocks parsés. **PASS.**
- **Validation runtime (2026-04-09, chromium Docker):** `bin/test-e2e.sh specs/keyboard-nav.spec.js` — **6 / 6 tests green** après gap-closure. Quatre vraies régressions a11y découvertes au premier run et corrigées :
  1. `login.html` — `autofocus` sur `#email` capturait le focus, empêchant le premier Tab d'atteindre le skip-link → `autofocus` retiré (commit `d5557e12`).
  2. `vote.htmx.html` (shell voter) — `auth-ui.js` préfixait la `.auth-banner` en premier enfant de `<body>`, poussant le skip-link en deuxième position → insertion désormais *après* les `.skip-link` (commit `e5c4d1c5`).
  3. `operator.htmx.html` — deux overlays `position: fixed; display: flex` (`.op-quorum-overlay`, `.op-transition-card`) ne respectaient pas l'attribut `[hidden]` et interceptaient les pointer events → règle `[hidden]{display:none}` ajoutée (commits `def35a99`, `7e07ea57`).
  4. `ag-modal` composant — `_trapFocus` comparait `document.activeElement` (qui s'arrête au shadow boundary et retourne le host `ag-modal`) au lieu de l'élément focalisé réel à l'intérieur du shadow root → Shift+Tab depuis le premier focusable ne faisait jamais wrap vers le dernier. Correction : traverser la chaîne `shadowRoot.activeElement` pour obtenir l'élément actif réel (commit `ef0fc529`).
- **Résultat final :** 6/6 green, 16.9s d'exécution, 1 worker chromium Docker. Validation runtime complète, plus aucun test à stabiliser.

| Dimension | Statut | Justification |
| --- | --- | --- |

### Limitations connues

1. **Focus-trap `ag-modal` Shift+Tab** -- corrige en phase 16-03 (commit `ef0fc529`). Le wrap shift-tab traverse desormais correctement la chaine `shadowRoot.activeElement`. 6/6 tests green.
2. **Audit manuel exhaustif screen reader** -- defere, scope QA.

### Non-cibles explicites

- WCAG 2.1 niveau AAA (contraste 7:1, etc.)
- Audit automatique des emails HTML transactionnels
- Accessibilité des PDF générés par dompdf (procurations — spot-check manuel recommandé)
- i18n / traduction
- Refonte des raccourcis clavier (scope-limit utilisateur, cf. feedback)

### Méthodologie résumée

- **Automatique** : axe-core 4.10 via `@axe-core/playwright`, 22 pages, 4 specs (accessibility + contrast-audit + keyboard-nav + legacy)
- **Manuel** : revue des rules non auto-détectables (focus visible, semantic landmark presence) en spot-check
- **Contenant** : conteneur Docker Playwright pour éliminer les variations environnement hôte
- **Fréquence de revue** : prochaine itération prévue à **2026-10-09** (6 mois) ou lors du ship v1.4, selon ce qui arrive en premier
