## Fixes selected for this plan

1. **DashboardController.php:120** — Add `BallotRepository::countByMotionIds(array $motionIds, string $tenantId, string $meetingId): array<string,int>`. Refactor controller to call once before the foreach. Test: `BallotRepositoryBatchCountsTest`, `DashboardControllerN1Test`.
2. **EmailController.php:166** — Add `InvitationRepository::findStatusesByMeetingAndMembers(string $meetingId, array $memberIds, string $tenantId): array<string,string>`. Refactor `EmailController::sendBulkInvitations()` to pre-fetch statuses once (when `only_unsent=true`) and look up via array. Test: `InvitationRepositoryBatchStatusTest`.
3. **AttendancesController.php:123** — Add `AttendanceRepository::upsertModeBulk(string $meetingId, array $memberIds, string $mode, string $tenantId): array{created:int, updated:int}`. Single `INSERT … ON CONFLICT … RETURNING (xmax=0) AS inserted` with `VALUES (…), (…), …` build. Test: `AttendanceRepositoryBatchUpsertTest`.
4. **VoteTokenController.php:57** — Add `VoteTokenRepository::deleteUnusedByMotionAndMembers(string $meetingId, string $motionId, array $memberIds, string $tenantId): void` + `insertMany(array $rows): void`. Refactor controller to two batch calls. Test: `VoteTokenRepositoryBatchTest`.
5. **MemberGroupsController.php:239** — Add `MemberGroupRepository::findManyByIds(array $groupIds, string $tenantId): array<string,array>` (id => row). Refactor controller to one pre-fetch then loop validates from map. Test: `MemberGroupRepositoryBatchFindTest`.

## Audit conclusion

`app/Controller/` is largely eager-loaded already — only **5 real N+1 sites** found across 55 foreach occurrences. All five are fixed in this plan. The remaining foreach sites are pure in-memory operations (rendering, validation, accumulation) over already-batched query results. The dashboard hotspot (#1) is the highest-impact fix because `/api/v1/dashboard` is polled by every operator session.
