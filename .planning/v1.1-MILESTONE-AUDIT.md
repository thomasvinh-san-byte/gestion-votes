## Hotfixes Delivered (Bonus)

Three critical hotfixes were delivered alongside the planned work:

1. **`ddcf02f2`** — `RateLimiter::configure()` boot regression from v1.0 Phase 1. The cleanup left dead callsites in `SecurityProvider::init()` and `WorkflowValidationTest::setUp()` that caused every API request to return 500. App was completely unusable until fixed.

2. **`2f441469`** — Multi-week nginx routing regression. `/dashboard`, `/meetings`, `/hub` all served `index.html` instead of `dashboard.htmx.html` etc. Root cause: `rewrite ... last` re-entered location matching and fell through to `try_files /index.html` fallback. Fix: per-page exact-match `location =` blocks with `try_files /file.htmx.html =404` (no re-entry). Both `nginx.conf` and `nginx.conf.template` updated.

3. **`448f38a3`** — Login 2-panel polish stretch goal. After DESIGN-02 was technically complete, the user requested better proportions (40/60 instead of 50/50), filled brand panel content, and coherent project colors. Delivered: hero title in Fraunces, 3 feature highlights with icons, trust pills, gradient background with subtle grid texture and animated glow.
