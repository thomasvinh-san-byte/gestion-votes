# Phase 20.4 — Design System Enforcement

## Problem Statement

After phases 20.1–20.3, pages are visually inconsistent ("éclatées"). Global CSS overrides conflict across pages, there is no unified visual hierarchy, and some pages remain far from the wireframe. The accumulation of piecemeal CSS patches has created contradictory rules that need systematic cleanup.

## Goal

Every page matches the wireframe's design language: consistent spacing, color hierarchy, typography, and component usage. JS bugs fixed along the way.

## Critical Path (user-facing workflow order)

1. **dashboard.htmx.html** — First page after login, sets visual expectations
2. **meetings.htmx.html** — Session list, gateway to all session workflows
3. **wizard.htmx.html** — Session creation (+ fix import button bug)

## Approach

### Step 0: CSS Cleanup
- Remove contradictory overrides accumulated in phases 20.1-20.3
- Establish clear cascade: design-system.css → page-specific.css (no cross-page leaks)
- Audit which global rules help vs. harm

### Step 1-N: Per-page reconstruction (critical path first)
For each page:
1. Read wireframe equivalent section in ag_vote_wireframe.html
2. Read current HTML + CSS implementation
3. Identify gaps (spacing, colors, typography, layout, missing components)
4. Fix CSS and HTML to match wireframe
5. Fix any JS bugs found on that page
6. Verify at 1920x1080

## Known Bugs to Fix

- tour-overlay was missing hidden state (fixed in 20.3, verify no regression)
- launch-modal hidden override (fixed in 20.3, verify no regression)
