## Interaction Contracts

### Confirmation Pattern

**Chosen pattern: `AgConfirm.ask()` — applied universally** (Decision D-01)

Only truly destructive/irreversible actions show a confirmation dialog:
- Delete session, delete member, delete user, delete resolution, bulk delete, reset to draft
- Password reset trigger

All other actions (status toggles, minor edits, saves) execute directly and display a toast result.

`AgConfirm.ask()` call signature for destructive actions:
```js
const ok = await AgConfirm.ask({
  title: '[Nom de l\'action] ?',
  message: 'Cette action est irréversible.',
  confirmLabel: '[Verbe d\'action]',
  variant: 'danger'
});
```

Non-destructive confirmation (e.g. close session with pending votes) uses `variant: 'warning'`.

No `window.confirm()` or `window.alert()` calls are permitted anywhere.

### Modal Behavior (D-02)

All `ag-modal` instances must close on:
1. Backdrop click — already implemented at line 172 of `ag-modal.js`, audit to confirm `closable` attribute is not overridden to `false` unintentionally
2. Escape key — already implemented at line 32 of `ag-modal.js`
3. Explicit close button (X icon in header)

Focus trap is already implemented in `ag-modal.js` `_trapFocus()`. Verify it is active on all modal instances.

On modal open: first focusable element inside modal receives focus.
On modal close: focus returns to the element that triggered the open (`_previousFocus`).

### API Feedback Pattern (D-08, D-09)

Every fetch/API call follows one of two patterns:

**Pattern A — Create / Delete / Long operation:**
1. Disable the trigger button, add `ag-spinner` attribute or swap button text to "Chargement..."
2. On success: re-enable button, show `AgToast.show('success', '[Message de confirmation]')`
3. On error: re-enable button, show `AgToast.show('error', '[Description erreur] — [Action corrective]', 8000)`

**Pattern B — Toggle / Quick action (optimistic):**
1. Update UI state immediately (toggle checked, badge updated)
2. Fire fetch in background
3. On error only: revert UI state, show `AgToast.show('error', 'Modification non enregistrée', 8000)`

No fetch call may silently discard errors. Every `.catch()` or rejected promise must call `AgToast.show('error', ...)`.

Toast durations: success/info = 5000ms (auto-dismiss), error/warning = 8000ms (auto-dismiss, per existing defaults).

### Loading States

- Button-level: disabled state + spinner icon (16px, inline) replaces button label during request
- Page-level skeleton: not applicable to this phase (no new pages)
- `ag-spinner` component: use for full-panel loading states in operator console tabs and wizard transitions

### SSE Connection Status (D-11)

When `event-stream.js` detects a connection drop and activates fallback polling:
- A persistent warning banner appears at the top of the affected page (not a dismissable toast)
- Banner text: "Connexion temps reel interrompue — actualisation automatique toutes les 30 secondes"
- Banner color: `--color-warning-subtle` background, `--color-warning` text and border
- Banner disappears automatically when SSE reconnects

### Unsaved Changes Warning (D-12)

Forms with meaningful edits (wizard steps, settings page, member edit modal) must warn before navigation away:
- Trigger: `beforeunload` event or shell navigation intercept
- Warning approach: `AgConfirm.ask()` with `variant: 'warning'`, title "Quitter sans enregistrer ?", confirm label "Quitter"
- Only applies if the user has modified at least one field from its loaded state

## Layout Contracts

### General Rule (D-07)

All pages exploit horizontal space. No single-column stacking of form fields unless the viewport is below 768px. Minimum two-column CSS grid for any form with 4+ fields on desktop.

### Per-Page Width Strategy (D-06)

| Page | Width Strategy |
|------|---------------|
| Operator console | Full-width (`width: 100%`), no max-width constraint |
| Dashboard | Full-width |
| Analytics | Full-width |
| Meetings list | Full-width |
| Members list | Full-width |
| Audit log | Full-width |
| Settings | `max-width: var(--content-narrow)` (720px), centered |
| Admin | `max-width: var(--content-narrow)` (720px), centered |
| Mon Compte | `max-width: var(--content-narrow)` (720px), centered |
| Wizard | `max-width: 960px`, centered, multi-column fields inside |
| Hub | Full-width |
| Vote/ballot | `max-width: 720px`, centered |

### Wizard Layout (D-04, D-05)

- Multi-step remains (ag-stepper component)
- Step transitions: slide + fade, `--duration-deliberate` (300ms), `--ease-emphasized`
- Each step content uses 2-column CSS grid for fields: `grid-template-columns: 1fr 1fr` with `gap: var(--gap-md)`
- Fields that span full width (textarea, file upload, title inputs): `grid-column: 1 / -1`
- No per-step scrolling: each step's content must fit in viewport at 1024px without scroll
- Labels positioned above inputs (stacked), using 13px weight-500

### Form Field Grid Rule (D-05)

Forms with 4+ fields on pages wider than 768px must use:
```css
display: grid;
grid-template-columns: repeat(auto-fit, minmax(240px, 1fr));
gap: var(--gap-md);
```

Single-column exceptions: textarea fields, file upload fields, full-width search inputs.

## Animation Contracts (D-15)

All timing uses existing `--duration-*` and `--ease-*` tokens. No new values.

| Component | Enter | Exit | Easing |
|-----------|-------|------|--------|
| Modal (`ag-modal`) | 150ms scale(0.96→1) + fade | 200ms fade | `--ease-spring` enter, `--ease-standard` exit |
| Confirm dialog (`ag-confirm`) | 150ms scale(0.96→1) + fade | 150ms fade | `--ease-spring` |
| Toast (`ag-toast`) | 200ms slide-in from right | 300ms fade | `--ease-out` |
| Wizard step transition | 300ms slide + fade | 200ms fade | `--ease-emphasized` |
| SSE warning banner | 200ms slide-down | 200ms slide-up | `--ease-out` |
| Dropdown/popover | 150ms scale(0.97→1) + fade | 100ms fade | `--ease-standard` |
| Loading spinner | Continuous rotate 800ms | — | linear |

No animation may exceed 500ms. Respects `prefers-reduced-motion: reduce` by setting duration to 0ms.

## Component Variant Contracts (D-14)

Components must use exactly these variants — no ad-hoc inline style overrides.

### Buttons

| Variant | Class | Usage |
|---------|-------|-------|
| Primary | `.btn .btn-primary` | One per form/dialog, main CTA |
| Secondary | `.btn .btn-secondary` | Alternative actions, cancel |
| Danger | `.btn .btn-danger` | Destructive confirm button inside AgConfirm only |
| Ghost | `.btn .btn-ghost` | Icon-only toolbar actions, table row actions |
| Small | `.btn .btn-sm` | Table row actions, compact panels |

### Badges

| Variant | Semantic color token | Usage |
|---------|---------------------|-------|
| Success (vert) | `--color-success` | Seance ouverte, vote valide |
| Warning (ambre) | `--color-warning` | En attente, quorum insuffisant |
| Danger (rouge) | `--color-danger` | Erreur, seance suspendue |
| Info (bleu) | `--color-info` | Neutral status, non commence |
| Neutral (stone) | `--color-neutral` | Inactive, archive |

No badge uses the accent color (`--color-primary`) for its background.
