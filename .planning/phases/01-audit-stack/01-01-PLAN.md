# Plan 01-01 — Audit stack technique (Stage 2 post-pivot)

## Objective

Audit STATIQUE sandbox de chaque dépendance Composer + composant custom AgVote + infra runtime. Pour chaque ligne : verdict `keep` / `replace` / `remove` avec coût/bénéfice. Output unique : `.planning/STACK-AUDIT.md`. Aucun fichier de production touché.

## Context

- @.planning/REQUIREMENTS.md (14 reqs AUDIT-STACK-01..14)
- @.planning/PROJECT.md (Stage 2 du pivot 2026-05-05)
- @.planning/CRITICAL-PATH-AUDIT.md (Stage 1 — recoupement étape 02 / 10 / 12)
- @.planning/codebase/STACK.md (knowledge base tech stack)
- @composer.json (deps versions)
- @Dockerfile (runtime stack)

## Boundary

Fichiers production INTOUCHÉS. `git status` sur `app/ public/ database/ tests/ composer.json composer.lock Dockerfile` doit rester vide après l'audit. Seuls modifiés :
- `.planning/STACK-AUDIT.md` (créé incrémentalement)
- `.planning/phases/01-audit-stack/01-01-PLAN.md` (ce fichier)
- `.planning/phases/01-audit-stack/01-01-SUMMARY.md` (créé en fin de plan)

## Tasks

### Tasks 01-05 — Audit dépendances Composer

1. **AUDIT-STACK-01** : `dompdf/dompdf ^3.1` (génération PV PDF + procuration). Audit poids, alternatives (mPDF / wkhtmltopdf / Typst), risques runtime mémoire/CSS subset. Recoupement étape 10 audit chemin.
2. **AUDIT-STACK-02** : `phpoffice/phpspreadsheet ^1.29` + `openspout/openspout ^5.6` (export XLSX dual-track). Auditer si phpspreadsheet encore nécessaire vs OpenSpout-only.
3. **AUDIT-STACK-03** : `erusev/parsedown ^1.8` (Markdown templates email/doc). Usage réel + alternatives PHP 8.4-compat.
4. **AUDIT-STACK-04** : `symfony/mailer ^8.0` (SMTP MailerService). Audit version bleeding-edge ^8 vs alternatives.
5. **AUDIT-STACK-05** : Extensions PHP (`gd`, `intl`, `zip`, `pdo_pgsql`, `pgsql`, `mbstring`, `redis`, `Zend OPcache`). Auditer si toutes utilisées.

### Tasks 06-10 — Audit composants custom AgVote

6. **AUDIT-STACK-06** : `AgVote\Core\Router` (348 lignes, 162 routes). Comparer Slim 4 / Symfony Routing.
7. **AUDIT-STACK-07** : `AgVote\Core\Logger` (359 lignes, 29 sites usage). Comparer Monolog (PSR-3 standard).
8. **AUDIT-STACK-08** : `AgVote\Core\Security\IdempotencyGuard` (87 lignes). Comparer Symfony Lock + interceptor.
9. **AUDIT-STACK-09** : `AgVote\Core\Http\*` (Request, JsonResponse, ApiResponseException, HttpCache, ClientIp, UrlValidator — 6 fichiers, ~644 lignes). Évaluer remplacement par PSR-7 + PSR-15.
10. **AUDIT-STACK-10** : `AgVote\SSE\*` (SseAuthGate, EventBroadcaster, HeartbeatPayloadBuilder — 3 fichiers, ~477 lignes). Custom sur custom — évaluer Mercure/Centrifugo.

### Tasks 11-13 — Audit infra

11. **AUDIT-STACK-11** : Redis (cache + rate-limit + SSE queue + idempotency + presence). Évaluer fallbacks fichier, possibilité retirer pour single-tenant self-hosted.
12. **AUDIT-STACK-12** : PostgreSQL extensions + indexes audit. Vérifier covering indexes v2.7 + détecter manquants probables.
13. **AUDIT-STACK-13** : Docker multi-stage Alpine + nginx + supervisord. Évaluer FrankenPHP (1 binary).

### Task 14 — Synthèse

14. **AUDIT-STACK-14** : Synthèse `.planning/STACK-AUDIT.md` consolidant 13 audits + recommandations actionnables Stage 3 (M-DECISION). Décompte keep/replace/remove + estimation effort migration. Top 3 priorités. Verdict global Voie A.

