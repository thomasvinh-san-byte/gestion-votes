## State Inventory

This phase introduces or modifies 4 distinct UI states. Every state must render at the correct element.

### State 1: Empty List / Grid (FEED-01)

**Trigger:** API returns 0 items for a list or grid  
**Component:** `<ag-empty-state>` web component (already exists)  
**Applies to pages:** meetings.htmx.html, members.htmx.html, archives.htmx.html, users.htmx.html, email-templates.htmx.html, audit.htmx.html  
**Visual contract:**
- Centered column layout, 48px vertical padding
- Icon: 64x64px inline SVG in `--color-text-muted`, icon variant matches page context (see Icon Map below)
- Title: 18px semibold, `--color-text`
- Description: 13px regular, `--color-text-muted`, max-width 400px
- CTA button: `btn btn-primary btn-sm`, label is page-specific actionable verb (see Copywriting Contract)
- animate-fade-in on mount (class already defined in design-system.css)

**Icon Map:**
| Page | Icon value |
|------|-----------|
| meetings | `meetings` |
| members | `members` |
| archives | `archives` |
| users | `members` |
| email-templates | `generic` |
| audit | `generic` |

### State 2: Vote Confirmation Persistent (FEED-02)

**Trigger:** Voter casts a vote successfully (API response ok)  
**Element:** `div#voteConfirmedState.vote-confirmed-state` (already exists in vote.htmx.html)  
**Change from current:** Remove `setTimeout(function() { setVoteAppState('waiting'); }, 3000)` in `showConfirmationState()`. State stays `confirmed` until the next vote opens (existing `[data-vote-state]` CSS rules already hide the block in `waiting` and `voting` states).  
**Visual contract:**
- Block remains visible with `--color-success-subtle` background tint and `--color-success-border` border (existing `.vote-confirmed-state` styles)
- Icon: success checkmark, `--color-success`
- Line 1: voter's choice (`.vote-confirmed-choice`), 16px semibold
- Line 2: "Vote enregistré" (`.vote-confirmed-text`), 16px regular
- Line 3 (NEW): timestamp, format "Vote enregistré le JJ/MM/AAAA à HH:MM", 13px regular `--color-text-muted`
- Line 4: "Ce vote est définitif et irréversible." (`.vote-confirmed-irreversible`), 13px muted

**Timestamp injection:** JS writes to a new `<p class="vote-confirmed-timestamp" id="voteConfirmedTimestamp">` element inserted after `.vote-confirmed-text`, using `new Date().toLocaleDateString('fr-FR')` + `toLocaleTimeString('fr-FR', {hour:'2-digit', minute:'2-digit'})`.

### State 3: Filter / Search No-Results (FEED-03)

**Trigger:** Active filter or search term yields 0 matching items  
**Component:** `<ag-empty-state>` with search icon variant  
**Icon:** Use `generic` icon (search variant SVG will be added to `EMPTY_SVG` map as `'search'`)  
**Visual contract:**
- Same layout as State 1
- Title: "Aucun résultat" (13px description below, page-specific)
- Reset link: inline text link "Réinitialiser les filtres", 13px, `--color-primary`, placed as slotted action inside `<ag-empty-state>`
- Reset action: clears all filter state and triggers HTMX reload via `hx-get` without filter params

**Applies to pages with filters:**
- meetings.htmx.html (filter pills)
- members.htmx.html (filter chips)
- archives.htmx.html (search input)
- audit.htmx.html (date/type filters)

### State 4: Loading Indicator (FEED-04)

**Trigger:** HTMX request in flight (`.htmx-request` class present)  
**Pattern:** Add visible text label "Chargement..." adjacent to existing skeleton loaders, inside `.htmx-indicator` wrapper  
**Visual contract:**
- `.htmx-indicator` wrapper: `display: none` by default, `display: inline-flex` when `.htmx-request` is active (already handled by design-system.css)
- Content: `<span class="spinner spinner-sm"></span><span class="loading-label">Chargement...</span>`
- Spinner: existing `.spinner` `.spinner-sm` classes (24px)
- Label: 13px regular `--color-text-muted`, 8px gap from spinner
- Placement: above the skeleton loader region, centered horizontally

## Interaction Contracts

### Empty State CTA Click
- Pages with creation flow: CTA is `<a href="..." class="btn btn-primary btn-sm">` pointing to creation page/wizard
- Pages without creation (archives, audit): no CTA rendered — omit `action-label` attribute from `ag-empty-state`

### Filter Reset Click
- Reset link is `<a>` with `hx-get` attribute pointing to the list endpoint with no filter params
- `hx-target` points to the list container element
- `hx-swap="innerHTML"`
- No confirm dialog required

### Vote Confirmation Persistence
- `confirmed` state remains until `setVoteAppState('waiting')` is called by the next vote open event (SSE or polling)
- Timestamp is written once at confirmation time; it does not update
- The block is hidden by `[data-vote-state="waiting"] .vote-confirmed-state` and `[data-vote-state="voting"] .vote-confirmed-state` rules (already in vote.css) — this behavior is preserved

### Loading Indicator Visibility
- `.htmx-indicator` is shown automatically by HTMX when a request is in flight (`.htmx-request` class on ancestor)
- No JS changes needed — only HTML structure addition
- Skeleton loaders remain visible alongside the text label during loading

## Accessibility Notes

- Empty state CTA: focusable `<a>` with descriptive label matching the page context
- Filter reset link: `aria-label="Réinitialiser les filtres de cette liste"` for screen readers
- Loading indicator: `aria-live="polite"` on `.htmx-indicator` wrapper so screen readers announce "Chargement..." on request start
- Vote confirmation: `role="status"` on `#voteConfirmedState` so screen readers announce confirmation
- Timestamp in vote confirmation: plain text content, no ARIA needed
