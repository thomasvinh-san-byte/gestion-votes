<objective>
Fix three frontend robustness holes in the SSE/async layer:

1. (FE-01) public.js opens an EventSource connection but never stores the handle, so it cannot call close() on navigation. Add `pagehide` cleanup.
2. (FE-03) handleSSEEvent in operator-realtime.js chains .then() calls on async functions (loadBallots, loadResolutions) with no .catch(). Uncaught rejections are silently swallowed. Add .catch() that calls setNotif('error', ...).
3. (FE-04) When EventStream reaches MAX_RECONNECT_ATTEMPTS and calls close(), the operator gets no UI feedback that real-time updates have stopped. Add an onFallback callback to event-stream.js and use it in operator-realtime.js to show a persistent toast.

Purpose: Prevent connection leaks and make async failures visible to operators.
Output: Three small JS edits, all changes verifiable by grep.
</objective>

**event-stream.js — add onFallback callback:**

The `source.onerror` handler (lines ~116–125) currently logs a warning and calls `close()` when `reconnectAttempts >= MAX_RECONNECT_ATTEMPTS`. After calling `close()`, invoke `handlers.onFallback()` if defined:

```javascript
    source.onerror = function () {
      reconnectAttempts++;
      if (handlers.onDisconnect) handlers.onDisconnect();

      if (reconnectAttempts >= MAX_RECONNECT_ATTEMPTS) {
        console.warn('[EventStream] Max reconnect attempts reached, closing.');
        close();
        if (handlers.onFallback) handlers.onFallback();
      }
      // Otherwise EventSource auto-reconnects
    };
```

Also update the JSDoc on `connect()` to document the new option:
```
 * @param {Function} [opts.onFallback] - Called when max reconnects exceeded (polling-only mode)
```

**operator-realtime.js — show persistent toast on fallback:**

In the `connectSSE()` function, add an `onFallback` handler to the `EventStream.connect()` call. When SSE falls back to polling, show a persistent warning toast (duration=0) and set the indicator to 'offline'.

When SSE reconnects (`onConnect`), dismiss the fallback toast if present.

Declare a module-level variable `var _sseFallbackToastId = null;` near the other module vars (top of the IIFE, near `var sseConnected = false`).

Modify `connectSSE()`:

```javascript
  function connectSSE() {
    if (!window.EventStream || !O.currentMeetingId) return;

    // Close previous connection
    if (sseStream) sseStream.close();

    sseStream = EventStream.connect(O.currentMeetingId, {
      onConnect: function() {
        sseConnected = true;
        setSseIndicator('live');
        // Dismiss fallback toast if it exists
        if (_sseFallbackToastEl) {
          _sseFallbackToastEl.dismiss();
          _sseFallbackToastEl = null;
        }
        console.info('[operator] SSE connected for meeting', O.currentMeetingId);
      },
      onDisconnect: function() {
        sseConnected = false;
        setSseIndicator('reconnecting');
        // If still disconnected after 5s, show offline state
        setTimeout(function() {
          if (!sseConnected) setSseIndicator('offline');
        }, 5000);
      },
      onFallback: function() {
        sseConnected = false;
        setSseIndicator('offline');
        // Show persistent fallback notification (duration=0 = no auto-dismiss)
        if (typeof AgToast !== 'undefined' && AgToast.show) {
          _sseFallbackToastEl = AgToast.show('warning', 'Connexion temps reel interrompue — passage en mode poll', 0);
        } else {
          setNotif('warning', 'Connexion temps reel interrompue — passage en mode poll', 0);
        }
      },
      onEvent: function(type, data) {
        handleSSEEvent(type, data);
      },
    });
  }
```

Note: `AgToast.show()` returns the `ag-toast` DOM element. Store it in `_sseFallbackToastEl` (module-level var, declared near `var sseConnected`). The element has a `.dismiss()` method that fades it out.

Add `var _sseFallbackToastEl = null;` near other module-level vars.
