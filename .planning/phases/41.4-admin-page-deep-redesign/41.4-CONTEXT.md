# Phase 41.4: Admin Page Deep Redesign - Context

**Gathered:** 2026-03-20
**Status:** Ready for planning

<domain>
## Phase Boundary

Deep visual redesign of admin.htmx.html — the most complex single page in the app. Every section (KPIs, tabs, user management, policies, state diagram, settings, system health) must be redesigned to modern standards. Horizontal layout, proper visual hierarchy, consistent badge sizes, modern interactions.

</domain>

<decisions>
## Implementation Decisions

### KPI Cards
- **Visual hierarchy:** Primary KPI (sessions en cours) gets larger/accent treatment. Others smaller supporting role
- **Context:** Each KPI shows a supporting detail below the number ("+2 cette semaine", "3 en attente", etc.)
- **Trend indicator:** Colored up/down arrow if trend data available
- **Full width horizontal row** — KPIs use available width, not narrow centered grid

### Tab Navigation
- **Modern style:** Underline accent animation on active tab (2px primary bottom border, slides to active tab)
- **Icon + text:** Icons larger (20px), with active state coloring (primary when active, muted when inactive)
- **Panel transitions:** Subtle fade animation between tab panels (already partially done)
- **Badge counts:** Where applicable (Users tab shows user count badge)

### User Management
- **Single list pattern:** Remove dual table/row inconsistency — one clean pattern
- **Larger badges:** Role and status badges at normal size (not 0.65rem cramped badges)
- **Inline actions:** Hover-reveal edit/role-change/deactivate buttons per user row
- **User create form:** Horizontal — fields in .form-grid-2 or .form-grid-3, not stacked
- **Pagination:** Modern with prev/next chevrons, current page highlight, total count

### Policy Cards
- **Icon per policy type:** Visual indicator for quorum policy vs majority policy
- **Status badge:** Active/inactive indicator on each policy card
- **Action buttons:** Clean icon buttons with tooltips, not cramped text links
- **Horizontal layout:** Policy details and actions side by side, not stacked

### State Diagram
- **Proper visual flow:** CSS flexbox/grid diagram with styled connector arrows (not text →)
- **Color-coded states:** Each state type has its own color (draft=neutral, live=danger, archived=success)
- **Interactive tooltips:** Each state node has ag-tooltip showing available transitions
- **Larger nodes:** Currently 0.75rem padding — increase to proper badge-sized nodes with readable labels

### System Health
- **Dashboard-style strip:** Larger values with colored status indicators
- **Threshold visualization:** Progress bars or gauges for disk/memory usage if applicable
- **Clean separation:** Each health metric in its own mini-card, not cramped flex row

### Settings Integration
- **Already redesigned in Phase 40** (toggle switches, tooltips, CNIL cards) — verify consistency
- **Horizontal form grids** where applicable (SMTP settings, etc.)

### Forms Across Admin
- **All forms:** .form-grid-2 or .form-grid-3, no single-column stacking
- **Inline validation:** Red/green border + helper text on input events
- **Password strength:** Larger bar (8px height), animated fill

### Claude's Discretion
- Exact state diagram connector arrow CSS implementation
- Whether to add sparklines to system health metrics
- User list row exact layout proportions
- Tab animation easing and duration

</decisions>
