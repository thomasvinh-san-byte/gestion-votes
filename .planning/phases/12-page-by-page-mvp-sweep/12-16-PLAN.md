    Meeting context setup: same defensive pattern as previous plans — fetch first meeting via API, set in localStorage, skip if unavailable.

    ⚠️ CRITICAL SAFETY: this spec MUST NOT click `#btnModalConfirm`. The test asserts modal wiring only, then cancels.

    For each PRIMARY interaction:

    1. **Page mount + summary grid visible**
       - Navigate to `/validate.htmx.html`
       - Assert `#summaryGrid` is visible.
       - Assert all 8 summary items exist: `#sumMembers`, `#sumPresent`, `#sumMotions`, `#sumAdopted`, `#sumRejected`, `#sumBallots`, `#sumQuorum`, `#sumDuration`.

    2. **Checklist panel loads**
       - Assert `#checksList` is visible.
       - Wait up to 15s for the loading spinner to be replaced:
         ```js
         await page.waitForFunction(() => {
           const el = document.getElementById('checksList');
           return el && !el.querySelector('.spinner');
         }, { timeout: 15000 });
         ```
       - Assert `#readyBadge` is visible (has text content, not empty placeholder `—`).

    3. **Recheck button — reloads checks**
       - Click `#btnRecheck`.
       - Wait 500ms.
       - Assert `#checksList` is still visible after reload.

    4. **President name input — is interactive and has pattern**
       - Fill `#presidentName` with "Jean Test".
       - Assert `#presidentName` value equals "Jean Test".
       - Assert the input has a `pattern` attribute (per HTML: `pattern="[A-Za-zÀ-ÿ0-9\s\-'.,()]{2,100}"`).

    5. **Validation zone + main validate button present**
       - Assert `#validationZone` is visible with class `validation-zone-danger`.
       - Assert `#btnValidate` is visible.
       - The button MAY be disabled (depends on checklist state) — assert its existence but do not require enabled state.

    6. **Modal open (conditional on validate button enabled)**
       - Check if `#btnValidate` is enabled: `const validateEnabled = await page.locator('#btnValidate').isEnabled();`
       - If enabled:
         - Click `#btnValidate`.
         - Assert `#validateModal` becomes visible (no `hidden` attribute): `await expect(page.locator('#validateModal')).not.toHaveAttribute('hidden', /.*/, { timeout: 5000 });`
         - Assert `#validateModalTitle` is visible.
         - Assert `#confirmIrreversible` checkbox is unchecked.
         - Assert `#btnModalConfirm` is DISABLED initially (because neither checkbox nor text entered).
       - If disabled:
         - Skip the modal interaction steps. Log: "btnValidate disabled — checklist not ready, skipping modal open".

    7. **Modal checkbox + text validation (only if modal opened in step 6)**
       - Check `#confirmIrreversible`.
       - Assert `#btnModalConfirm` is still disabled (needs both checkbox AND "VALIDER" text).
       - Fill `#confirmText` with "VALIDER".
       - Wait 300ms.
       - Assert `#btnModalConfirm` becomes ENABLED (proves the dual-guard wiring works).
       - 🚨 DO NOT CLICK `#btnModalConfirm` — would archive the meeting irreversibly.
       - Click `#btnModalCancel`.
       - Assert `#validateModal` gets the `hidden` attribute back (or becomes not visible) within 3s.

    8. **Width verification**
       - Assert `document.documentElement.scrollWidth <= document.documentElement.clientWidth + 1`.

    Read-only, non-mutative, re-runnable. Cancel always taken at the end to ensure modal is closed.
