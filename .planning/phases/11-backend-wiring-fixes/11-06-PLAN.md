<objective>
Close DEBT-02. `MeetingReportsController.php` is 727 lines — the largest controller after v1.0 refactoring. It has 5 public methods (`report`, `generatePdf`, `generateReport`, `sendReport`, `exportPvHtml`) each doing significant in-method business logic: HTML generation via DOMPDF, snapshot caching, merging motions + attendance + proxies, PDF bytes encoding.

Extract that logic into a new `MeetingReportsService` (new file under `app/Services/`). Leave in the controller ONLY: request validation, tenant/meeting resolution, authorization calls, response formatting (`api_ok` / `api_fail` / `header()` + `echo`). Target: controller under 300 lines.

Purpose: Maintainability + testability. A 727-line controller cannot be reasoned about; extracting the service gives clean unit-testable functions.

Output: New service file, slimmed controller, service unit tests, zero regression on existing controller tests.
</objective>
