  <files>public/assets/js/pages/audit.js</files>
  <action>
Modify `public/assets/js/pages/audit.js` to remove all demo data and wire real API:

1. **Update header comment** (line 1): Change "Demo data, table/timeline views" to "Real API data, table/timeline views". Update JSDoc (lines 2-6): remove "demo fallback" wording, replace with "Loads audit events from API with error handling".

2. **Delete `DEMO_EVENTS` constant** (lines 17-268 approximately — the entire `var DEMO_EVENTS = [...]` block containing 25 demo event objects). This is ~250 lines of hardcoded data to remove.

3. **Add `showAuditError()` function** (place it near the loadData function). Follow hub.js pattern:
   - Call `Shared.showToast("Impossible de charger les événements d'audit.", 'error')`
   - Render error state in `_tableBody` (table view): a `<tr><td colspan="6">` containing `Shared.emptyState({icon: 'generic', title: 'Erreur de chargement', description: 'Impossible de contacter le serveur.'})` plus a centered "Réessayer" button with id `auditRetryBtn`
   - Render error state in `_timeline` (timeline view): same `Shared.emptyState()` content
   - Retry button click calls `loadData()`
   - IMPORTANT: Also hide the KPI section or show dash values to avoid showing stale KPIs from a previous load

4. **Rewrite `loadData()` function** (lines 651-666):
   a. **Check for meeting_id first**: Extract from URL query string: `var params = new URLSearchParams(window.location.search); var meetingId = params.get('meeting_id');`
   b. **If no meeting_id**: Don't call the API. Set `_allEvents = []`. Render a guidance empty state in `_tableBody`:
      ```javascript
      _tableBody.innerHTML = '<tr><td colspan="6">' + Shared.emptyState({
        icon: 'generic',
        title: 'Sélectionnez une séance',
        description: 'Ouvrez une séance depuis le tableau de bord pour afficher son journal d\'audit.'
      }) + '</td></tr>';
      ```
      Also set `_timeline.innerHTML` to the same emptyState. Then return early (don't call populateKPIs/applyFilters).
   c. **Fix API URL**: Change `/api/v1/audit.php` to `/api/v1/audit_log.php?meeting_id=` + encodeURIComponent(meetingId)
   d. **Map response fields**: After getting `res.body.data.items`, map each item to match the field names used throughout the rendering code:
      ```javascript
      _allEvents = items.map(function(item) {
        return {
          id: item.id,
          timestamp: item.created_at || item.timestamp,
          category: item.category || 'system',
          severity: item.severity || 'info',
          event: item.action_label || item.event || '',
          user: item.actor || item.user || '',
          hash: item.hash || '',
          description: item.description || item.action_label || ''
        };
      });
      ```
      This mapping is defensive — it accepts both the real API shape (`action_label`, `actor`, `created_at`) and falls back to the old field names in case any other endpoint uses them.
   e. **Error handling**: In the catch block, replace `_allEvents = DEMO_EVENTS` with `showAuditError(); return;` — do NOT call populateKPIs/applyFilters after an error.
   f. **Add retry-once pattern**: Wrap in `tryLoad(attempt)` like hub.js. First failure → wait 2s → retry. Second failure → `showAuditError()`.

5. **Update console.warn message** (line 660): Change "API unavailable, using demo data" to "API unavailable" (no mention of demo data).

6. **Verify empty-state handling for zero events**: The existing `renderTable()` (lines 350-356) and `renderTimeline()` (lines 414-420) already handle empty arrays via `Shared.emptyState()`. No changes needed there — when API returns an empty items array, the existing code handles it correctly.

IMPORTANT: The field mapping in step 4d is critical. `renderTable()` uses `evt.event`, `evt.user`, `evt.hash`, `evt.severity`. `renderTimeline()` uses the same fields. `exportCSV()` uses `evt.event`, `evt.user`, `evt.severity`, `evt.hash`. `showEventDetail()` uses `evt.user`, `evt.severity`, `evt.hash`. The `applySortToFiltered()` function sorts by `evt.severity`. Search filters on `evt.event` and `evt.user`. All these must continue to work with the mapped field names.
