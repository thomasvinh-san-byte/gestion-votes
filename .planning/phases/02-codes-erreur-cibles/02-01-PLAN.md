<objective>
Decomposer le code generique `business_error` (1 site d'emission unique : `AbstractController.php:53`) en codes cibles observables, en exploitant le fait que la majorite des `RuntimeException` metier portent deja un code snake_case en `getMessage()` (cf. audit grep : 49 sites levent deja `meeting_not_found`, 7 levent `motion_not_found`, etc., tous catch-alles en `business_error`).

**Strategie en 3 mouvements complementaires :**

1. **Detection passive au catch.** Ajout d'un test regex `^[a-z][a-z0-9]*(_[a-z0-9]+)*$` dans `AbstractController:catch (RuntimeException)` : si le message correspond, on l'utilise comme error_code API au lieu du generique. Cette seule modification fait basculer ~80% des `RuntimeException` services vers leur code specifique sans toucher aux services. Pas d'allowlist au runtime — tout code snake_case bien forme passe (ErrorDictionary fournit deja ~120 entrees, et `getDefaultMessage()` genere un fallback FR pour tout code inconnu).

2. **Normalisation des 3 derniers throws francais.** L'audit grep identifie 3 `throw new RuntimeException('Seance ...')` qui passent encore en `business_error` car leur message n'est pas snake_case. On les renomme (cf. table d'action ci-dessous).

3. **Enrichissement ErrorDictionary** avec les 2 nouveaux codes (next-step Norman v2.3 ERR-02 — virgule + verbe d'action). `archived_meeting_locked` et `validated_meeting_locked` n'existent pas encore dans le dictionnaire (seuls `meeting_archived_locked` et `meeting_validated_locked` existent — naming different, on choisit le pattern symetrique `<state>_meeting_locked`).

**Pourquoi cette strategie est correcte (vs. reecrire 94 services).** Le grep a montre que 94 sites de services levent deja `RuntimeException` avec un code snake_case en message (`meeting_not_found`, `motion_not_found`, `consolidation_forbidden`, `cannot_toggle_self`, etc.). Les normaliser un par un couterait ~94 modifications avec risque d'erreur. La detection au catch fait le travail en 1 fichier modifie (AbstractController.php), avec retrocompat parfaite.

```
private static function extractBusinessErrorCode(string $message): ?string {
    $trimmed = trim($message);
    if ($trimmed === '' || strlen($trimmed) < 3 || strlen($trimmed) > 40) {
        return null;
    }
    if (preg_match('/^[a-z][a-z0-9]*(_[a-z0-9]+)*$/', $trimmed) === 1) {
        return $trimmed;
    }
    return null;
}
```
