<objective>
Fix the multi-consumer SSE event loss bug and add proper nginx/PHP-FPM configuration for long-lived SSE connections.

Purpose: The current events.php uses a destructive LRANGE+DEL on a shared per-meeting Redis list — the first consumer to poll gets all events, subsequent consumers get nothing. This plan replaces the shared list with per-consumer fan-out queues so operator, voters, and projection screens all receive every event. It also adds a dedicated nginx location block and documents PHP-FPM sizing.

Output: Multi-consumer safe SSE pipeline + nginx SSE config + PHP-FPM sizing docs.
</objective>

private static function publishToSse(array $event): void {
    $meetingId = $event['meeting_id'] ?? null;
    if ($meetingId === null) { return; }
    if (self::useRedis()) {
        $redis = RedisProvider::connection();
        $sseKey = 'sse:events:' . $meetingId;

## Task 3: Nginx dedicated SSE location block

**File:** `deploy/nginx.conf`
**What:** Add a `location = /api/v1/events.php` block with SSE-specific configuration.

**Changes:**
1. Insert new location block after the login endpoint block (after line 107) and before the generic `.php$` handler (line 109):

```nginx
    # ── SSE endpoint — long-lived, no rate limiting, no buffering ──────────
    # Exact-match takes priority over regex \.php$ regardless of order.
    # Each SSE connection holds a PHP-FPM worker for ~30s (event loop duration).
    location = /api/v1/events.php {
        try_files $uri =404;
        fastcgi_pass 127.0.0.1:9000;
        fastcgi_index index.php;
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
        include fastcgi_params;

        # SSE requires unbuffered streaming to push events immediately
        fastcgi_buffering off;

        # Match PHP loop duration (30s) + 5s margin
        fastcgi_read_timeout 35s;
        fastcgi_send_timeout 35s;

        # No rate limiting — SSE auto-reconnects every 30s via EventSource API;
        # rate limiting causes 503 during simultaneous reconnections after blips
    }
```

2. Re-declare security headers inside the block (nginx inheritance rule — child locations with add_header lose parent headers). Copy the same security header block used in the static assets location.

**Must verify:**
