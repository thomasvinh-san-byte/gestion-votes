## Spacing Scale

All tokens already declared in design-system.css `--space-*` series. This phase MUST reference these tokens — no raw px values in any modified CSS file.

| Token | CSS Variable | Value | Usage |
|-------|-------------|-------|-------|
| xs | `--space-1` | 4px | Icon gaps, badge dot, inline padding |
| sm | `--space-2` | 8px | Compact badge padding, tight element gaps |
| md | `--space-4` | 16px | Default field spacing, card padding inner |
| lg | `--space-6` | 24px | Section padding, panel inner padding |
| xl | `--space-8` | 32px | Layout column gaps, card outer margin |
| 2xl | `--space-12` | 48px | Major section breaks |
| 3xl | `--space-16` | 64px | Page-level vertical rhythm |

Exceptions:
- Login 2-panel layout: branding panel uses `--space-12` (48px) horizontal padding to breathe
- Touch targets (interactive badges, nav items): minimum 44px height — use `min-height: 2.75rem` where needed

Gap aliases already defined: `--gap-xs` (4px), `--gap-sm` (8px), `--gap-md` (16px), `--gap-lg` (24px), `--gap-xl` (32px). Prefer these over raw `--space-*` in flex/grid contexts.

## Typography

Font sizes are already declared in design-system.css `--text-*` series. This phase does not change the type scale — it enforces existing tokens in per-page CSS files.

| Role | CSS Variable | Value | Weight | Line Height |
|------|-------------|-------|--------|-------------|
| Body / UI chrome | `--text-base` | 14px | `--font-normal` (400) | `--leading-base` (1.571) |
| Label / badge | `--text-xs` | 12px | `--font-medium` (500) | 1 (single-line, fixed) |
| Heading (card/section) | `--text-xl` | 20px | `--font-semibold` (600) | `--leading-xl` (1.35) |
| Display (branding panel h1) | `--text-4xl` | 36px | `--font-bold` (700) | `--leading-4xl` (1.1) |

Font family rule:
- All UI elements: `--font-sans` (Bricolage Grotesque)
- Login branding panel h1 and tagline: `--font-display` (Fraunces) — already used in login.css header

Weight constraint: only two weights in active use per view — 400 (body) + 600 (semibold for headings and labels). 700 reserved exclusively for the login branding panel h1.

Accent is NOT used for: badges (use semantic colors below), informational text, decorative icons, table row hover.

## Layout Contracts

### Login 2-Panel Layout (DESIGN-02)

Source: CONTEXT.md decisions + memory feedback_horizontal_design.md + project_login_redesign.md

```
[  Branding Panel (left 50%)  ] [  Form Panel (right 50%)  ]
[  gradient orb background    ] [  white surface-raised     ]
[  logo + AgVote + tagline    ] [  420px form, centered     ]
```

- Container: `display: grid; grid-template-columns: 1fr 1fr; min-height: 100vh`
- Branding panel: uses the existing `orb-drift` animation + `--color-primary-subtle` / `--color-primary-glow` gradient (already in login.css). Dark overlay: `oklch(0 0 0 / 0.25)` for text legibility.
- Form panel: `background: var(--color-surface-raised)` (white). Form remains at `max-width: 420px`, horizontally centered within the panel.
- Breakpoint `768px`: `grid-template-columns: 1fr`. Branding panel collapses to hidden (`display: none`). Form panel takes full width.
- The existing `.login-card` card UI is NOT removed — it is placed inside the form panel instead of full-page.
- No new CSS files. Modify `login.css` only. Replace `.login-page` layout. Preserve `.login-orb`, `.login-card`, `.login-brand` internals.

### Horizontal-first enforcement (per-page CSS)

Screens are horizontal (minimum layout target: 1366px wide). Per-page CSS token cleanup must not collapse multi-column layouts to single column. Specific rules:
- Dashboard KPI row: minimum 4 columns at `768px+`, flex-wrap allowed only below `480px`
- Hub / meetings list: 2-column grid at `1024px+`
- Form sections in wizard/settings: 2-column field layout at `768px+` (not single column)
- Replace any `flex-direction: column` on content-width containers above `768px` with row or grid

### Loading States (DESIGN-03)

The design-system.css already defines `.skeleton` shimmer, `.htmx-indicator`, and `skeleton-shimmer` keyframe. Phase 6 applies these consistently:

- Network request on element with `.htmx-request` class: add `.skeleton` pulse class to child loading placeholders — NOT a full-page overlay
- Button loading state: replace button text with inline `<span class="htmx-indicator">...</span>` + spinner SVG icon, hide with `.htmx-indicator { display: none }` / show with `.htmx-request .htmx-indicator { display: inline-flex }`
- Table/list load: use `.skeleton-row` pattern (already defined) as placeholder rows
- Opacity fade-in on content arrival: `opacity: 0` → `opacity: 1` over `200ms var(--ease-standard)` via `@starting-style` where browser support allows (progressive enhancement)
- No custom CSS animation keyframes to be created — reuse `skeleton-shimmer` from design-system.css
