For each of the 4 target component files, locate every hex/rgba literal inside the `render()` / template string (which injects into Shadow DOM) and replace it with the appropriate `var(--*)` token from `design-system.css`.

**Replacement rules:**
1. **Hex inside a `var(--token, #fallback)` fallback:** remove the fallback entirely. Design-system.css tokens are guaranteed to resolve on all pages (shell.js loads design-system.css before any component registers). Fallbacks were defensive relics from Phase 6 and are now the primary dark-mode leak.
   - `var(--color-text, #1a1a1a)` → `var(--color-text)`
   - `var(--color-border, #d5dbd2)` → `var(--color-border)`
   - `var(--color-surface-raised, #ffffff)` → `var(--color-surface-raised)`
2. **Standalone hex outside any var():** replace with the closest semantic token.
   - `#1a1a1a` (text) → `var(--color-text)`
   - `#ffffff` / `#fff` (surface) → `var(--color-surface-raised)` when used as background, `currentColor` when used as SVG stroke on a tinted chip
   - `#666` or `#95a3a4` (muted text) → `var(--color-text-muted)`
   - Any hex paired with `background:` → map to `--color-surface*` or `--color-*-subtle`
   - Any hex paired with `border-color:` → `--color-border` or `--color-border-strong`
3. **rgba() for shadows:** replace with the `--shadow-*` token from the existing scale.
   - `rgba(0,0,0,0.1)` in a box-shadow → `var(--shadow-sm)` or `var(--shadow-md)` depending on elevation intent.
4. **SVG icon strokes currently hex:** change to `stroke="currentColor"`. Set the parent element's `color:` via a `var(--color-*)` token. This lets the icon re-tint on theme change automatically.
5. **Type-variant shadows in ag-badge/ag-toast-like patterns:** the semantic accent color for success/error/warning/info should come from `--color-success`, `--color-danger`, `--color-warning`, `--color-info` (no fallbacks).
