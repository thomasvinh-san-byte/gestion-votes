## Spacing Scale

Declared values (8px base grid, expressed as `--space-*` tokens):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`--space-1`) | Icon-to-label gap, tight inline spacing |
| sm | 8px (`--space-2`) | Toast gap stack, compact badge padding |
| md | 16px (`--space-4`) | Form field vertical rhythm, button padding horizontal |
| lg | 24px (`--space-6`) | Card internal padding (`--space-card`), section dividers |
| xl | 32px (`--space-8`) | Layout column gaps |
| 2xl | 48px (`--space-12`) | Major section breaks (`--space-section`) |
| 3xl | 64px (`--space-16`) | Page-level outer padding |

Exceptions:
- Toast container offset from viewport edge: 16px (`--space-4`) — fixed position, on grid
- Sidebar collapsed rail: 56px (`--sidebar-rail`) — 4×14, fixed by sidebar design contract
- Sidebar expanded: 252px (`--sidebar-expanded`) — fixed by sidebar design contract
- Touch targets for sidebar nav items: minimum 44px height enforced by existing nav-item styles

## Component Contracts

### POLISH-01 — Toast Notification System

`ag-toast` component and `AgToast.show()` static method are already implemented.

**API (locked):**
```
AgToast.show(type, message, duration?)
  type: 'success' | 'error' | 'warning' | 'info'
  message: string (French, escaped HTML)
  duration: ms — default success/info=5000, warning/error=8000, 0=persistent
```

**Visual contract:**
- Container: `position: fixed; top: 16px; right: 16px; z-index: var(--z-toast, 800)`
- Toast width: 356px (`--toast-width`)
- Accent stripe: 3px left inset box-shadow in semantic color (`--toast-accent-width`)
- Dismiss animation: `translateX(20px) scale(0.96)` over 300ms
- Entry animation: reverse of dismiss, 200ms
- Maximum 3 toasts stacked; oldest removed on 4th arrival
- `role="alert" aria-live="polite"` on inner `.toast` div
- `prefers-reduced-motion: reduce` disables animation entirely

**Adoption target:** At minimum these 5 pages must call `AgToast.show()` for save/error/info feedback instead of `alert()` or inline DOM messages:
1. `settings.js` (save settings)
2. `members.js` (add/remove member)
3. `operator.js` (vote control actions)
4. `hub.js` (attachment upload)
5. `users.js` (user create/edit/delete)

**Integration pattern** (how JS files must call it):
```js
// On HTMX success response
AgToast.show('success', 'Enregistrement reussi');
// On API error
AgToast.show('error', 'Une erreur est survenue. Veuillez reessayer.');
// On warning
AgToast.show('warning', 'Attention : quorum non atteint.');
// On neutral info
AgToast.show('info', 'Synchronisation en cours...');
```

### POLISH-02 — Dark Mode Parity

**Audit scope:** All 21 applicative HTML pages rendered with `[data-theme="dark"]`.

**Fix criteria (per page):** Zero hardcoded hex/rgba/hsl values in the page's CSS file. All colors via `var(--*)` tokens. Verified by:
```bash
grep -nE '#[0-9a-fA-F]{3,6}|rgba?\(|hsla?\(' public/assets/css/{page}.css
```

**Known debt to fix (from STATE.md):**
- 25 per-page CSS files with ~15 hardcoded hex/rgba values: `analytics.css`, `meetings.css`, `hub.css`, `vote.css`, `public.css`, `users.css`
- 23 Web Component Shadow DOM files with fallback hex literals — update fallback values to match OKLCH equivalents
- `critical-tokens` inline styles in 22 HTML files — 6 hex values

**Dark mode token direction (from STATE.md decisions):**
- Dark surface hue: `oklch(_, _, 78)` warm-neutral — do not change
- Dark hover: `color-mix(in oklch, base 88%, white)` — lightening direction, never darkening

**Test method:** Toggle via `#btnToggleTheme` in sidebar footer (already wired), assert visual appearance per page.

### POLISH-04 — Micro-interactions

All interactive elements must implement the following states using existing tokens. No new tokens introduced.

**Focus ring (all focusable elements):**
```css
:focus-visible {
  outline: none;
  box-shadow: var(--shadow-focus);
  /* which resolves to: 0 0 0 2px var(--color-surface-raised), 0 0 0 4px var(--color-border-focus) */
}
```
- `--color-border-focus`: `oklch(0.520 0.195 265 / 0.50)` — blue-600 at 50% alpha
- Focus ring must be visible in both light and dark mode
- Destructive buttons use `var(--shadow-focus-danger)` instead

**Hover states (primary buttons):**
```css
.btn-primary:hover {
  background: var(--color-primary-hover); /* color-mix(in oklch, primary 88%, black) */
  transform: translateY(-1px);
  box-shadow: var(--shadow-md);
  transition: var(--transition-ui);
}
```

**Button press (active/pressed):**
```css
.btn-primary:active {
  transform: scale(0.98);
  background: var(--color-primary-active); /* color-mix(in oklch, primary 76%, black) */
  box-shadow: var(--shadow-sm);
}
```

**Secondary/outline button hover:**
```css
.btn-secondary:hover {
  background: var(--color-neutral-subtle);
  border-color: var(--color-border-strong);
}
```

**Nav item hover (sidebar):**
```css
.nav-item:hover {
  background: var(--sidebar-hover); /* oklch(1 0 0 / 0.10) */
}
```

**Loading / skeleton states:**
- Lists and KPI cards use skeleton shimmer while data loads
- Shimmer: `background: linear-gradient(90deg, var(--color-surface-alt), var(--color-surface-raised), var(--color-surface-alt))` animated at `1.5s infinite`
- Shimmer disabled if `prefers-reduced-motion: reduce` — use static `var(--color-surface-alt)` instead
- Spinner: `<ag-spinner>` component — use for in-button loading states

**Transition durations:**
- Color/border transitions: `var(--duration-normal)` = 150ms
- Transform transitions: `var(--duration-moderate)` = 200ms
- Enter/exit animations: `var(--duration-deliberate)` = 300ms
- All transitions disabled when `prefers-reduced-motion: reduce`

## Copywriting Contract

All visible text is in French. Never use "copropriete" or "syndic".

| Element | Copy |
|---------|------|
| Toast: save success | "Enregistrement reussi" |
| Toast: generic error | "Une erreur est survenue. Veuillez reessayer." |
| Toast: network error | "Connexion perdue. Verifiez votre reseau." |
| Toast: warning | "Attention : [operation] pourrait etre incomplete." |
| Toast: info loading | "Chargement en cours..." |
| Toast: copy/export success | "Copie reussie" / "Export telecharge" |
| Empty state heading | "Aucune donnee disponible" |
| Empty state body | "Cette liste est vide pour le moment. Commencez par [action specifique]." |
| Empty state CTA | "Creer [element]" (specific verb + noun per page) |
| Error state | "Impossible de charger les donnees. [Reessayer] ou contactez l'administrateur." |
| Destructive: confirm delete | "Supprimer [element]" — confirmation copy: "Cette action est irreversible. Confirmer ?" |
| Destructive: confirm close | "Cloturer la seance" — confirmation copy: "La seance sera marquee comme cloturee. Continuer ?" |

Destructive confirmation approach: `AgConfirm.ask(message)` — Promise-based confirm dialog (existing `ag-confirm` component). Never `window.confirm()`.

## Interaction States — Summary

Every button, input, and link on every page must satisfy:

| State | Visual indicator |
|-------|-----------------|
| Default | Base token colors, no transform |
| Hover | Background shift via color-mix, translateY(-1px) for primary CTA |
| Focus-visible | `var(--shadow-focus)` ring — 2px white + 4px blue outline |
| Active/Pressed | `scale(0.98)` transform |
| Loading | `<ag-spinner>` inside button OR skeleton shimmer on list |
| Disabled | `opacity: 0.5`, `cursor: not-allowed`, no hover/focus effects |
| Error | `var(--color-danger)` border + `var(--shadow-focus-danger)` on focus |
