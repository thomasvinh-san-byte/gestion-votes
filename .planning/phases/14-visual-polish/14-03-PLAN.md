<!-- filterSidebar() mechanism — do not modify, rely on it -->
From public/assets/js/pages/auth-ui.js (lines 370-381, 387-404):
```javascript
function filterSidebar() {
  var sidebar = document.querySelector('[data-include-sidebar]');
  if (!sidebar) return false;
  var role = window.Auth.role;
  var meetingRoles = window.Auth.meetingRoles || [];
  sidebar.querySelectorAll('[data-requires-role]').forEach(function (el) {
    hasAccess(el.getAttribute('data-requires-role'), role, meetingRoles) ? _show(el) : _hide(el);
  });
  return true;
}
// MutationObserver re-applies filterSidebar() once sidebar partial is injected
```

`_hide(el)` sets `el.style.display = 'none'`.
`hasAccess(required, currentSystemRole, meetingRoles)` admits admin to everything; otherwise splits comma list and tests each role against the SYSTEM_ROLE_LEVEL hierarchy or meetingRoles exact match.
