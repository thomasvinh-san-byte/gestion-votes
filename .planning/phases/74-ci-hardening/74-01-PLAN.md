Replace the `migrate-check` job with a version that:
1. Adds a `services.postgres` block (copy from the `integration` job — same image postgres:16.8-alpine3.21, same health check options)
2. Adds a "Setup PHP" step (needed because validate-migrations.sh uses psql client — but actually psql is available on ubuntu-latest without PHP; use `apt-get install -y postgresql-client` instead)
3. Removes `--syntax-only` from the validate-migrations.sh call
4. Passes the correct env vars so the script connects to the service postgres

Replace the entire migrate-check job with:

```yaml
  # ---------------------------------------------------------------------------
  # Migration idempotency check: full two-pass PostgreSQL validation
  # ---------------------------------------------------------------------------
  migrate-check:
    runs-on: ubuntu-latest
    services:
      postgres:
        image: postgres:16.8-alpine3.21
        env:
          POSTGRES_DB: agvote_migration_ci
          POSTGRES_USER: agvote_ci
          POSTGRES_PASSWORD: agvote_ci
        ports:
          - 5432:5432
        options: >-
          --health-cmd "pg_isready -U agvote_ci -d agvote_migration_ci"
          --health-interval 10s
          --health-timeout 5s
          --health-retries 5
    steps:
      - name: Checkout
        uses: actions/checkout@v6

      - name: Install PostgreSQL client
        run: sudo apt-get install -y --no-install-recommends postgresql-client

      - name: Validate migrations (full idempotency, two-pass PostgreSQL)
        run: bash scripts/validate-migrations.sh
        env:
          DB_HOST: localhost
          DB_PORT: 5432
          DB_USERNAME: agvote_ci
          DB_DATABASE: agvote_migration_ci
          DB_PASSWORD: agvote_ci
```

Key decisions:
- Use a dedicated DB name/user (agvote_migration_ci) to avoid any name collision with the integration job
- The script uses `createdb` which requires the user to have CREATEDB privilege. On the postgres service container, the user is created via POSTGRES_USER which gets CREATEDB by default in the postgres Docker image.
- Remove `--syntax-only` completely — the full run catches SQLite patterns on first-pass failure AND idempotency bugs on second pass
- The `build` job's `needs` array already lists `migrate-check` — no change needed there

Do NOT add PHP setup — the script only needs bash + psql client tools (createdb, dropdb, psql).
