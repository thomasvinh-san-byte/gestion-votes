# Phase 20.1: Refonte UI alignement wireframe et réduction charge mentale - Context

**Gathered:** 2026-03-17
**Status:** Ready for planning

<domain>
## Phase Boundary

Réaligner TOUTES les pages du site avec la composition du wireframe v3.19.2 "Acte Officiel" pour que l'application guide l'utilisateur et allège la charge mentale. Les tokens couleurs sont déjà alignés (Phase 4) — cette phase corrige la composition, la hiérarchie visuelle, la densité d'information, et le bug FOUC. On travaille avec le code existant, en améliorant ce qui est là.

</domain>

<decisions>
## Implementation Decisions

### Layout votant (tablette)
- Boutons de vote en **grille 2×2** comme le wireframe (Pour/Contre en haut, Abstention/NSP en bas) — pas de stack vertical
- Confirmation de vote via **bouton inline coloré** ("Confirmer : Pour" dans la couleur du choix) — supprimer le bottom sheet overlay sombre actuel
- Header votant **minimal** : compteur résolution (3/5) + timer uniquement — supprimer titre "Vote", sous-titre, bouton actualiser
- Identité votant : appliquer les bonnes normes de design — le wireframe place nom + poids en bas de page, discret mais toujours visible

### Projection salle
- Fond en **gradient** : `linear-gradient(170deg, #0B0F1A, #111827)` au lieu de l'aplat uni actuel
- Titre de séance en **grand** (wireframe: `clamp(32px, 3.6vw, 56px)`) avec **barre quorum horizontale** intégrée
- Barres de résultats avec **`clamp()` responsive** pour s'adapter de petit écran à projecteur 4m
- Mode clair/sombre avec thème adapté (wireframe fournit les deux palettes)

### Hiérarchie visuelle globale
- Approche **"moins c'est plus"** sur toutes les pages : chaque page a UN focus clair, éléments non essentiels supprimés
- Espacement généreux, moins de pills/badges/boutons secondaires
- Toutes les pages reprennent le format wireframe et l'améliorent — mixer densité et options quand pertinent

### Dashboard
- Pattern wireframe : **3 cartes raccourcis** (Créer une séance, Piloter un vote, Consulter le suivi) + **4 KPIs essentiels** en temps réel
- La page d'accueil est un lanceur d'actions, pas un tableau de bord dense

### Console opérateur
- Modales simples : **titre + contenu essentiel + 2 boutons** (annuler/confirmer) — pas de formulaires complexes dans les modales

### FOUC / assombrissement
- **Inline les tokens CSS critiques** dans le `<head>` de chaque page HTML pour éliminer le flash de contenu non stylé causé par la chaîne `@import`

### Claude's Discretion
- **KPIs projection** : Claude évalue si les 3 KPIs compacts (Présents, Quorum, Résolution) sont pertinents dans le header projection
- **Hiérarchie typographique** : Claude applique la bonne répartition Fraunces (display/titres) vs Bricolage Grotesque (corps) pour renforcer le caractère officiel
- **Densité tableaux** (membres, etc.) : Claude adapte la densité au contexte de chaque page
- **Vue exec opérateur** : Claude réorganise pour réduire la charge mentale sans perdre de fonctionnalité — focus résolution en cours, onglet Avancé secondaire
- **États vides/erreur** : Claude applique le pattern approprié (icône + titre + description + CTA optionnel) selon le contexte
- **Sidebar** : Claude évalue si les labels, groupes et icônes ont besoin d'améliorations
- **Navigation mobile** : Claude évalue si le responsive a besoin d'ajustements
- **Pages settings/help** : Claude évalue page par page, reprend le format wireframe et l'améliore

</decisions>

<specifics>
## Specific Ideas

- Le wireframe v3.19.2 est la **référence absolue** pour la composition — fichier `ag_vote_wireframe.html` dans la racine du projet
- Le wireframe utilise des tailles `clamp(min, preferred, max)` pour la projection — s'adapter du petit écran au projecteur 4m
- Le wireframe a un style "Acte Officiel" — hiérarchie forte, peu d'éléments, chaque écran guide l'action suivante
- L'utilisateur veut que le site "guide" et "aide à avancer" — pas juste un squelette fonctionnel mais un outil qui réduit la charge mentale
- "Mélanger les 2 pour avoir le plus d'options possibles" — sur les pages admin, combiner densité informationnelle et clarté wireframe

</specifics>
