1. **Capturer la baseline** (avant modification) :

```bash
BASELINE=$(grep -cE "(padding|margin):\s+[0-9]+(\.[0-9]+)?(px|rem|em)" public/assets/css/audit.css)
echo "Baseline hardcoded padding/margin: $BASELINE"
```

Noter cette valeur dans le SUMMARY.

2. **Remplacement systématique**. Pour chaque déclaration `padding: ...` ou `margin: ...` ou `padding-{top,right,bottom,left}: ...` ou `margin-{top,right,bottom,left}: ...` ou `padding-inline: ...` / `padding-block: ...` / `margin-inline: ...` / `margin-block: ...` :

   - Si la valeur est `0`, `auto`, ou en `%` ou `em` → laisser.
   - Sinon → remplacer chaque valeur numérique par le token correspondant (cf. table).
   - Pour les déclarations multi-valeurs (ex: `padding: 8px 16px;`) → `padding: var(--space-2) var(--space-4);`.

3. **Bonus shadow / border** (TECH-01) — si une règle CSS touchée contient aussi `box-shadow:` ou `border:` :

   - `box-shadow: 0 1px 2px ...` (élévation discrète) → `box-shadow: var(--shadow-xs)`
   - `border: 1px solid #...` ou `border: 1px solid var(--color-border)` → `border: var(--border-default)`
   - `border: 1px solid var(--color-border-subtle)` → `border: var(--border-subtle)`
   - `border: 1px dashed ...` → `border: var(--border-dashed)`
   - Si la valeur ne mappe pas évidemment, **laisser** et noter dans SUMMARY.

4. **Vérifier post-modification** :

```bash
AFTER=$(grep -cE "(padding|margin):\s+[0-9]+(\.[0-9]+)?(px|rem|em)" public/assets/css/audit.css)
echo "After: $AFTER"
# Cible : 0
```

Note importante : la regex baseline capture aussi les valeurs multi-arg comme `padding: 8px 16px;` (commence par chiffre). Pour les multi-arg, utiliser une regex plus permissive si besoin :

```bash
grep -nE "(padding|margin)(-(top|right|bottom|left|inline|block))?:\s+[0-9]" public/assets/css/audit.css
```

Cible : 0 ligne en sortie après cleanup (à part celles avec `0` seul ou `auto`, qui ne matchent pas `[0-9]` puisque `0` matche mais `0` seul est OK — utiliser une regex qui exclut `0;` et `0$` pour éviter faux positifs).

Regex finale recommandée pour validation :
```bash
grep -nE "(padding|margin)(-[a-z]+)?:\s+[0-9]+(\.[0-9]+)?(px|rem|em)" public/assets/css/audit.css
```
Cette regex match uniquement les valeurs avec unité explicite (px/rem/em), donc exclut `padding: 0;`. **Cible : 0 lignes.**

5. **Vérifier syntaxe CSS** : pas d'outil PHP-CS-Fixer pour CSS, mais s'assurer manuellement qu'aucun `;` ou `}` n'est cassé. Optionnel : `npx --yes csso public/assets/css/audit.css --no-restructure -o /tmp/audit.min.css` pour parse-check (skip si csso non dispo).
