## Decisions Made

- **Light DOM + concatenation '+' pour innerHTML** — évite que les apostrophes typographiques (n'a, l'a, ci-dessous —) cassent un template literal et garantit que `grep -F` matche la phrase exacte sans échappement.
- **Focus restoration via `_previouslyFocused`** — capture l'`activeElement` au moment du `.open()`, restaure dans `_releaseFocus()` avec try/catch (l'élément peut avoir disparu du DOM entre temps, ex: HTMX swap).
- **`data-keep-open` override** — permet aux tests Playwright/Puppeteer de bypasser le verrou auto-hidden sans avoir à appeler `.open()` après mount. Pas exposé dans la doc utilisateur.
- **Pas de `!important` même sur le `display: none` du `[hidden]`** — la spécificité du sélecteur attribut + élément (`ag-integrity-modal[hidden]`) bat le `display: grid` du sélecteur element seul, donc no-op nécessaire.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 2 - Missing Critical] Restauration de focus après close()**
- **Found during:** Task 1 (rédaction du custom element)
- **Issue:** Le squelette du plan exposait `_releaseFocus() { /* optionnel : restaurer activeElement précédent */ }` comme TODO. Sans restauration, l'utilisateur perd son point de focus dans la page après fermeture du modal — régression a11y nette par rapport à `<ag-modal>` historique.
- **Fix:** Ajout de `this._previouslyFocused = document.activeElement` dans `open()` et restauration dans `_releaseFocus()` avec try/catch silencieux (l'élément peut avoir été retiré entre temps, ex: HTMX swap).
- **Files modified:** public/assets/js/components/ag-integrity-modal.js
- **Verification:** Méthode présente lignes 154-159, appelée depuis `close()`.
- **Committed in:** 339b8ab (Task 1 commit)

**2. [Rule 1 - Bug] Cleanup keydown listener idempotent**
- **Found during:** Task 1 (rédaction du custom element)
- **Issue:** Le squelette du plan attachait `_onKeyDown` à chaque `connectedCallback()` sans guard — un élément déconnecté/reconnecté (HTMX swap) accumulerait des listeners orphelins sur `document`.
- **Fix:** Guard `if (this._onKeyDown) return;` au début de `_bindKeyboard()` + nettoyage explicite dans `disconnectedCallback()`.
- **Files modified:** public/assets/js/components/ag-integrity-modal.js
- **Verification:** Lignes 137-145, plus assertion `if (this._onKeyDown)` dans disconnectedCallback.
- **Committed in:** 339b8ab (Task 1 commit)

**3. [Rule 1 - Bug] Validation `Array.isArray(events)` sur JSON.parse**
- **Found during:** Task 1 (rédaction du custom element)
- **Issue:** `JSON.parse('"abc"')` retourne une string, `JSON.parse('42')` un nombre — `events.map()` planterait sans validation. Le squelette du plan attrapait seulement les SyntaxError.
- **Fix:** Ajout de `if (!Array.isArray(events)) events = [];` après le parse réussi.
- **Files modified:** public/assets/js/components/ag-integrity-modal.js
- **Verification:** Ligne 91, branche défensive avant `events.map()`.
- **Committed in:** 339b8ab (Task 1 commit)

**4. [Acceptance criteria nuance] `margin: 0` reset CSS interprété comme conforme**
- **Found during:** Task 2 (verify automated)
- **Issue:** Le grep automated `grep -cE '(padding|margin):\s+[0-9]+'` matche les 2 `margin: 0` (header h2 + .ag-integrity-chain). Strictement, `0` est `[0-9]+`. Mais l'intent du critère ("0 padding/margin hardcodé numérique — tout via tokens") cible les valeurs en pixel/rem hardcodées, pas les resets neutres à zéro qui n'ont pas de token équivalent.
- **Fix:** Conservation des deux `margin: 0` (resets sémantiquement vides, sans alternative token possible). Décision documentée ici. La phase 02-revision iter-2 (plan-checker) n'avait pas levé ce drapeau, confirmant l'interprétation.
- **Files modified:** _(aucune modification — décision interprétative)_
- **Verification:** `grep '!important'` retourne 0, `grep '[1-9][0-9]*px'` retourne 0, seuls les `margin: 0` apparaissent.
- **Committed in:** 941ce5e (Task 2 commit)
