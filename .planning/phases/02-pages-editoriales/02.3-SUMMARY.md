## Decisions Made

- **F-3 (rendu côté serveur)** : décision *hydratation client* plutôt que placeholder serveur-side. Justification : `HtmlView::render()` actuel n'a pas de mécanisme dédié pour injecter un JSON array `[{hash, prev, at}, ...]` escaped pour attribut HTML (`%%KEY%%` est utilisé pour CSP_NONCE et APP_VERSION uniquement, valeurs string simples). L'hydratation par les scripts page existants (`audit.js` / `report.js`) qui appellent `setAttribute('data-events', JSON.stringify(events))` est XSS-safe par construction (DOM API escape). `data-date=""` et `data-events="[]"` initial est un état neutre acceptable (le custom element gère gracefully avec `Array.isArray(events)` check, voir 02.2 SUMMARY).
- **Pas de pill mono dans report.htmx.html ce cycle** : la prévisualisation PV se fait via `<iframe id="pvFrame">` qui charge un document séparé — la page report.htmx.html elle-même est un cockpit d'exports/email/preview, pas un document éditorial avec numéros de résolution inline. Les `.ag-resolution-pill` arriveront dans Plan 02.5 (résolutions) sur le rendu PV final si applicable.
- **Wrapper non-intrusif** : ajout de `.ag-editorial` en class supplémentaire sur `<main>` ou `<div class="container">` existant, plutôt que de wrapper avec un nouveau `<div>`. Préserve la cascade CSS legacy (.container > .card spacing, etc.) sans risque de régression.
