## Decisions Made

- **F-5 lock Schoger respecté** : pas de `position: running(name)` ni de `@page { @top-left { content: element(...) } }`. Ces propriétés CSS Paged Media sont supportées uniquement par Prince/antiword (moteurs PDF dédiés) et IGNORÉES par Chrome/Firefox/Safari à l'impression navigateur. Pragmatic fallback : `<header class="ag-editorial-print-header">` masqué par défaut (déclaré `display: none` ligne 88, hors @media print), révélé dans `@media print` via `display: block`. Apparaît une fois en début de document.
- **EDITORIAL-07 livré PARTIEL** : la spec demande "en-tête répété (titre séance + date) et numéro de page en footer". Le numéro de page est livré (counter(page)/counter(pages) au footer @bottom-right). L'en-tête répété sur chaque page n'est PAS livrable en CSS navigateur standard ; livré comme en-tête présent en début de document uniquement. Si répétition par page requise (besoin métier confirmé), route via dompdf (ProcurationPdfService déjà présent dans le codebase) en backlog v2.4 — la rendering pipeline serveur supporte les CSS Paged Media completes.
- **`.ag-resolution-pill` retiré du `page-break-inside: avoid`** : NIT N-2 du checker iter 1. Une pill inline (`display: inline-flex`) ne peut pas, par construction CSS, être coupée par un saut de page (les inlines suivent leur ligne parent). La règle sur `.ag-resolution-pill` était donc redondante. La pill reste typographiquement protégée (`white-space: nowrap` hérité + `font-family: var(--font-mono)` redéclaré en print).
- **`!important` autorisé dans `@media print`** : exception explicite à la règle "no `!important`" de Plan 02.1 (le fichier a `grep -c "!important" = 0` hors @media print). C'est le pattern standard CSS Paged Media : forcer l'override des styles écran à l'impression (les sélecteurs print ne peuvent pas tous gagner par spécificité face aux frameworks utilitaires).
- **Marges A4 2cm/1.5cm** : convention "document légal" (PV de séance, audit). 2cm haut/bas pour la respiration et le footer, 1.5cm gauche/droite pour maximiser la largeur de texte (le wrapper `.ag-editorial` passe en `display: block` `max-width: 100%`).
- **`.ag-integrity-footer` conservé visible en print** : c'est le hash du PV (preuve d'intégrité). Le lien clickable (`.ag-integrity-footer a`) est masqué — inutile sur papier — mais le hash lui-même reste visible. `border-top: 1px solid #000` pour signaler la séparation, `page-break-before: avoid` pour le coller au contenu précédent (le hash doit rester avec son PV).
- **Liens externes affichés en mono** : `.ag-editorial a[href]:not([href^="#"]):after { content: " (" attr(href) ")" }`. Sur papier, un lien est inutile cliquable — l'URL doit être donnée comme contexte. Sélecteur `:not([href^="#"])` exclut les ancres internes (`#section`), inutiles sur papier (le lecteur tourne déjà les pages). Mono pour distinction visuelle de l'URL vs texte.
