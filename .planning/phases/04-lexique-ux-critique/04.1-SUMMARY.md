## Migrations détaillées

### Migration 1 — operator.htmx.html L1062

```diff
- <input type="text" class="form-input form-input-sm" id="manualVoteSearch" placeholder="Nom du membre…" aria-label="Rechercher un votant">
+ <input type="text" class="form-input form-input-sm" id="manualVoteSearch" placeholder="Nom du votant…" aria-label="Rechercher un votant">
```

**Justification** : ce champ vit dans le bloc de **saisie manuelle d'un vote** (mode dégradé opérateur). Le `<label>` ligne 1061 dit déjà `Rechercher un votant` et l'`aria-label` aussi — seul le `placeholder` était incohérent. Aligné sur le terme dominant. 1 ligne diff.

### Migration 2 — ErrorDictionary.php L227

```diff
- 'token_member_mismatch' => 'Le token ne correspond pas à ce membre.',
+ 'token_member_mismatch' => 'Le token ne correspond pas à ce votant.',
```

**Justification** : ce message d'erreur est émis dans le contexte exclusif d'un **token de vote** (scrutin actif). Le token n'a de sens qu'attaché à un éligible au scrutin courant — c'est le cas typique LEX-01 où la nuance "éligible scrutin" est mieux portée par `votant`. La clé technique `token_member_mismatch` reste inchangée pour préserver le contrat API (callers `ErrorDictionary::getMessage('token_member_mismatch')`).
