# Phase 04 Plan 04.5 : UxConventionsTest — Filet permanent ERR-02 + ERR-03 — Summary

Création de `tests/Security/UxConventionsTest.php` : 3 tests PHPUnit qui scannent `app/Services/ErrorDictionary.php` via son API publique et garantissent (a) la présence d'un next-step actionnable (virgule + verbe d'action) sur les 50 codes les plus émis et (b) l'absence des 5 phrases creuses interdites par ERR-03 sur l'ensemble du dictionnaire. 3 tests verts au premier run, 4 assertions, 4 ms. Filet permanent : toute PR future qui dégrade un message du top 50 ou réintroduit une phrase bannie échoue immédiatement en CI.

## What Was Built

### `tests/Security/UxConventionsTest.php` (227 lignes)

Classe `final class UxConventionsTest extends TestCase` dans le namespace `Tests\Security`, utilisant `AgVote\Service\ErrorDictionary`.

3 méthodes de test publiques :

| # | Test | Asserts | Couverture |
|---|------|--------:|------------|
| 1 | `testTop50CodesContainsAtLeast50Entries` | 1 (count >= 50) | Garde-fou F2 : la liste TOP_50_CODES doit contenir au moins 50 codes. Sans cette assertion, testNextStepPresence pourrait passer vert avec une couverture <50 — ERR-02 partiellement satisfait sans détection. |
| 2 | `testNextStepPresence` | 1 (assertSame [], $missing) | ERR-02 : pour chaque code de TOP_50_CODES, charge le message via `ErrorDictionary::getMessage()`, vérifie qu'il contient au moins une virgule ET un verbe d'action de la liste blanche. Échec explicite citant chaque code défaillant. |
| 3 | `testNoForbiddenPhrases` | 2 (assertSame [], $hits ; implicite via getCodes() iter) | ERR-03 : pour chaque code retourné par `ErrorDictionary::getCodes()`, charge le message et vérifie qu'il ne matche AUCUNE des 5 regex bannies. Scan exhaustif (top 50 ou non). |

3 constantes de classe :

- `TOP_50_CODES` (50 strings, ranking copié depuis 04.4-SUMMARY.md table)
- `ACTION_VERBS` (53 verbes en français : impératif 2e personne pluriel + variantes accentuées/non-accentuées + pivot `'que '`)
- `FORBIDDEN_PATTERNS` (5 regex case-insensitive : réessayer., contactez (le|l')admin, erreur survenue, une erreur est survenue, veuillez réessayer plus tard)

## Out-of-scope captured (suivi v2.4 ou plan ultérieur)

1. **4 codes émis mais absents du dictionnaire** (`authentication_required`, `access_denied`, `invalid_pdf_magic`, `file_not_found`) — déjà documenté dans 04.4-SUMMARY out-of-scope. Ces codes ne sont **pas inclus dans TOP_50_CODES** car ils ne respectent pas le critère "doit exister dans ErrorDictionary::MESSAGES". Si ajoutés ultérieurement, leur enrichissement next-step + insertion dans TOP_50_CODES se ferait simultanément (1 commit).
2. **`rate_limit_exceeded`** : 1 émission grep, hors top 50, message borderline (« réessayer dans quelques instants »). Ne matche aucune des 5 regex bannies par `testNoForbiddenPhrases` (la phrase bannie est `/veuillez réessayer plus tard/i` — « plus tard » absent). À enrichir si remontée terrain le justifie.
3. **Codes longue traîne (~150 codes émis 0-1 fois)** : pas dans TOP_50_CODES, donc non couverts par `testNextStepPresence`. **Couverts par `testNoForbiddenPhrases` qui scanne le dictionnaire entier**. Acceptable car peu vus en production.
4. **Révision périodique TOP_50_CODES** : à effectuer ~1× par milestone si le ranking d'émissions change significativement (nouveaux endpoints, dépréciations). Process : re-grep statique `api_fail` sur `app/` + `public/`, comparer avec snapshot, mettre à jour la constante + commentaire de rank/émissions. Sans révision, le test reste vert mais la couverture peut dériver vers des codes plus rarement émis.
