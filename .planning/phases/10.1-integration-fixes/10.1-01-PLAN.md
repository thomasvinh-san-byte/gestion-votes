  <files>public/assets/js/pages/wizard.js</files>
  <action>
In wizard.js, fix the session creation code in the btnCreate click handler (around line 695-724):

1. **Line 703 — Fix api() argument order:** Change `api('POST', '/api/v1/meetings', payload)` to `api('/api/v1/meetings', payload, 'POST')`. The api() signature is `api(url, data, method)`, so currently 'POST' is passed as URL (causing 404), '/api/v1/meetings' as data, and payload as method.

2. **Line 712 — Fix response extraction:** Change `res.id || res._id || ''` to `res.body?.id || res.body?._id || ''`. The api() function returns `{ status, body }`, not the raw JSON body.

3. **Lines 719-720 — Replace Shared.showToast with AgToast.show:** Change:
   ```javascript
   if (window.Shared && Shared.showToast) {
     Shared.showToast('Erreur lors de la creation. Veuillez reessayer.', 'error');
   }
   ```
   to:
   ```javascript
   if (typeof AgToast !== 'undefined') {
     AgToast.show('error', 'Erreur lors de la creation. Veuillez reessayer.');
   }
   ```
   Note the argument order: AgToast.show(type, message), not (message, type).
