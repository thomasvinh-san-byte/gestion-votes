**B. Update btnGenerateReport click handler in postsession.js (lines 425-448):**

Replace the current handler that calls `meeting_generate_report.php` (HTML report) and uses `srcdoc` iframe with an inline PDF approach:

```javascript
click('btnGenerateReport', async function () {
  var btn = document.getElementById('btnGenerateReport');
  Shared.btnLoading(btn, true);
  try {
    var preview = document.getElementById('pvPreview');
    if (preview) {
      // Embed PDF inline via iframe src pointing to the PDF endpoint with inline=1
      var url = '/api/v1/meeting_generate_report_pdf.php?meeting_id='
        + encodeURIComponent(meetingId) + '&inline=1';
      preview.innerHTML = '<iframe src="' + url + '" style="width:100%;height:600px;border:none"></iframe>';
    }
    setNotif('success', 'Procès-verbal généré');
    // Show hash bar (fetch hash from response header is not possible via iframe,
    // so just indicate PV is generated)
    var hashBar = document.getElementById('pvHashBar');
    if (hashBar) hashBar.hidden = false;
  } catch (e) {
    setNotif('error', 'Erreur de génération du PV');
  } finally {
    Shared.btnLoading(btn, false);
  }
});
```

Key changes:
- Use `iframe src=` pointing to `meeting_generate_report_pdf.php?meeting_id=X&inline=1` instead of `srcdoc` with HTML content
- Remove the `window.api()` call to `meeting_generate_report.php` — we no longer need the HTML report endpoint for Step 3
- The browser will natively render the PDF inline in the iframe
- Keep the `setNotif('success', ...)` call
- Keep `Shared.btnLoading` pattern
