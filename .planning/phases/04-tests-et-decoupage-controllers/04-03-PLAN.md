**Test 1: testRedisConnectionLossHandling** (@group redis)
Purpose: Verify that Redis connection loss is handled gracefully by isServerRunning() and that queueRedis()/publishToSse() are explicitly documented as NOT handling connection loss (they propagate exceptions).

Implementation approach — two parts:
1. **Behavioral test (requires Redis):** Configure RedisProvider with a bogus host (e.g., `RedisProvider::configure(['host' => '255.255.255.255', 'port' => 1])`) so that `RedisProvider::connection()` throws. Call `EventBroadcaster::isServerRunning()` and assert it returns `false` (catch(Throwable) path). Then restore RedisProvider config in a finally block.
2. **Structural assertion (works without Redis too):** Read the source of `EventBroadcaster.php` via `file_get_contents()` or `ReflectionMethod::getFileName()`, find the `isServerRunning` method body, and assert it contains `catch` and `Throwable`. Also verify that `queueRedis` method body does NOT contain `catch` — documenting that connection loss in queue path is not handled (exception propagates).

Use this combined approach: try the behavioral test first. If RedisProvider::configure with bogus host works, great. If it throws during configure itself, fall back to structural-only. Wrap the behavioral part in try/catch so the test is robust.

The structural assertions should use ReflectionMethod to get start/end lines, then file_get_contents + array_slice to extract method bodies:
```php
$ref = new \ReflectionMethod(EventBroadcaster::class, 'isServerRunning');
$filename = $ref->getFileName();
$startLine = $ref->getStartLine();
$endLine = $ref->getEndLine();
$source = implode('', array_slice(file($filename), $startLine - 1, $endLine - $startLine + 1));
$this->assertStringContainsString('catch', $source, 'isServerRunning must have catch block for Redis failure');
$this->assertStringContainsString('Throwable', $source, 'isServerRunning must catch Throwable');
```

Same pattern for queueRedis (private method — ReflectionMethod still works for reading source):
```php
$ref = new \ReflectionMethod(EventBroadcaster::class, 'queueRedis');
// ... extract source ...
$this->assertStringNotContainsString('catch', $source, 'queueRedis must NOT catch exceptions — Redis failure propagates to caller');
```

**Test 2: testClientReconnectionDeliversBufferedEvents** (@group redis, requires Redis)
Purpose: Simulate a client disconnect/reconnect cycle. Events pushed while consumer is "disconnected" (not reading) remain in the per-consumer queue and are available on "reconnect" (reading the queue).

Implementation:
1. Generate unique meetingId and consumerId.
2. Register consumer: `$redis->sAdd("sse:consumers:{$meetingId}", $consumerId)`.
3. Enable push: `putenv('PUSH_ENABLED=1')`.
4. Push 3 events via `EventBroadcaster::toMeeting()` — simulating events during client disconnect (client is not consuming).
5. Read the consumer queue directly (simulating reconnect): `$redis->lRange("sse:queue:{$meetingId}:{$consumerId}", 0, -1)` with SERIALIZER_NONE.
6. Assert 3 events are present and in correct order (FIFO).
7. Pop all events from queue: `$redis->del("sse:queue:{$meetingId}:{$consumerId}")`.
8. Assert queue is now empty (clean reconnect — no duplicate delivery).
9. Push 1 more event. Assert queue has exactly 1 event (new events after reconnect work normally).
10. Track all keys in $this->testKeys for cleanup. Restore PUSH_ENABLED.

This test is distinct from testPublishToSseFansOutToRegisteredConsumers because it specifically tests the reconnection flow: multiple events buffered, read in order, queue drained, new events arrive normally.
