  <name>Task 1: Create audit page JavaScript with full interactivity</name>
  <files>public/assets/js/pages/audit.js</files>
  <action>
Create `public/assets/js/pages/audit.js` using the IIFE pattern. Use `var` keyword (not const/let) per project conventions. Use `Utils.escapeHtml()` for all user-facing text in innerHTML.

**Demo data:**
- Define `DEMO_EVENTS` array with 25+ sample audit events, each with: `id`, `timestamp` (ISO string), `category` (one of: votes, presences, securite, systeme), `severity` (info|success|danger|warning), `event` (French event description), `user` (name string), `hash` (64-char hex SHA-256), `description` (longer French detail text)
- Include variety: vote cast events, attendance changes, login attempts, system config changes, security alerts

**State variables:**
- `_allEvents`, `_filteredEvents`, `_currentPage`, `_perPage` (15), `_currentView` ('table'), `_activeFilter` (''), `_selectedIds` (Set or array), `_searchQuery`

**Data loading:**
- `loadData()`: try `api('/api/v1/audit.php')`, on success use response data, on failure fall back to DEMO_EVENTS with `console.warn` only (same pattern as hub.js)
- After loading, populate KPI cards: kpiIntegrity (show "100%" or integrity status), kpiEvents (total count), kpiAnomalies (count where severity=danger), kpiLastSession (most recent event date formatted)

**Rendering - Table view:**
- `renderTable(events)`: generate `<tr>` rows for auditTableBody with:
  - Checkbox `<input type="checkbox" class="audit-row-check" data-id="{id}">`
  - Index number
  - Timestamp in monospace: `<span class="audit-timestamp">{formatted}</span>` — use `Shared.formatDate()` or manual formatting (YYYY-MM-DD HH:mm:ss)
  - Event with severity dot: `<div class="audit-event-cell"><span class="audit-severity-dot {severity}"></span>{event text}</div>`
  - User as tag: `<span class="tag tag-accent">{user}</span>`
  - Hash truncated: `<span class="audit-hash-cell" title="{full hash}">{first 12 chars}...</span>`
- Each row gets `data-event-id="{id}"` and click handler to open detail modal (but not on checkbox click)
- Empty state: if no events match filter, show `Shared.emptyState()` in table body

**Rendering - Timeline view:**
- `renderTimeline(events)`: generate `.audit-timeline-item` divs inside `#auditTimeline` with:
  - Severity-colored dot (`.audit-timeline-dot.{severity}`)
  - Content card with title + time on line 1, category tag + user tag + truncated hash on line 2
  - Chevron arrow on right
- Click handler on each item opens detail modal

**Filtering:**
- Filter tabs: click handler on `.filter-tab` buttons — set `_activeFilter`, toggle `.active` class, re-filter and re-render
- `applyFilters()`: filter `_allEvents` by category (if activeFilter not empty) AND by search query (if not empty, match against event/user/description case-insensitive), store in `_filteredEvents`, reset to page 1, render current view

**Search:**
- Debounced input handler on `#auditSearch` (use `Shared.debounce()` with 300ms), calls `applyFilters()`

**Sort:**
- `#auditSort` select with options: "Plus recent" (default, desc timestamp), "Plus ancien" (asc timestamp), "Severite" (danger first)
- On change, sort `_filteredEvents` and re-render

**View toggle:**
- Click handlers on `.view-toggle-btn` buttons: set `_currentView`, toggle `.active` class, show/hide `#auditTableView` / `#auditTimelineView` using `hidden` attribute, re-render

**Pagination:**
- `renderPagination()`: compute total pages from `_filteredEvents.length / _perPage`, render page buttons in `#auditPagination`
- Each page button: `<button class="btn btn-sm {btn-primary if current else btn-ghost}" data-page="{n}">{n}</button>`
- Include Previous/Next buttons (disabled at bounds)
- On click, update `_currentPage`, re-render current view with slice

**Checkbox selection:**
- `#selectAll` checkbox: toggles all visible `.audit-row-check` checkboxes
- Individual checkbox changes: update `_selectedIds`, show/hide `#btnExportSelection` button, update its text to "Exporter la selection ({count})"
- Unchecking selectAll unchecks all

**Event detail modal:**
- `openDetailModal(eventId)`: find event in `_allEvents`, populate modal fields:
  - `#detailTimestamp`: full formatted timestamp
  - `#detailCategory`: category with appropriate tag class
  - `#detailUser`: user name
  - `#detailSeverity`: severity text with colored dot
  - `#detailDescription`: full description text
  - `#detailHash`: complete SHA-256 hash (not truncated)
- Show modal + backdrop (set display/hidden)
- Close: click close button, click backdrop, or Escape key — hide modal + backdrop
- Exporter button in modal footer: trigger download of event as JSON

**Export:**
- `#btnExportAll`: trigger CSV download of all filtered events
- `#btnExportSelection`: trigger CSV download of selected events only
- CSV helper: generate CSV string with headers (Horodatage,Evenement,Categorie,Utilisateur,Severite,Hash), trigger download via Blob + URL.createObjectURL + click on temporary anchor

**Init:**
- On DOMContentLoaded: call `loadData()`, bind all event handlers
