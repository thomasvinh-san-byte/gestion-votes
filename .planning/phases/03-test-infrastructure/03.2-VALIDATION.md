Source : `REQUIREMENTS.md`, décisions D-08 et D-10 du `03-CONTEXT.md`.

## Statut de la validation

**Documentation** : livrée par cette phase (commit `5259732` — `docs(03-2): enrich gsd-code-reviewer with --scope, --timeout-min, --exclude`). Voir `.claude/agents/gsd-code-reviewer.md` section "Arguments supportés" + "Pattern de chunking".

Le doc agent reste **shipped** et utilisable dès qu'invoqué dans Claude Code interactif.

### Étape 3 — Critères de succès attendus

| Critère | Mesure | Seuil |
|---|---|---|
| Pas de timeout | Le sub-agent retourne un REVIEW.md complet | Status `clean` ou `issues_found`, **pas** `timeout` |
| Output complet | REVIEW.md contient `files_reviewed_list` avec ≥ 50 entrées | `wc -l` sur le YAML |
| Chunking activé si > 50 fichiers | Présence de `.planning/.review-progress.json` durant le run, supprimé à la fin | Inspecter timestamps |
| Findings classifiés | Toutes les entrées ont une sévérité (Critical/Warning/Info) | Grep sur le markdown |

## Procédure de fallback (si chunking échoue)

Si le run échoue à mi-chemin :

1. Vérifier `.planning/.review-progress.json` — note les chunks complétés
2. Re-lancer avec `--scope=<sous-périmètre>` plus serré
3. En dernier recours, downgrade à `--depth=quick` pour validation rapide du chemin nominal
