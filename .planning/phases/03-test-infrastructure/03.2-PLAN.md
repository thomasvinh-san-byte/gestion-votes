## Arguments supportés

- `--scope=<value>` (optionnel, défaut `all`) — filtre les fichiers à reviewer
  - `js` : `**/*.js` (sauf `vendor/**`, `node_modules/**`, `dist/**`)
  - `php` : `app/**/*.php` + `public/api/**/*.php`

  - `all` : tous les ci-dessus

## Pattern de chunking (anti-timeout)

Si le scope dépasse l'un de ces seuils :
- > 50 fichiers
- > 500 KB de code total
- > timeout-min × 0.7 (estimation)

L'agent split en 2-3 chunks séquentiels, avec un checkpoint inter-chunk (write progress to `.planning/.review-progress.json`). Si un chunk échoue, les chunks précédents restent committés.

## Exemples

`/gsd:code-review --scope=js --timeout-min=30`
`/gsd:code-review --scope=php --exclude=**/Repository/Traits/**`
```

Le test suite utilise des fixtures auth (admin, operator, voter, assessor).

**Pour bypasser en dev**, exporter `TEST_BYPASS_RATELIMIT=1` :

```

# Explore Patterns — Anti-BEM-substring scan guide

**Quand utiliser quoi pour scanner le codebase sans faux-positifs.**

Source : v2.3 Phase 3 Schoger S-8 (count erroné `shortcut-cards` matchait `shortcut-cards__title` ET `shortcut-cards-grid`, gonflant le total). Ce doc évite la répétition.

## Anti-pattern #1 : BEM substring match

**Symptôme** : `grep "shortcut-cards"` matche aussi `shortcut-cards__title`, `shortcut-cards--variant`, `shortcut-cards-grid`.

**Mauvais** :
```bash
grep -c "shortcut-cards" public/assets/css/*.css  # gonfle le compte
```

**Correct** :
```bash
# Word boundary \b — match `shortcut-cards` SEUL (suivi par espace, virgule, accolade, etc.)
grep -cE '\.shortcut-cards\b' public/assets/css/*.css
# Ou exclure les sub-patterns BEM
grep -cE '\.shortcut-cards($|[^a-zA-Z_-])' public/assets/css/*.css
```

## Anti-pattern #2 : Module name prefix match

**Symptôme** : `grep "import { meeting }"` peut matcher `import { meetingRepo }`.

**Correct** :
```bash
# Délimiter par caractère non-identifier
grep -E 'import \{ meeting[ ,}]' app/**/*.php
```

## Anti-pattern #3 : PHP namespace shadow

**Symptôme** : `grep "namespace AgVote\\Service"` matche `AgVote\Service\Auth\Foo` (préfixe partagé).

**Correct** :
```bash
grep -E 'namespace AgVote\\Service;' app/Services/**/*.php
# ou avec point-virgule terminale, scope-équivalent
```

## Pattern correct générique

Pour `<token>` qu'on veut matcher exactement, en évitant les faux-positifs prefix :

| Contexte | Regex | Test guard |
|---|---|---|
| CSS BEM class | `\\.<token>\\b` | `\\.shortcut-cards\\b` |
| JS identifier | `\\b<token>\\b` | `\\bmeetingRepo\\b` |
| Shell command | `<token>($\|[^a-zA-Z_-])` | `op-tab($\|[^a-zA-Z_-])` |

## Cas d'usage validation v2.4

**Cas** : scan de `.op-tab` (cockpit operator tabs) sans matcher `.op-tab--current` ni `.op-tab-panel`.

```bash
grep -cE '\.op-tab\b' public/assets/css/operator.css
# ✓ Compte exact de .op-tab seul
# ✗ N'inclut pas .op-tab--current, .op-tab-panel, .op-tab__icon
```

Vérification : `grep -cE '\.op-tab--' public/assets/css/operator.css` doit donner 0 dans le résultat précédent.
