# Phase 41.1: Form Fields Modernization - Context

**Gathered:** 2026-03-20
**Status:** Ready for planning

<domain>
## Phase Boundary

Modernize all form field styling across the entire application. Fields must look as polished as the rest of the redesigned pages. Compact, modern inputs with better focus states, inline validation, and professional styling. No new functionality — purely visual/UX upgrade of existing form elements.

</domain>

<decisions>
## Implementation Decisions

### Field Styling
- **Height:** Compact 36px inputs (already set in Phase 31) — verify all pages use it
- **Border:** 1px solid --color-border, border-radius --radius-md (6px). On focus: --color-primary border + double-ring focus (already from Phase 31)
- **Background:** var(--color-surface-raised) — fields should be the brightest layer (white on light theme)
- **Label style:** 13px semibold, --color-text-secondary, uppercase tracking for form group labels. Regular 14px for inline labels
- **Placeholder:** --color-text-muted, italic style for differentiation from entered text
- **Field groups:** Related fields in 2-column grid where appropriate (e.g., first name + last name, date + time). Short fields should not stretch full width
- **Select dropdowns:** Same height/border as inputs, custom chevron icon, consistent padding
- **Textareas:** Auto-resize or fixed height with clear border, same border/focus as inputs
- **Checkboxes/radios:** If not already CSS toggle switches (Phase 40 added toggles to settings), style with colored check/radio indicators

### Inline Validation
- **Error state:** Red border (--color-danger) + red helper text below field. Icon indicator (✗) next to label
- **Success state:** Green border (--color-success) on validated fields. Subtle, not distracting
- **Required indicator:** Small asterisk (*) in --color-danger next to label, with ag-tooltip "Champ obligatoire"

### Field Density
- **Vertical gap:** 16px (--space-field) between field groups — not 24px. Fields should be compact
- **Horizontal gap:** 12px between side-by-side fields in a 2-column grid
- **Section separators:** Subtle divider line or --space-section (48px) gap between form sections, not just stacked fields

### Pages Affected
- All pages with form inputs: wizard, settings, admin, members (import), hub (convocation), operator (proxy), login (already done Phase 35)
- design-system.css base input styles cascade to all pages — most changes go there
- Page-specific overrides where needed for density (wizard fields especially)

### Claude's Discretion
- Whether to use floating labels or above-field labels (recommend: above-field for clarity)
- Exact textarea min-height
- Whether to add character count on limited-length fields
- Radio button group visual treatment

</decisions>

<code_context>
## Existing Code Insights

### Already Done
- Input height 36px, radius 6px, double-ring focus (Phase 31)
- Toggle switches for boolean settings (Phase 40)
- Login field errors with red border (Phase 35)
- ag-tooltip on settings fields (Phase 40)

### What Needs Work
- Many page forms still have generic styling without the modern polish
- Wizard fields stretch full width when they should be 2-column for short fields
- Placeholder text is same weight/color as entered text — hard to distinguish
- No consistent inline validation pattern beyond login
- Select dropdowns look browser-default on some pages
- Textarea styling inconsistent

</code_context>

<specifics>
## Specific Ideas

- Form fields should look like Stripe's checkout fields — clean, compact, professional
- The difference between "filled" and "empty" fields should be immediately obvious
- Error states should be unambiguous — red border + message, not just a toast

</specifics>
