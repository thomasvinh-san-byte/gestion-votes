Tokens spacing — RELIRE avant migration via :
`grep -nE "^\s*--space-" public/assets/css/design-system.css | head -20`

Mapping recommandé (typique d'une échelle 4px-base — à VERROUILLER après lecture réelle des tokens) :
| Valeur brute | Token cible |
|---|---|
| `2px`  | `var(--space-px)` ou `2px` (à garder si pas de token 2px) |
| `4px`  | `var(--space-1)` |
| `8px`  | `var(--space-2)` |
| `12px` | `var(--space-3)` |
| `16px` | `var(--space-4)` |
| `20px` | `var(--space-5)` |
| `24px` | `var(--space-6)` |
| `32px` | `var(--space-7)` |
| `40px` | `var(--space-8)` ou `var(--space-10)` |
| `48px` | (token correspondant ou conserver) |
| `64px+` | (token correspondant ou conserver avec exception) |

Règles de migration :
- `padding: 12px;` → `padding: var(--space-3);`
- `padding: 0 16px;` → `padding: 0 var(--space-4);` (le `0` reste no-token)
- `margin: 16px 8px 12px;` → `margin: var(--space-4) var(--space-2) var(--space-3);`
- `margin: 16px auto;` → `margin: var(--space-4) auto;` (`auto` reste mot-clé)
- `padding-left: 24px;` → `padding-left: var(--space-6);`

Exclusions (ne PAS toucher, ne comptent pas dans les hits) :
- `0` (no-token-needed)
- `auto`, `inherit`, `initial`, `unset`
- valeurs en `%`, `vh`, `vw`, `fr`, `ch`, `ex`
- valeurs fractionnaires (`0.5rem`, `1.5em`) — ces unités n'apparaissent pas dans le grep cible
- bordures `1px`, `2px`, `3px` dans `border:` (couvertes par `--border-*` tokens v2.6)
- valeurs hors-grille `7px`, `13px`, `27px` etc. : DÉCISION par site —
  soit (a) arrondir à la valeur de grille la plus proche et tokeniser (perte mineure
  de fidélité visuelle), soit (b) garder la valeur brute et la documenter comme exception
  dans v2.7-TOKENS-FINAL.md. Préférence : option (a) si la différence ≤2px et le contexte
  le permet ; option (b) si valeur d'alignement précise (ex : centrage d'icône 13px).
