## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Server SSE event → DOM attribute | `quorum.updated` and `attendance.updated` payloads flow into `setAttribute()` calls on `<ag-health-bar>`. Numeric coercion via `Number(...)` and ratio strings reach the user as text. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-07 | Tampering | SSE payload → setAttribute | mitigate | All numeric fields coerced via `Number()` and bounded with `isFinite()` before division/comparison. The motion-title is HTML-escaped inside ag-health-bar render() (Plan 01.1 Task 1 step 6). |
| T-01-08 | Information Disclosure | health-bar visible to operator role | accept | The bar exposes only data the operator already has access to (quorum, votes count). No new disclosure. |
| T-01-09 | Denial of Service | SSE flood → render storm | accept | `attributeChangedCallback` re-renders only when `oldValue !== newValue` (Plan 01.1 Task 1 step 3); duplicate events are no-ops. SSE reconnect-and-replay storms therefore don't multiply renders. |
</threat_model>
