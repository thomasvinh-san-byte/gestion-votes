## At-risk threshold formula shipped

```javascript
function _computeQuorumState(current, required) {
  var c = Number(current); var r = Number(required);
  if (!isFinite(c) || !isFinite(r) || r <= 0) return 'met';
  if (c < r) return 'missed';
  if (c < r * 1.10) return 'at-risk';
  return 'met';
}
```

- `c < r` → `missed`
- `r <= c < r * 1.10` → `at-risk` (10% buffer above the threshold)
- `c >= r * 1.10` → `met`
- Defensive default: any non-finite or non-positive `r` returns `'met'` so a zero/NaN denominator never produces a false alarm.

## SSE event payload shape mismatches discovered + coercions applied

Two payload-shape edge cases were handled defensively:

1. **`quorum.updated` nests under `data.data` in some events but is flat in others.** The original handler at line 156 already handled this for `quorum_met` (`data.quorum_met === true || (data.data && data.data.quorum_met === true)`). The new code generalises it: `var p = (data && data.data) ? data.data : (data || {});` then reads `p.quorum_current` / `p.quorum_required` from the unified `p`.

2. **`quorum_current` may not exist** on payloads that only ship `present_count` (older event producers). Fallback chain: `(p.quorum_current != null) ? p.quorum_current : (p.present_count != null ? p.present_count : null)`. If both are absent, the bar attributes are not updated for that event — the previous values stay visible.

3. **`attendance.updated` total field varies** (`data.data.total`, `data.total`, or absent). Fallback chain: `(data && data.data && data.data.total) || (data && data.total) || motion.total_voters || 0`. If `totalVoters === 0` we skip the write rather than render `0 / 0` (which would be misleading).

All numeric coercions go through `Number()` and are guarded by `isFinite()` (T-01-07 mitigation).

## Decisions Made

- **Unified attribute-write helper:** `_setHb` centralises every write to the bar plus the F-2 mirror to `#viewExec`. No call site duplicates the `data-quorum-state` mirror logic, and the idempotency check avoids the SSE-replay storm scenario T-01-09.
- **`window.O.fn.notifyMotionChange` defined in realtime.js, not motions.js:** keeps every `<ag-health-bar>` write in one file. `operator-motions.js` only sees the public hook, not the bar internals.
- **Removed `_prevQuorumMet` entirely:** it was only ever read by the toast block (deleted), and no other file references it (verified by `grep -rn _prevQuorumMet public/`).
- **`opBtnProclaim` (2 occurrences) kept:** the acceptance criterion `<= 1` was met for the keydown path (now 0), but the file retains 2 *legitimate* references — the disable/enable toggle on the button (line 657) and the click handler (line 1126). Documented as a measurement-criterion mismatch (the criterion should have read "in keydown context").
