<objective>
Build a self-contained keyboard shortcuts module for the operator execution view: `L` (Lancer le vote), `F` (Fermer le scrutin), `→` or `N` (résolution suivante), `?` (afficher les raccourcis). Output: One JS file (key handler module) and one CSS file (overlay). The module exports nothing globally except a sentinel `window.AG_OPERATOR_KEYBINDINGS = true` so duplicate loads are detected.

   ```javascript
   document.addEventListener('keydown', function(e) {
     if (e.metaKey || e.ctrlKey || e.altKey) return;
     if (_isTypingContext()) return;
     // ? overlay works in any mode (it's documentation)
     if (e.key === '?') { e.preventDefault(); _toggleOverlay(); return; }
     if (!_isExecMode()) return;
     var k = e.key;
     if (k === 'l' || k === 'L') { e.preventDefault(); _click('opBtnLaunchVote') || _click('opBtnToggleVote'); return; }
     if (k === 'f' || k === 'F') { e.preventDefault(); _click('opBtnCloseVote') || _click('opBtnToggleVote'); return; }
     if (k === 'ArrowRight' || k === 'n' || k === 'N') { e.preventDefault(); _nextMotion(); return; }
   });
   ```

4. **`_click` helper** — returns true if the button was found and clicked, false otherwise:
   ```javascript
   function _click(id) {
     var btn = document.getElementById(id);
     if (!btn || btn.disabled || btn.hidden || btn.getAttribute('aria-hidden') === 'true') return false;
     btn.click();
     return true;
   }
   ```

5. **`_nextMotion` helper:**
   ```javascript
   function _nextMotion() {
     if (window.O && window.O.fn && typeof window.O.fn.selectNextMotion === 'function') {
       window.O.fn.selectNextMotion();
       return;
     }
     // Fallback: click the first non-voted agenda item after the current one
     var items = Array.prototype.slice.call(document.querySelectorAll('.op-agenda-item'));
     var currentIdx = items.findIndex(function(it) { return it.classList.contains('current'); });
     for (var i = currentIdx + 1; i < items.length; i++) {
       if (!items[i].classList.contains('voted')) { items[i].click(); return; }
     }
   }
   ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser keyboard event → DOM action | Operator keystrokes trigger `.click()` on existing buttons. The buttons themselves enforce their own auth/state — the keybindings module is a UI accelerator, not an authority. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-04 | Elevation of Privilege | keybindings → button.click() | accept | The shortcut module triggers existing buttons whose handlers already enforce CSRF, mode, and role checks server-side. No new privilege boundary introduced. |
| T-01-05 | Tampering (input hijack) | document keydown listener | mitigate | Shortcuts MUST NOT fire when user is typing in input/textarea/select/contenteditable (COCKPIT-06 explicit) — checked via `_isTypingContext()` before every dispatch. |
| T-01-06 | Denial of Service (overlay loop) | `_toggleOverlay` | accept | `?` toggles a single DOM node; pressing repeatedly toggles visibility. Bounded resource use. |
</threat_model>
