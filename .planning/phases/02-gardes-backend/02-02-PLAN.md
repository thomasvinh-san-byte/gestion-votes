<task type="auto">
  <name>Task 2: Make workflow transitions idempotent (IDEM-05)</name>
  <files>app/Services/MeetingTransitionService.php, app/Controller/MeetingWorkflowController.php</files>
  <action>
Per decision D-06 and D-07: when a meeting is already in the target state, return success without side effects instead of throwing an error.

**MeetingTransitionService::transition()** (line ~47):
Change the `$fromStatus === $toStatus` check from throwing a RuntimeException to returning an idempotent result:
```php
if ($fromStatus === $toStatus) {
    return [
        'meeting_id' => $meetingId,
        'from_status' => $fromStatus,
        'to_status' => $toStatus,
        'already_in_target' => true,
    ];
}
```

**MeetingTransitionService::launch()** (line ~80):
Change the `'live'` match arm from throwing to returning idempotent result:
```php
'live' => [
    'meeting_id' => $meetingId,
    'from_status' => 'live',
    'to_status' => 'live',
    'path' => [],
    'already_in_target' => true,
],
```
And handle the result after the match: if `$path` is already a result array (has `already_in_target`), return it directly instead of proceeding with the launch path. Restructure: after the match, check if result has `already_in_target` key and return early.

Actually, cleaner approach: keep the match returning path arrays only. Before the match, check if already live:
```php
if ($fromStatus === 'live') {
    return [
        'meeting_id' => $meetingId,
        'from_status' => 'live',
        'to_status' => 'live',
        'path' => [],
        'already_in_target' => true,
    ];
}
```
Then the match `'live'` arm is unreachable and can be removed.

**MeetingWorkflowController::transition()** (line ~20):
After `$preCheck = $service->transition(...)`, check for the idempotent case:
```php
if (!empty($preCheck['already_in_target'])) {
    api_ok([
        'meeting_id' => $meetingId,
        'from_status' => $preCheck['from_status'],
        'to_status' => $toStatus,
        'transitioned_at' => date('c'),
        'already_in_target' => true,
        'warnings' => [],
        'results_emails' => 0,
    ]);
}
```
This returns BEFORE the transaction, auth check, or SSE broadcast -- no side effects.

Remove or adjust the catch block that converts "deja au statut" to `api_fail` (line ~38) since it's now unreachable for same-status transitions. Keep it as a safety net but it should no longer trigger.

**MeetingWorkflowController::launch()** (line ~84):
After `$preCheck = $service->launch(...)`, check for the idempotent case:
```php
if (!empty($preCheck['already_in_target'])) {
    api_ok([
        'meeting_id' => $meetingId,
        'from_status' => 'live',
        'to_status' => 'live',
        'already_in_target' => true,
    ]);
}
```
Remove or adjust the catch for "deja en cours" (line ~95).

Also update the lockForUpdate check inside the transaction in `transition()` (line ~60): change the `$lockedFrom === $toStatus` check from `api_fail` to return early with success:
```php
if ($lockedFrom === $toStatus) {
    return ['already_in_target' => true, 'from_status' => $lockedFrom];
}
```
Then after the transaction, check if the result indicates `already_in_target` and return success.
