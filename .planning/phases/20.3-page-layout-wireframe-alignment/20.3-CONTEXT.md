# Phase 20.3: Page Layout Wireframe Alignment - Context

**Gathered:** 2026-03-17
**Status:** Ready for planning

<domain>
## Phase Boundary

Reprendre la disposition (layout/grille) de TOUTES les pages pour correspondre au wireframe v3.19.2 — densité, placement des sections, espacement entre blocs. L'objectif est que les pages principales tiennent sur un écran 1920×1080 sans scroll inutile. Les éléments ajoutés après le wireframe sont conservés mais intégrés dans la grille wireframe. Cette phase touche HTML + CSS (pas CSS-only).

Hors scope :
- Tokens/composants (déjà alignés en 20.1 + 20.2)
- JavaScript / logique métier
- Nouvelles fonctionnalités

</domain>

<decisions>
## Implementation Decisions

### Contrainte de résolution
- Référence : **1920×1080** (Full HD) — les pages "cockpit" doivent tenir sans scroll à cette résolution
- Les pages à données longues (tableaux avec beaucoup de lignes) : Claude évalue au cas par cas le meilleur pattern UX (pagination, scroll interne, ou scroll naturel)

### Dashboard (zéro scroll)
- **Tout visible d'un coup** à 1080px : banner urgente + KPIs grid-4 + grid-2 (séances/tâches) + grid-3 raccourcis
- Réduire paddings/margins si nécessaire pour tout faire rentrer
- C'est un lanceur d'actions, pas un tableau de bord dense

### Console opérateur (split panel)
- **Reprendre le split panel wireframe** : résolution + résultats à gauche (flex: 1), présence/file à droite (~300px)
- Éléments compacts et normalisés, pas étendus sur toute la largeur
- KPI strip + quorum + progress track en haut (flex-shrink: 0)
- Zéro scroll sur la page — les panels internes scrollent si nécessaire

### Grille wireframe
- Approche **wireframe + ajustements** : base wireframe pour la disposition, adapter quand le contenu réel diffère (nombre de KPIs, sections additionnelles)
- grid-N adapté au contenu réel (grid-4 si 4 éléments, grid-3 si 3, etc.)

### Paramètres
- **Reprendre le layout wireframe** : sidebar nav verticale à gauche (200px) + contenu tabbé à droite
- Gap 14px entre nav et contenu

### Hub (fiche séance)
- **Reprendre le layout wireframe** : stepper vertical à gauche (220px) + action card + checklist à droite
- Gap 24px entre colonnes

### Tableaux (toutes pages)
- **Compacter comme wireframe** : 8px padding vertical sur les lignes
- Plus de données visibles sans scroll
- S'applique à : membres, séances, audit, archives, utilisateurs

### Wizard (création de séance)
- **Reprendre le layout wireframe** : progress bar horizontale + carte unique de contenu + footer nav fixe en bas
- Chaque étape doit tenir sans scroll

### Densité et espacements
- Claude ajuste page par page en visant "tient dans 1080px" comme contrainte principale
- Le wireframe utilise ~14px card padding, 10-14px section margins — l'implémentation actuelle est à ~20px
- Claude réduit là où c'est nécessaire pour respecter la contrainte 1080px

### Page headers
- Claude évalue l'impact page par page et compacte si nécessaire pour gagner de la hauteur

### Statistiques
- Claude optimise la densité des graphiques selon l'espace disponible en 1080px

### Toutes les pages
- Passage systématique de TOUTES les pages, pas seulement les plus problématiques
- Chaque page est comparée à son équivalent wireframe et ajustée

### Claude's Discretion
- Espacement exact (14px vs 16px vs 12px) page par page selon la contrainte 1080px
- Page headers : compacter ou garder selon l'espace restant
- Statistiques : taille des graphiques et disposition optimale
- Pages à données longues : choix du pattern (pagination / scroll interne / scroll naturel) selon le contexte UX
- `.app-main` padding : réduire ou garder selon les besoins

</decisions>

<specifics>
## Specific Ideas

- Le wireframe v3.19.2 (`ag_vote_wireframe.html`) est la référence pour la composition de chaque page
- L'opérateur doit passer de onglets à split panel — c'est un changement HTML significatif
- Le dashboard doit montrer TOUT le contenu sans scroller — c'est la contrainte la plus forte
- Les tableaux en 8px padding vertical gagneront ~40% de lignes visibles
- Le hub et les paramètres doivent avoir leurs sidebars respectives (220px et 200px)
- Le wizard doit avoir un footer nav fixe pour que chaque étape tienne dans le viewport

</specifics>
