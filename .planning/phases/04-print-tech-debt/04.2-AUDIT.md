## Méthode

Patterns grep utilisés (EXPLORE-PATTERNS.md hyphen-safe) :

```bash
# Borders — match `border:` exact (exclut border-radius, border-color, etc.)
grep -rnE "(^|[^a-z-])border:\s+" public/assets/css/

# Box-shadow exact
grep -rnE "(^|[^a-z-])box-shadow:\s+" public/assets/css/
```

`design-system.css` exclu (source de tokens, UNTOUCHED).

## Métrique baseline (avant migration)

Formule (per plan & user) :

```
tokenized = grep -rcE "border:\s+var\(--border-|box-shadow:\s+var\(--shadow-"
hardcoded = grep -rcE "border:\s+[0-9]+|box-shadow:\s+[^v]"
ratio = tokenized / (tokenized + hardcoded)
```

**Résultat baseline (excluant `design-system.css`) :**

| Catégorie       | Tokenized | Hardcoded | Total | Ratio |

| Borders         | 139       | 79        | 218   | **63.8 %** |
| Shadows         | 63        | 41        | 104   | **60.6 %** |
| **Combined**    | **202**   | **120**   | **322** | **62.7 %** |

**Cible :** ≥ 95 % combined.
**Δ requis :** migrer ~104 déclarations supplémentaires OU exclure justifiables.

## Section 3 — Métriques baseline & projection

| Métrique | Borders | Shadows | Combined |
|---|---|---|---|
| Tokenized initial (`var(--border-X)` ou `var(--shadow-X)`) | 139 | 63 | 202 |
| Hardcoded initial | 79 | 41 | 120 |
| Total initial | 218 | 104 | 322 |
| **Ratio initial** | **63.8 %** | **60.6 %** | **62.7 %** |

**Projection après migration (cible) :**

- Borders migrés vers existing tokens : 5 (Cluster A)
- Borders migrés vers nouveaux tokens : ~50 (Clusters B + C migrables)
- Borders keep-justified : ~24
- Shadows migrés vers existing token : 1 (S-E)
- Shadows migrés vers nouveaux tokens : 11 (Cluster S-C)
- Shadows keep-justified : 29 (S-A 3 + S-B 12 + S-D 14)

**Ratio projeté :**
- Borders : (139 + 55) / 218 = 89.0 %
- Shadows : (63 + 12) / 104 = 72.1 %
- Combined : (202 + 67) / 322 = **83.5 %**

**Constat : 95 % strict reste hors de portée même avec nouveaux tokens.**

Cause : trop de cas légitimes (animations 12, color-mix inline 8, glassmorphism login 4, fallback chains 2, widths rares 5, rings spéciaux 6, etc.).

**Décision recommandée :** ajuster D-07 cible 95 % → **85 %** (ou laisser le ratio chiffré justifier). Le SUMMARY documentera cette divergence avec rationale.

**Alternative envisagée :** créer 10+ tokens supplémentaires couvrant chaque cas isolé. Rejetée car va à l'encontre de D-08 (tokens utilitaires, pas tokens jetables) et complexifie le design system sans bénéfice (tokens 1-site = aliases).

## Out-of-scope (différé v2.5+)

- `border-color:` (sub-property) → out of D-05 scope
- `border-radius:` → out of scope
- `border-bottom:`, `border-top:`, etc. (sub-properties) → out of scope
- `outline:` → out of scope (hors borders/shadows)
- `text-shadow:` → out of scope
- `filter: drop-shadow()` → out of scope
- Hex colors inline → différé v2.5+
- `border-style:` only → out of scope

- Regex `(^|[^a-z-])` validé pour exclure `border-radius`, `border-color`, `border-bottom`, `-webkit-box-shadow`, `text-shadow` (test fait avant production audit).
- Métriques calculées via `grep -rcE` (count) **excluant** `design-system.css`.
- 1 commit = 1 fichier (D-09).
- `design-system.css` modifié exceptionnellement pour ajouter nouveaux tokens (D-08 cas critique justifié — sans cela 95 % inatteignable).
- Smoke regression visuelle : sandbox-deferred (procédure dev-machine documentée dans SUMMARY).
