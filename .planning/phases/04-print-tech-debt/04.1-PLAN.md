# Plan 04.1 — dompdf header/footer

## Goal

Tous les exports PDF générés par dompdf (PV, procurations, rapports) affichent un **header répété** (titre séance + date) sur chaque page et un **footer** (`Page X sur Y`) via CSS `@page` rules. Vérifié sur 3 PVs longs (≥10 pages).

Source : TECH-V24-01 (REQUIREMENTS.md), CONTEXT.md décisions D-02..D-04.

## Acceptance Criteria

1. **Audit PDF templates** livré dans `04.1-AUDIT.md` : enumération de tous les services qui appellent dompdf + identification du HTML/CSS injecté. Templates listés avec file:line.
2. **CSS `@page` rules** ajoutées à chaque template HTML PDF :
   - `@page { @top-center { content: "[séance titre] — [date FR]" } @bottom-center { content: "Page " counter(page) " sur " counter(pages) } }`
   - Variables session/date interpolées via PHP avant émission HTML
3. **Header français** : format `[Titre séance] — [JJ/MM/YYYY]` (ou équivalent fixé par audit en français correct).
4. **Footer français** : `Page X sur Y` (ou `Page X / Y`). Counters dompdf natifs.
5. **3 PVs longs testés** : génération sur dev-machine (sandbox-deferred si pas praticable). Capture/inspection visuelle 1ère + 5ème + dernière page → header présent + footer paginé.
6. **Régression nulle** sur PDF existants (procuration, PV court) — restent générables sans erreur. Test PHPUnit existant si présent reste vert.
7. **Page break preservation** : `page-break-inside: avoid` existant sur blocs résolution/hash (livré v2.3 P2 EDITORIAL-07) reste fonctionnel — pas écrasé par les @page rules.

## Task Breakdown

### Task 1 — Audit PDF templates

```bash
grep -rn "Dompdf\\|dompdf\\|new Dompdf" app/Services/ | head -20
grep -rn "renderHtml\\|getOutput\\|loadHtml" app/Services/ | head -20
```

Pour chaque service identifié, dans `04.1-AUDIT.md` :
- File:line
- Templates HTML utilisés (chemin si externe, ou inline si construit en string)
- CSS actuel injecté (style block existant ?)

Cible attendue : 3-4 services PDF (`ProcurationPdfService`, `MeetingReportsService`, `MeetingReportService`, `ReportGenerator`).

### Task 2 — Identifier point d'injection CSS

Pour chaque service, déterminer le point d'injection :
- Option A : style block dans le template HTML existant → ajouter @page rules dedans
- Option B : pas de style block → créer un `<style>` au début du `<head>` de chaque template
- Option C : CSS file externe → ajouter @page rules dans ce fichier

Documenter le choix dans audit + appliquer.

### Task 3 — Implémenter @page rules

Pour chaque template, ajouter :

```css
@page {
  margin-top: 3cm;
  margin-bottom: 2cm;

  @top-center {
    content: "[VARIABLE_HEADER]";
    font-family: 'DejaVu Sans', sans-serif;
    font-size: 10pt;
    color: #666;
  }

  @bottom-center {
    content: "Page " counter(page) " sur " counter(pages);
    font-family: 'DejaVu Sans', sans-serif;
    font-size: 9pt;
    color: #888;
  }
}

/* Preserve existing page-break rules */
.resolution, .hash-block { page-break-inside: avoid; }
```

`[VARIABLE_HEADER]` interpolé en PHP avant émission HTML :

```php
$headerText = sprintf('%s — %s', 
    htmlspecialchars($meetingTitle, ENT_QUOTES, 'UTF-8'),
    $meetingDate->format('d/m/Y')
);
$css = str_replace('[VARIABLE_HEADER]', $headerText, $cssTemplate);
```

### Task 4 — Test syntax + smoke

`php -l` chaque service modifié.

Si test PHPUnit existant pour ProcurationPdfService :

```bash
timeout 60 php vendor/bin/phpunit tests/Unit/ProcurationPdfServiceTest.php --no-coverage
```

Vérifier qu'il reste vert (assertion sur HTML output ne doit pas casser).

### Task 5 — Capture validation (sandbox-deferred si applicable)

2. Générer 1 PV de séance longue (≥10 pages) → header répété sur chaque page, footer "Page X sur Y" évolue

## Risks & Mitigations

| Risk | P | I | Mitigation |
|---|---|---|---|
| dompdf 3.1 ne supporte pas counter(page) correctement | L | H | Test smoke 1 PV avant migration tous services. Fallback : `dompdf::page_text()` API si CSS échoue (D-CONTEXT discrétion). |
| @page rules écrasent `page-break-inside` existant | L | M | Préserver explicitly les règles existantes dans le même bloc CSS. Test régression. |
| Header text overflow long titres séance | M | L | `text-overflow: ellipsis` ou troncature PHP pré-injection. Documenter limite (~80 chars). |
| Encoding UTF-8 cassé (accents français dans header) | M | M | `htmlspecialchars(..., ENT_QUOTES, 'UTF-8')` + dompdf font supporte UTF-8 (DejaVu Sans built-in). |
