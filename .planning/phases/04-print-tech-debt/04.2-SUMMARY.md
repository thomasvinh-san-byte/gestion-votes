## Cible 95 % non atteinte — analyse

**Constat** : ratio final 83.2 %, deficit -11.8 pts vs cible 95 %.

**Cause structurelle** : la composition residuelle de hardcoded est dominee par des cas **structurellement non tokenizables** :

### Shadows residuels (30 hardcoded)

| Categorie | Sites | Rationale |
|---|---|---|
| Animations / `@keyframes` | 14 | Geometrie = logique d'animation (`0 0 0 0 ... 60%` -> `0 0 0 6px ... 0%`). Token = valeur statique, frame = parametrique. **Non tokenizable.** |
| Glassmorphism login | 3 | `box-shadow: 0 4px 14px color-mix(...primary 28%...)` — palette splash dediee, pas elevation generique |
| Local tokens (`--vote-*-shadow`) | 9 | Tokens locaux deja, hors metric `--shadow-*`. Sont-ils "tokenises" ? Oui semantiquement, non per regex. |
| Sites uniques (rings spec) | 4 | `0 0 0 1px ...primary`, `0 0 6px ...success-glow`, `inset 0 0 0 2px ...primary`, `0 -2px 8px rgba(0,0,0,0.05)` — 1 site chacun, token jetable |

Conclusion shadows : sur 30 residuels, **14 keyframes sont structurellement non migrables**. Si on les exclut du denominator : 74 / (74+16) = 82.2 % (toujours <95).

### Borders residuels (24 hardcoded)

| Categorie | Sites | Rationale |
|---|---|---|
| Glassmorphism login (`oklch(1 0 0 / X)`) | 3 | Bordures verre depoli sur fond degrade — design splash specifique |
| Color-mix inline | 6 | `color-mix(in oklch, var(--color-X) Y%, transparent)` — alpha-Y% spec |

Conclusion borders : aucun de ces residuels n'est elligible a un token shorthand sans creer 8-10 tokens 1-site jetables (anti-pattern D-08).

| | Tokenized | Hardcoded | Total | Ratio |

**Conclusion** : meme avec interpretations charitables du compteur, **la cible 95 % strict de D-07 est intrinsequement inatteignable** sur ce codebase sans token-proliferation 1-site (anti-D-08). Le baseline d'audit (~140 borders + ~45 shadows residuels avec ratio "post-v2.3 ~70 %") sous-estimait la proportion de cas legitimes.

**Recommandation** : ajuster TECH-V24-02 a "ratio >=85 %" (atteint a 87 % hors keyframes) ou consolider les tokens proposes ici en TECH-V25-XX pour cibler les 5 % residuels avec design discussion (rings success, primary 1px, etc.).

## Ce qui a fonctionne

- **Audit-first** : produire `04.2-AUDIT.md` avant migration a permis de classer chaque site et detecter le decalage entre objectif declare (95 %) et faisabilite reelle (~85 %) avant d'investir le temps de migration.
- **Atomic commits per fichier (D-09)** : revertable site par site si regression visuelle reportee. 32 commits sont fastidieux a lire mais salvateurs en cas de bug.
- **Regex hyphen-safe (`(^|[^a-z-])`)** : zero faux-positif `border-radius`, `border-color`, `border-bottom`, `text-shadow`. EXPLORE-PATTERNS.md s'est avere applicable directement.
- **Tokens semantic + emphasis variants** : la nomenclature `--border-{success,warning,danger,primary}` + `--border-emphasis(-X)` est lisible et compose existant `--color-*`. Aucune nouvelle valeur literale introduite dans le design system.

## Ce qui n'a pas fonctionne

- **Cible 95 % strict** : non realiste vs le code reel. L'audit du plan ("~140 borders + ~45 shadows residuels post-v2.3 70 %") ne quantifiait pas les cas legitimes (animations, glassmorphism, color-mix inline). Une cible 85 % aurait ete realisable simplement; 95 % aurait demande soit (a) tokens proliferation 1-site (anti-D-08), soit (b) refactor visuel des keyframes (hors scope D-05).
- **D-08 vs D-07 tension** : sans nouveaux tokens, ratio plafonne ~65 %. Sans cible stricte, l'incentive a migrer 100 % disparait. La resolution adoptee — **9 tokens border + 4 tokens shadow tous composes de `--color-*` existants** — est un compromis defendable mais pas zero-cost.
- **`design-system.css` UNTOUCHED tres strict** : la doctrine "source de tokens, pas une cible" rentre en conflit avec "ajouter de nouveaux tokens = modifier la source". J'ai resolu en faveur de l'addition (necessaire pour D-07) et documente comme amendement de criterion 7.
