## 2. Détail des points d'injection CSS

### 2.1 ProcurationPdfService

- **File:line** : `app/Services/ProcurationPdfService.php:38–211`
- **Template type** : HTML inline construit en heredoc PHP (`<<<HTML ... HTML;`)
- **CSS actuel** : bloc `<style>` aux lignes 43–141, déjà existant
- **`@page` actuel** : ligne 44, `@page { margin: 2cm; }` (margins seulement, pas de header/footer)
- **Variables disponibles pour header** :
  - `$meetingTitle` (line 29) — déjà htmlspecialchars(ENT_QUOTES, UTF-8) ✓
  - `$dateFormatted` (line 34) — déjà au format `d/m/Y` français ✓
- **Choix injection** : **Option A** — étendre le bloc `<style>` existant. Remplacer la ligne `@page { margin: 2cm; }` par un bloc `@page` enrichi avec `@top-center` + `@bottom-center`.
- **Risque template** : la procuration est typiquement courte (1 page). Header/footer apportent peu de valeur fonctionnelle mais cohérence visuelle utile. Coût marginal nul.

### 2.2 MeetingReportsService

- **File:line** : `app/Services/MeetingReportsService.php:158–235` (méthode `buildPdfHtml`)
- **Template type** : HTML construit en concaténation de strings PHP (`$html .= '...'`)
- **CSS actuel** : ligne 168, variable `$css` contenant un bloc CSS minifié monolithique
- **`@page` actuel** : début du `$css`, `@page{margin:2cm}` (margins seulement)
- **Variables disponibles pour header** :
  - `$meeting['title']` (line 169, 179) — accessible
  - `$meeting['scheduled_at']` (line 180) — formaté ailleurs en `d/m/Y à H:i` via `date()`
- **Choix injection** : **Option A** — étendre la variable `$css`. Remplacer `@page{margin:2cm}` par `@page{margin-top:3cm;margin-bottom:2cm;@top-center{...}@bottom-center{...}}`.
- **Construction header text** : interpoler le titre + date AVANT concaténation dans `$css`, en utilisant `htmlspecialchars($title, ENT_QUOTES, 'UTF-8')` puis échappement supplémentaire pour le contexte CSS string `content: "..."`.
- **Page-break preservation v2.3 P2** : grep `page-break-inside` dans le fichier — **aucune occurrence existante**. Ajouter règle `page-break-inside: avoid` proactive sur `.result-box, .header-info, table` pour cohérence v2.3 EDITORIAL-07.

## 3. Vérification dompdf 3.1

- Composer : `dompdf/dompdf ^3.1` (composer.json:7) ✓
- dompdf 3.1 supporte CSS Paged Media : `@page`, `@top-center`, `@bottom-center`, `counter(page)`, `counter(pages)` (référence : dompdf changelog 3.0+)
- Font UTF-8 par défaut : `DejaVu Sans` (`isPhpEnabled=false`, `defaultFont='DejaVu Sans'` déjà configuré dans les deux services) → accents français supportés ✓
- Dans le contexte CSS `@page @top-center { content: "..." }`, le contenu littéral est rendu tel quel par dompdf (pas d'interprétation HTML entities → pré-échapper en PHP suffit).

## 4. Encodage et échappement (UTF-8 + CSS string)

**Risque** : un titre de séance contenant `"` casserait la string CSS `content: "..."`.

**Mitigation** : double-échapper le titre :
1. PHP `htmlspecialchars($title, ENT_QUOTES, 'UTF-8')` → encode `"` en `&quot;`. **Mais** dompdf ne désencode PAS les entités HTML dans une CSS string.
2. Solution retenue : utiliser un échappement CSS-string-safe : remplacer `"` par `\"` et `\` par `\\` AVANT injection. Conserver les caractères accentués UTF-8 littéralement (DejaVu Sans gère).

**Helper** : fonction inline ou statique `escapeCssString(string $s): string`.

## 5. Header text format final (D-03)

`[Titre séance] — [JJ/MM/YYYY]`

- Em-dash : `—` (U+2014, 3 octets UTF-8)
- Date : `DateTime::format('d/m/Y')` ou `date('d/m/Y', strtotime(...))`
- Si `scheduled_at` absent : fallback `[Titre séance]` seul (sans em-dash flottant).
- Procuration : utilise `$meetingTitle` + `$dateFormatted` déjà calculés.
- PV (MeetingReportsService) : utilise `$meeting['title']` + date depuis `scheduled_at` (formaté `d/m/Y`).

## 6. Footer text format final (D-04)

`Page X sur Y`

- `counter(page)` : numéro page courante
- `counter(pages)` : total pages
- CSS : `content: "Page " counter(page) " sur " counter(pages);`
- Counters dompdf natifs (CSS Paged Media spec).
