Grid properties on a flex container are silently ignored. The fix is to explicitly add `display: grid`.

1. **Fix the grid bug** — update the existing `[data-page-role="operator"] .app-shell` rule in operator.css (line ~14). Use the **nested grid approach** from research (cleaner because the fixed sidebar stays out of the grid):
   ```css
   [data-page-role="operator"] .app-shell {
     display: grid;
     grid-template-rows: auto auto 1fr;
     grid-template-columns: 1fr;
     grid-template-areas:
       "statusbar"
       "tabnav"
       "main";
     height: 100vh;
     overflow: hidden;
   }
   ```
   The `.app-sidebar` is `position: fixed` and does NOT participate in grid flow — this is correct.

2. **Apply grid areas to existing elements**:
   ```css
   [data-page-role="operator"] .meeting-bar {
     grid-area: statusbar;
     background: var(--color-surface-raised);
   }

   [data-page-role="operator"] .tabs-nav {
     grid-area: tabnav;
   }

   [data-page-role="operator"] .app-main {
     grid-area: main;
     display: grid;
     grid-template-columns: 280px 1fr;
     padding: 0;
     overflow: hidden;
   }
   ```
