  <action>
    Read every .sql file in database/migrations/ and search for these SQLite-specific patterns:

    1. `AUTOINCREMENT` (PostgreSQL uses SERIAL or GENERATED ALWAYS AS IDENTITY)
    2. `datetime('now')` (PostgreSQL uses `now()` or `CURRENT_TIMESTAMP`)
    3. `INTEGER PRIMARY KEY` without GENERATED (SQLite auto-rowid pattern)
    4. `BOOLEAN` used as column type without proper PostgreSQL boolean syntax
    5. `TEXT` used where PostgreSQL would use `varchar` (acceptable but flag for review)
    6. `PRAGMA` statements (SQLite-only)
    7. Backtick quoting `` ` `` instead of double-quote `"` for identifiers

    For each file, run:
    ```bash
    grep -nE 'AUTOINCREMENT|datetime\(.now.\)|PRAGMA|`[a-z]' database/migrations/*.sql
    ```

    If any matches are found, fix them:
    - `AUTOINCREMENT` → remove (PostgreSQL SERIAL handles this)
    - `datetime('now')` → `now()` or `CURRENT_TIMESTAMP`
    - `INTEGER PRIMARY KEY AUTOINCREMENT` → `SERIAL PRIMARY KEY` or `INTEGER GENERATED ALWAYS AS IDENTITY`
    - `PRAGMA` → remove entirely
    - Backtick quotes → double quotes

    Known state: Prior audit found zero matches for AUTOINCREMENT, datetime('now'), INTEGER PRIMARY KEY patterns. This task is a formal verification pass.


       ```bash

           echo "FAIL: $(basename "$f")"

         fi
       done
       ```

       ```bash
       echo "=== Pass 2: Idempotency test ==="
       for f in database/migrations/*.sql; do
         psql -v ON_ERROR_STOP=1 -h "$HOST" -p "$PORT" -U "$USER" -d "$TESTDB" -f "$f" 2>&1

       ```
