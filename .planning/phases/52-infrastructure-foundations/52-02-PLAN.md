  <action>
    **A. Fix Dockerfile HEALTHCHECK (DOC-01):**

    The current HEALTHCHECK on line 90-91 is:
    ```dockerfile
    HEALTHCHECK --interval=30s --timeout=5s --start-period=60s --retries=3 \
        CMD curl -sf http://127.0.0.1:${PORT:-8080}/api/v1/health.php || exit 1
    ```

    The bug: Docker evaluates `${PORT:-8080}` at BUILD time (Dockerfile instruction), not at container runtime. The `CMD` form in HEALTHCHECK does NOT run through a shell, so the variable is baked into the image.

    Replace with a shell-form healthcheck that evaluates at runtime:
    ```dockerfile
    HEALTHCHECK --interval=30s --timeout=5s --start-period=60s --retries=3 \
        CMD sh -c 'curl -sf http://127.0.0.1:${PORT:-8080}/api/v1/health.php || exit 1'
    ```

    The `sh -c '...'` wrapper ensures the shell expands `$PORT` at runtime from the container's environment.

    **B. Create nginx.conf.template (DOC-02):**

    Copy deploy/nginx.conf to deploy/nginx.conf.template and replace the hardcoded port:
    - Line with `listen 8080 default_server;` → `listen ${LISTEN_PORT} default_server;`

    Everything else in the template stays identical to the current nginx.conf. The `${LISTEN_PORT}` placeholder will be substituted by `envsubst` at container startup.

    **C. Update deploy/entrypoint.sh PORT handling (DOC-02):**

    Replace lines 166-183 (the current sed-based PORT handling block):
    ```bash
    # ---------------------------------------------------------------------------
    # Dynamic port binding (Render injects PORT=10000 by default)
    # ---------------------------------------------------------------------------
    LISTEN_PORT="${PORT:-8080}"
    if ! echo "$LISTEN_PORT" | grep -qE '^[0-9]+$'; then
      echo "[FATAL] PORT invalide: '${LISTEN_PORT}' — doit être un nombre."
      exit 1
    fi
    if [ "$LISTEN_PORT" != "8080" ]; then
      if ! sed -i "s/listen 8080/listen ${LISTEN_PORT}/" /etc/nginx/http.d/default.conf 2>/dev/null; then
        echo "[FATAL] Impossible de modifier le port Nginx (filesystem read-only)."
        echo "        Utilisez PORT=8080 ou désactivez read_only dans docker-compose.yml."
        exit 1
      fi
      echo "Nginx port: ${LISTEN_PORT} (from \$PORT)"
    else
      echo "Nginx port: 8080 (default)"
    fi
    ```

    With this new block:
    ```bash
    # ---------------------------------------------------------------------------
    # Dynamic port binding (Render injects PORT=10000 by default)
    # ---------------------------------------------------------------------------
    export LISTEN_PORT="${PORT:-8080}"
    if ! echo "$LISTEN_PORT" | grep -qE '^[0-9]+$'; then
      echo "[FATAL] PORT invalide: '${LISTEN_PORT}' — doit être un nombre."
      exit 1
    fi
    # Use envsubst to generate nginx config from template — works on read-only FS
    # because /etc/nginx/http.d/ is writable (chown'd to www-data in Dockerfile).
    envsubst '${LISTEN_PORT}' < /var/www/deploy/nginx.conf.template > /etc/nginx/http.d/default.conf
    echo "Nginx port: ${LISTEN_PORT}"
    ```

    **D. Update Dockerfile COPY for template:**

    After the existing line `COPY deploy/nginx.conf /etc/nginx/http.d/default.conf` (line 68), add:
    ```dockerfile
    COPY deploy/nginx.conf.template /var/www/deploy/nginx.conf.template
    ```

    The static nginx.conf is still copied as the default (used if entrypoint doesn't run, e.g., `docker run --entrypoint sh`). The template is stored at `/var/www/deploy/nginx.conf.template` and the entrypoint's `envsubst` writes to `/etc/nginx/http.d/default.conf` at startup, overwriting the static copy.

    **E. Verify envsubst is available:**

    `envsubst` is part of the `gettext` package. Check if Alpine's nginx package includes it. If not, add to the Dockerfile's `apk add --no-cache` line (line 24-27):
    ```
    apk add --no-cache ... gettext
    ```

    Only add `gettext` if `envsubst` is not already available in the base image. The `php:8.4-fpm-alpine3.21` image may not include it by default, so adding `gettext` to the runtime libs line is the safe approach.

  <action>
    Rewrite `public/api/v1/health.php` to check three subsystems: database, redis, filesystem.

    The full replacement file content:

    ```php
    <?php
    /**
     * Health check endpoint for container orchestration (Docker, Render, etc.).
     *
     * Verifies:
     *  1. PHP-FPM is responsive (implicit — this file is executing)
     *  2. Database is reachable (SELECT 1)
     *  3. Redis is reachable (PING)
     *  4. Filesystem is writable (touch test file in upload dir)
     *
     * Does NOT require authentication or CSRF.
     * Returns HTTP 200 + JSON on success, HTTP 503 on failure.
     */

    declare(strict_types=1);

    header('Content-Type: application/json; charset=utf-8');
    header('Cache-Control: no-store');

    $checks = [
        'database' => false,
        'redis' => false,
        'filesystem' => false,
    ];

    // --- Database check ---
    try {
        $dsn = getenv('DB_DSN') ?: '';
        $user = getenv('DB_USER') ?: getenv('DB_USERNAME') ?: 'vote_app';
        $pass = getenv('DB_PASS') ?: getenv('DB_PASSWORD') ?: '';

        if ($dsn === '') {
            throw new RuntimeException('DB_DSN not configured');
        }

        $pdo = new PDO($dsn, $user, $pass, [
            PDO::ATTR_ERRMODE => PDO::ERRMODE_EXCEPTION,
            PDO::ATTR_TIMEOUT => 5,
        ]);
        $pdo->query('SELECT 1');
        $checks['database'] = true;
    } catch (Throwable $e) {
        error_log('health: db check failed: ' . $e->getMessage());
    }

    // --- Redis check ---
    try {
        $redisHost = getenv('REDIS_HOST') ?: '127.0.0.1';
        $redisPort = (int)(getenv('REDIS_PORT') ?: 6379);
        $redisPass = getenv('REDIS_PASSWORD') ?: null;

        if (class_exists('Redis')) {
            $redis = new Redis();
            $redis->connect($redisHost, $redisPort, 2.0); // 2 second timeout
            if ($redisPass) {
                $redis->auth($redisPass);
            }
            $pong = $redis->ping();
            // Redis::ping() returns true or '+PONG' depending on phpredis version
            $checks['redis'] = ($pong === true || $pong === '+PONG');
            $redis->close();
        } else {
            // Redis extension not loaded — mark as unavailable but not a failure
            // in environments where Redis is optional
            error_log('health: Redis extension not loaded');
        }
    } catch (Throwable $e) {
        error_log('health: redis check failed: ' . $e->getMessage());
    }

    // --- Filesystem check ---
    try {
        $uploadDir = getenv('AGVOTE_UPLOAD_DIR') ?: '/var/agvote/uploads';
        $testFile = $uploadDir . '/.health-check-' . getmypid();

        if (is_dir($uploadDir) && file_put_contents($testFile, 'ok') !== false) {
            $checks['filesystem'] = true;
            @unlink($testFile);
        } else {
            error_log('health: filesystem check failed: cannot write to ' . $uploadDir);
        }
    } catch (Throwable $e) {
        error_log('health: filesystem check failed: ' . $e->getMessage());
    }

    // --- Aggregate status ---
    $allOk = $checks['database'] && $checks['redis'] && $checks['filesystem'];
    $status = $allOk ? 'ok' : 'degraded';
    $httpCode = $allOk ? 200 : 503;

    http_response_code($httpCode);

    echo json_encode([
        'status' => $status,
        'checks' => $checks,
        'timestamp' => gmdate('Y-m-d\TH:i:s\Z'),
    ], JSON_UNESCAPED_UNICODE);
    ```

    Key design decisions:
    - Redis check uses the `Redis` extension (phpredis, already installed in Dockerfile via `pecl install redis`)
    - Redis connect timeout is 2 seconds (health check should be fast)
    - Filesystem check writes a temp file to AGVOTE_UPLOAD_DIR and deletes it immediately
    - PID-suffixed test file avoids collisions from parallel health checks
    - Status is `ok` only when ALL three checks pass; otherwise `degraded` with 503
    - Never leaks error details to callers — only writes to error_log
