<token_mapping_table>
**Mapping figé pour la substitution mécanique :**

| Valeur hardcodée | Token cible |
|------------------|-------------|
| `4px`            | `var(--space-1)`     |
| `6px`            | `var(--space-1-5)`   |
| `8px`            | `var(--space-2)`     |
| `10px`           | `var(--space-2-5)`   |
| `12px`           | `var(--space-3)`     |
| `16px`           | `var(--space-4)`     |
| `20px`           | `var(--space-5)`     |
| `24px`           | `var(--space-6)`     |
| `32px`           | `var(--space-8)`     |
| `0.25rem`        | `var(--space-1)`     |
| `0.5rem`         | `var(--space-2)`     |
| `1rem`           | `var(--space-4)`     |
| `1.5rem`         | `var(--space-6)`     |
| `2rem`           | `var(--space-8)`     |

**Cas spéciaux préservés (NE PAS remplacer) :**
- `0` (sans unité, pour reset)
- `auto` (centrage horizontal, etc.)
- `%` (pourcentages)
- `em` quand utilisé pour `line-height` ou tailles relatives au texte

**Variantes pad/gap :** si un token `--pad-*` ou `--gap-*` plus sémantique existe pour le contexte (ex: padding interne de bouton, gap de grid), le préférer à `--space-*` quand le nommage l'autorise. Sinon `--space-*` reste valable.

**Tokens absents :** si une valeur dans le code n'a pas de correspondance directe dans le mapping (ex: `5px`, `14px`), prendre le token le plus proche (`var(--space-1)` pour 5px, `var(--space-3-5)` ou `var(--space-4)` pour 14px selon ce qui existe). Documenter dans le SUMMARY les cas où un arrondi a été fait.
</token_mapping_table>
