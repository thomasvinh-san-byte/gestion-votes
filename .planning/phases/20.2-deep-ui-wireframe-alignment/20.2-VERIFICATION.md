### Notable Observations

1. **ag-toast.js positioning conflict:** The plan explicitly documented (plan 03 decisions) that `ag-toast.js` uses inline `top:20px; right:20px` which overrides the CSS class. The CSS design-system contract is bottom-right. If bottom-right toasts are needed in the web component context, a separate update to `ag-toast.js` is required.

2. **Hover gradient reverts to flat:** All three colored button variants (primary, success, danger) transition to flat `var(--color-X-hover)` on hover rather than maintaining a gradient. This matches the plan's explicit task instructions (the hover background line specifies `var(--color-primary-hover)`). The gradient is the resting state identity; hover uses flat for clarity. This is intentional.

3. **New components are CSS-only:** `.scroll-top`, `.tip-wrap`, `.confirm-dialog`, `.theme-toggle`, and `.op-kbd` are CSS definitions with no HTML usage yet. They are ready for consumption when features need them. This is explicitly the plan's intent.

4. **Global scrollbar coexistence:** `::-webkit-scrollbar` at 5px (line 419) is correctly overridden by `.sidebar-scroll::-webkit-scrollbar` at 3px (line 686) via specificity — intentional cascade behavior.
