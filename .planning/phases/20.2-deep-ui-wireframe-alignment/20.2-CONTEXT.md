<domain>
## Phase Boundary

This phase aligns ALL component CSS with wireframe v3.19.2 specifications. It covers:
- Design system tokens (design-system.css)
- Shared components (app.css, pages.css)
- Page-specific CSS (all 20+ page CSS files)
- Missing component patterns from wireframe

Reference wireframe: `/ag_vote_wireframe.html` (lines 1-1019 = CSS, tokens, components)

Out of scope:
- HTML structure changes (CSS-only phase)
- JavaScript behavior changes
- New pages or features

</domain>

<decisions>
## Implementation Decisions

### Token Mapping
- Wireframe uses short tokens (`--bg`, `--accent`, `--text`); implementation uses `--color-` prefixed (`--color-bg`, `--color-primary`, `--color-text`)
- Values already match — NO token renaming needed, just ensure CSS rules reference correct implementation tokens
- Dark theme `--color-bg` should stay as-is (#0B0F1A) — don't change to wireframe's #0B0D10 (implementation is intentional blue-shift)

### Buttons (HIGH PRIORITY)
- `.btn-primary`: Add gradient `linear-gradient(180deg, var(--color-primary) 0%, color-mix(in srgb, var(--color-primary), #000 8%) 100%)`
- Add `inset 0 1px 0 rgba(255,255,255,.12)` inner highlight
- Add `0 2px 4px rgba(22,80,224,.3)` colored shadow
- Font weight: increase to 700 (from 500)
- Add hover: `translateY(-1px)` lift + enhanced shadow
- Add active: `scale(.98) translateY(0)` press
- Add min-height: 40px
- `.btn-danger`, `.btn-success`: Same gradient + shadow pattern
- Default `.btn`: border should be `1px solid var(--color-border)` (not transparent)

### Cards (HIGH PRIORITY)
- Add transition: `box-shadow .18s ease, border-color .18s ease, transform .18s ease, background .18s ease`
- Add hover: `box-shadow: var(--shadow-sm)` on base `.card`
- Clickable cards: `translateY(-2px) + shadow-md + border-color accent-mix` on hover
- `.card-title`: Change to `13px; font-weight:700; text-transform:uppercase; letter-spacing:.7px` (from 18px heading)

### Form Fields (HIGH PRIORITY)
- Border: change from 2px to 1px
- Background: use `var(--color-surface-raised)` instead of `var(--color-surface)`
- Add min-height: 42px
- `.form-label` → uppercase, font-weight 700, font-size 13px, letter-spacing .5px

### Tables (MEDIUM PRIORITY)
- Add striped rows: `tbody tr:nth-child(even) td { background: rgba(21,21,16,.022) }`
- `th`: Add `text-transform:uppercase; letter-spacing:.7px; position:sticky; top:0`
- Add table cell overflow guard: `max-width:260px; overflow:hidden; text-overflow:ellipsis`
- Row hover: Use accent-tinted `color-mix(in srgb, var(--color-primary) 6%, var(--color-surface))`

### Tags (MEDIUM PRIORITY)
- Border radius: Change from `var(--radius-full)` (999px pill) to `5px` (slightly rounded rectangle)
- Add colored borders on semantic variants (`.tag-danger`, `.tag-success`, `.tag-warn`)

### Interactive Rows (MEDIUM PRIORITY)
- `.irow:hover .irow-title { color: var(--color-primary) }` — title accent on hover
- `.irow:hover .irow-arrow { opacity:1; transform:translateX(2px) }` — arrow slide
- `.irow-arrow { opacity:.3; transition:opacity .15s, transform .15s }` — base state

### Modals (MEDIUM PRIORITY)
- Backdrop: Add `backdrop-filter:blur(3px); -webkit-backdrop-filter:blur(3px)`
- Lighten backdrop: `rgba(0,0,0,.35)` instead of current .5
- Animation: Use bouncy easing `cubic-bezier(.34,1.2,.64,1)`
- Width: `min(520px, 100%)`

### Missing Components (NEW)
1. **Scroll-to-top button** (`.scroll-top`): Fixed bottom-right, circular, fade on scroll, accent hover
2. **Tooltip system** (`.tip-wrap` / `.tip-body`): Pure CSS, position above, arrow pseudo-element
3. **Confirm dialog** (`.confirm-dialog`): Centered icon circle (56px), Fraunces title, centered message
4. **Theme toggle switch** (`.theme-toggle`): CSS toggle with `::after` knob, translateX animation
5. **Keyboard hints** (`.op-kbd`): Mono font, 10px, background badge for operator shortcuts

### Toasts (MEDIUM PRIORITY)
- Move to bottom-right (from top-right)
- Add exit animation (`toastOut`)
- Add `.toast-icon`, `.toast-title`, `.toast-close` styling
- Add border-left per type (success, info, warn, error)

### Animations (LOW PRIORITY)
- `pageIn`: Adjust to `translateY(4px)` over `.18s` (from 6px/.25s)
- Add `@keyframes popIn` for popovers
- Add `@keyframes toastOut` for toast exit
- Use bouncy easing on modal (`cubic-bezier(.34,1.2,.64,1)`)
- Add card transition on base `.card` class

### Global Scrollbar (LOW PRIORITY)
- `::-webkit-scrollbar { width:5px; height:5px }`
- Thumb: `var(--color-border); border-radius:3px`
- Hover: `var(--color-text-light)`

### Print Styles (LOW PRIORITY)
- Extend hide list: sidebar, header, footer, toasts, search, theme toggle, mobile nav, scroll-top
- `.card` in print: `border:1px solid #ccc; box-shadow:none`
- Add `.print-only { display:none }` / `@media print { .print-only { display:block!important } }`

### Claude's Discretion
- Exact transition timing values can be adjusted if they feel off in practice
- If a CSS change breaks existing hover/focus states, preserve accessibility over wireframe spec
- Order of implementation within each plan
</decisions>

<specifics>
## Specific Ideas

- Wireframe reference: `ag_vote_wireframe.html` lines 1-1019 (CSS section)
- Button gradient pattern uses `color-mix(in srgb, ...)` — check browser compat (supported in all modern browsers)
- Card `.card-title` currently used in 15+ pages — the uppercase change is a global visual shift, ensure it looks right everywhere
- Form border change from 2px to 1px is also global — verify focus ring still visible with 1px
- Tag border-radius change from pill to 5px affects `.tag`, `.tag-accent`, `.tag-danger`, `.tag-success`, `.tag-warn`, `.tag-purple`
</specifics>

## Deferred Ideas

- Votant shell (`.votant-shell`, `.votant-header`, `.votant-question`) — vote page already redesigned, keep current architecture

- Device bar (`.device-bar`) — only relevant for multi-device preview, defer

</deferred>
