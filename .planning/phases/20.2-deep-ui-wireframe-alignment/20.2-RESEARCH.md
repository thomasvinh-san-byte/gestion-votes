## Gap Analysis: Implementation vs Wireframe

### 1. BUTTONS (design-system.css:1063-1227)

**Current `.btn`:**
- `font-weight: var(--font-medium)` (500) → Wireframe: 700
- `border: 1px solid transparent` → Wireframe: `1px solid var(--border)` for default btn
- Missing `min-height: 40px`
- No gradient, no inner highlight, no colored shadow

**Current `.btn-primary`:**
- Flat color only → Wireframe: gradient `linear-gradient(180deg, var(--accent), color-mix(in srgb, var(--accent), #000 8%))`
- No `inset 0 1px 0 rgba(255,255,255,.12)` highlight
- No `0 2px 4px rgba(22,80,224,.3)` shadow
- Hover: only background change → Wireframe: `translateY(-1px)` + enhanced shadow

**Current `.btn-danger` / `.btn-success`:**
- Flat backgrounds → Wireframe: gradient + hover lift + colored shadow

**Wireframe `.btn-p` class** maps to our `.btn-primary`. Need to port those styles.

### 2. CARDS (design-system.css:1230-1263)

**Current `.card`:**
- Has `box-shadow: var(--shadow-sm)` already (good)
- Missing transition: `box-shadow .18s ease, border-color .18s ease, transform .18s ease, background .18s ease`
- Missing hover: `box-shadow: var(--shadow-sm)` (wireframe hover only adds shadow to base cards)

**Current `.card-title`:**
- `font-size: var(--text-lg)` (18px) → Wireframe: `13px; font-weight:700; text-transform:uppercase; letter-spacing:.7px`

## Key Implementation Risks

1. **`.card-title` uppercase change** — Global visual change across 15+ pages. Must verify all card titles look good in uppercase 13px.
2. **Form border 2px→1px** — Focus ring relies on border. Verify focus visibility.
3. **Toast position change** — top-right→bottom-right. JS code may reference position for stacking logic.
4. **Tag pill→rounded** — All tag variants affected. Visual regression check needed.
5. **color-mix() usage** — Supported in all modern browsers (Chrome 111+, Firefox 113+, Safari 16.2+). Already used elsewhere in codebase.

## Implementation Strategy

### Plan grouping (by file + risk level):

**Plan 1: Design System Core — Buttons & Cards** (HIGH PRIORITY)
- Buttons: gradient, shadow, hover lift, active press, font-weight
- Cards: transition, hover, clickable hover, card-title uppercase

**Plan 2: Design System Core — Forms, Tables, Tags** (HIGH PRIORITY)
- Forms: border 1px, background surface-raised, min-height, label uppercase
- Tables: striped, sticky header, cell overflow, accent hover
- Tags: border-radius 5px, colored borders

**Plan 3: Design System Core — Modals, Toasts, Animations** (MEDIUM)
- Modals: backdrop blur, bouncy animation, width
- Toasts: bottom-right, sub-elements, border-left, exit animation
- Animations: pageIn, popIn, toastOut

**Plan 4: New Components & Global** (MEDIUM)
- scroll-top, tooltip, confirm-dialog, theme-toggle, op-kbd
- Scrollbar styling, print styles
- Interactive rows (.irow)

All changes go to `design-system.css` (primary) with possible touch-ups in `app.css` for toast container position overrides.
