# Plan 01.2 — Palette danger confinée aux états critiques

## Goal

Migrer toutes les occurrences décoratives de `--color-danger`, `--color-danger-subtle`, `--color-danger-strong` dans le scope cockpit (`operator.css` + `sidebar*.css` + `components/ag-*.css` + autres CSS référencés par `operator.htmx.html`) vers une palette neutre ou warning, **en préservant le rouge danger uniquement** pour les 4 cas critical-path approuvés (D-07).

Source : COCKPIT-V24-02 (REQUIREMENTS.md), Schoger S-6 (Refactoring UI Ch.4 "Use color sparingly"), CONTEXT.md décisions D-07..D-08.

## Context anchors

- **Baseline mesurée** : `grep -cE "color-danger|--danger" public/assets/css/operator.css` = **21 occurrences**
- **Cible** : occurrences `--color-danger*` réduites à celles strictement liées aux 4 cas critical-path approuvés (D-07)
- **Décision préservée v2.3 P3 B1** : `.hero-card--live` garde `--color-danger-subtle` (pas une migration, c'est un cas critical-path approuvé)

## Acceptance Criteria (what must be TRUE)

1. **Audit grep livré** dans `01.2-AUDIT.md` (artifact dédié) : pour chaque occurrence `--color-danger*` dans le scope cockpit, classification en :
   - `keep-critical` : quorum perdu animé, vote raté, erreurs bloquantes, hero card live
   - `migrate-neutral` : indicateurs factuels (pills d'état "X non-votants", connexion, présence)
   - `migrate-warning` : anticipation/attention (quorum à risque, SSE reconnecting)
   - `remove` : pure décoration sidebar/nav (à supprimer purement, pas remplacer)
2. **Migrations appliquées** par sed/edit ciblé. Chaque migration documentée dans le commit message (origine → destination).
3. **`grep -cE "color-danger" public/assets/css/sidebar*.css`** retourne **0** après migration (sidebar = 0 rouge décoratif).
4. **Hero card live préservée** : `--color-danger-subtle` reste sur `.hero-card--live` (vérifié par test dédié — non-régression v2.3 P3 B1).
5. **Cas critical-path préservés** : 4 occurrences minimum restent sur `--color-danger*` correspondant à :
   - `<ag-health-bar>` état quorum-missed (pulse animé)
   - `.op-vote-failed` ou équivalent erreur bloquante
   - `<ag-error>` ou modal d'erreur
   - `.hero-card--live` (héritage v2.3 P3 B1)
6. **Test gardien permanent** : `tests/Security/CockpitPaletteTest.php` créé. Scanne CSS opérateur + sidebar via PHPUnit, assert `--color-danger` absent des sélecteurs interdits (regex liste de sélecteurs migrés).
7. **Régression nulle** sur tests Playwright cockpit (cockpit-health-bar, cockpit-keyboard-shortcuts, critical-path-operator).
8. **Capture before/after** : 2 screenshots cockpit (sidebar visible + KPI strip + pills d'état) dans `01.2-SCREENSHOTS/`. Diff visuel commenté dans `01.2-SUMMARY.md` (réduction visible du rouge).
9. **Contrast WCAG 2.1 AA** : pour chaque migration vers `--color-warning` ou `--color-neutral-strong`, vérifier ratio fg/bg ≥ 4.5:1 (texte) ou 3:1 (UI components/borders) avec axe-core. Pas de régression a11y.

Un commit par fichier CSS migré (revertable). Pour chaque fichier :


## Risks & Mitigations

| Risk | Probability | Impact | Mitigation |
|---|---|---|---|
| Migration accidentelle d'un cas critical-path | M | H | Audit explicit avec classification AVANT toute migration. Re-grep après chaque commit pour confirmer count attendu. |
| Régression visuelle cockpit-health-bar (quorum pulse) | L | H | `ag-health-bar.css` explicitement listé "untouched" + test `cockpit-health-bar.spec.js` doit rester vert |
