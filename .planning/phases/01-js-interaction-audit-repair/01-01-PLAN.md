<task type="auto">
  <name>Task 1: Audit and fix operator page (6 JS files + HTML + SSE)</name>
  <files>
    public/operator.htmx.html,
    public/assets/js/pages/operator-tabs.js,
    public/assets/js/pages/operator-speech.js,
    public/assets/js/pages/operator-attendance.js,
    public/assets/js/pages/operator-motions.js,
    public/assets/js/pages/operator-exec.js,
    public/assets/js/pages/operator-realtime.js,
    public/assets/js/core/event-stream.js
  </files>
  <action>
    **Step 1 — Read all files:** Read operator.htmx.html (1652 LOC) fully. Then read each of the 6 operator JS files and event-stream.js.

    **Step 2 — Build DOM inventory:** From the HTML, extract every element with an `id` attribute and every element with a distinctive `class` used as a selector target. Also extract all `hx-target`, `hx-swap`, `hx-post`, `hx-get`, `hx-patch`, `hx-delete` attribute values.

    **Step 3 — Cross-reference JS selectors against DOM:**
    For each JS file, find every `querySelector`, `querySelectorAll`, `getElementById`, `getElementsByClassName`, `getElementsByTagName` call. Extract the selector string. Verify the selector matches at least one element in the HTML.

    Flag as BROKEN: selectors that reference IDs, classes, or elements not present in the HTML.

    **Step 4 — Check event handlers:**
    Find every `addEventListener` call and HTMX event hook (`htmx:afterSwap`, `htmx:beforeRequest`, `htmx:afterSettle`, etc.). Verify the target element exists and the event name is valid.

    **Step 5 — Check HTMX form submissions:**
    For every `<form>` and every element with `hx-post`/`hx-patch`/`hx-delete`, verify:
    - The endpoint path looks valid (matches a known API route pattern like `/api/v1/...`)
    - The `hx-target` references an element that exists in the DOM
    - The `hx-swap` value is valid (innerHTML, outerHTML, beforeend, afterend, delete, none)

    **Step 6 — Audit SSE wiring (JSFIX-04):**
    In event-stream.js and operator-realtime.js:
    - Verify EventSource URL construction is correct
    - Verify each `addEventListener` for SSE message types updates DOM elements that exist
    - Verify reconnection logic handles errors gracefully
    - Check that SSE message handlers reference valid selectors in operator.htmx.html

    **Step 7 — Fix all issues found:**
    For each broken selector/handler/target:
    - If the element was renamed/moved in HTML: update the JS selector to match current HTML
    - If the JS references a removed element: remove the dead code or add the missing element if it should exist
    - If an HTMX target is wrong: fix the target selector
    - If an event handler targets a non-existent element: fix or remove
    - Use judgment per D-03: minimal fix for isolated issues, root cause fix for patterns

    **Step 8 — Verify syntax:** Run `php -l` on any PHP files touched (unlikely). For JS, verify no syntax errors by checking balanced braces and valid JS constructs.
  </action>
