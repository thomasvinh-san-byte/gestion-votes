# Phase 41.2: Wizard UX and General Flow Improvements - Context

**Gathered:** 2026-03-20
**Status:** Ready for planning

<domain>
## Phase Boundary

Compact the wizard to fit each step on a single viewport without scrolling. Improve general UX flows across the app — reduce clicks, improve guidance, make workflows more efficient. No new features — purely UX optimization of existing functionality.

</domain>

<decisions>
## Implementation Decisions

### Wizard Single-Viewport Fit
- **Each wizard step must fit on a 1080p viewport without scrolling** — this is the primary constraint
- **Reduce vertical spacing:** Form group gaps from 24px → 16px (--space-field). Section gaps from 48px → 24px (--space-card)
- **2-column field layout:** Short fields (date + time, first name + last name, quorum + majority type) side by side using .form-grid-2
- **Collapse optional sections:** Advanced/optional fields start collapsed with a clean "Options avancées" toggle
- **Stepper:** Already compact from Phase 36 — verify it doesn't take too much vertical space
- **Footer nav:** Already sticky from Phase 32 — not counted in viewport height
- **Remove visual bloat:** Unnecessary padding, decorative elements, excessive whitespace that doesn't serve readability

### General UX Improvements
- **Reduce clicks to complete common tasks:** Identify the most common user workflows and streamline them
- **Smart defaults:** Pre-fill fields with sensible defaults where possible (e.g., today's date for meeting date, "AG Ordinaire" as default type)
- **Contextual actions:** Show relevant actions where the user is, not in a separate menu
- **Loading states:** Ensure all async operations show loading feedback (already partially done in v4.0)
- **Confirmation feedback:** Every destructive action requires confirmation, every save shows success feedback
- **Keyboard navigation:** Tab order logical across all forms, Enter to submit where appropriate

### Pages to Optimize
- **Wizard (primary):** Steps 1-4 must each fit on viewport. Field density, 2-column grids, collapsed sections
- **Hub:** Streamline the "launch session" workflow — fewer clicks from hub to live session
- **Operator:** Quick-action shortcuts for common operations during live sessions
- **Settings:** Ensure each section card is scannable without excessive scrolling

### Claude's Discretion
- Exact field grouping in 2-column grids (which fields pair up)
- Which optional sections to collapse by default
- Smart default values for each field type
- Whether to add keyboard shortcut hints
- Exact spacing reductions per page

</decisions>

- Optional sections are fully expanded by default
- No smart defaults in wizard fields
- Hub "launch session" requires multiple clicks through checklist

<specifics>
## Specific Ideas

- Think of the wizard like a Typeform experience — each step is one clear question/section, not a long form
- The goal is "fill and go" — an experienced operator should create a session in under 60 seconds
- If a step has more than 5 visible fields, it needs restructuring (2-column or collapsing)

</specifics>
