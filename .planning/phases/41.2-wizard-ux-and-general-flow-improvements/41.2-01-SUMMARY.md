# Phase 41.2 Plan 01: Wizard Viewport Compaction Summary

**One-liner:** Reduced wizard CSS spacing by ~86px, collapsed Regles de vote behind details/summary, switched field rows to form-grid-2, collapsed reso-add-panel behind trigger button, and added smart date/time defaults — all steps now fit 1080p viewport without scrolling.

## What Was Built

### Task 1: CSS Spacing Reductions

Four spacing rules tightened to free up ~86px of vertical space:
- `.wiz-step-body` padding: `2rem 2.5rem` -> `1.25rem 1.5rem` (saves 24px)
- `.wiz-section` padding: `1.5rem 0` -> `0.875rem 0` (saves 10px x3 sections = 30px)
- `.wiz-section-title` margin-bottom: `1.25rem` -> `0.75rem` (saves 8px x3 = 24px)
- `.wiz-required-note` margin-bottom: `1rem` -> `0.5rem` (saves 8px)

Dependent rules updated to match:
- `.wf-step-success` negative margins updated to match new body padding
- `.step-nav` padding: `1.25rem 2.5rem` -> `1rem 1.5rem`
- `.wiz-toggle-row` padding: `1rem 0` -> `0.5rem 0` (saves 8px)

New component styles added:
- `.wiz-advanced-toggle` / `.wiz-advanced-summary` / `.wiz-advanced-hint` / `.wiz-advanced-chevron` / `.wiz-advanced-body` — full native `<details>/<summary>` styling with chevron rotation animation
- `.reso-add-trigger` — margin wrapper for the trigger button
- `.upload-zone` updated to compact horizontal flex layout (padding `10px 14px`, `display: flex`, `align-items: center`, `gap: 12px`) — saves ~64px from vertical stacked layout

### Task 2: HTML Restructuring

**Step 1 changes:**
- Identite section: `.row` (3 fields) -> `form-grid-2` (Type + Date) + standalone Heure field with `max-width: 200px`
- Lieu section: `.row` (2 fields) -> `form-grid-2`
- Regles de vote: entire `.wiz-section` replaced with `<details class="wiz-advanced-toggle">` — collapsed by default
- Quorum + Majorite placed inside `form-grid-2` within `wiz-advanced-body`
- Alert-warn moved inside `wiz-advanced-body` (hidden when collapsed)

**Step 2 changes:**
- `ctx-panel` Guide block removed (~56px saved)

**Step 3 changes:**
- `reso-add-trigger` div with `btnShowResoPanel` button added before reso-add-panel
- `reso-add-panel` given `style="display:none;"` by default
- `ctx-panel` Guide block removed (~56px saved)

**wizard.js changes:**
- `setupAddReso()`: `addPanel` and `triggerDiv` hoisted to function scope
- `btnShowResoPanel` click handler: shows panel, hides trigger, focuses resoTitle
- `btnAddReso` click handler: collapses panel and shows trigger after successful add
- `init()`: smart defaults in existing `DRAFT_KEY === null` guard — pre-fills today's date and 18:00 time on fresh visits
