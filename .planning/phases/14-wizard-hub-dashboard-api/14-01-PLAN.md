  <name>Task 1: Fix wizard api() call order and response field access</name>
  <files>public/assets/js/pages/wizard.js</files>
  <action>
Fix two bugs in wizard.js around the session creation flow (near line 695-722):

1. **Line 703 — Fix api() argument order:**
   Current (WRONG): `api('POST', '/api/v1/meetings', payload)`
   This passes 'POST' as the URL parameter. The global api() signature is `api(url, data, method)`.
   Fix to: `api('/api/v1/meetings', payload)`
   (When data is non-null, api() auto-selects POST — no need for explicit method arg.)

2. **Line 712 — Fix response field access:**
   Current (WRONG): `res.id || res._id || ''`
   The api() function returns `{ status, body }` where body is `{ ok, data: { meeting_id, title } }`.
   Fix to: `(res.body && res.body.data && res.body.data.meeting_id) || ''`

3. **Add error handling for non-ok response:**
   After the `.then(function(res) {` line, add a guard:
   ```javascript
   if (!res.body || !res.body.ok) {
     throw new Error(res.body && res.body.error || 'creation_failed');
   }
   ```
   This ensures a 4xx/5xx response triggers the `.catch()` handler which shows the error toast.
  </action>

<task type="auto">
  <name>Task 2: Wire ag-toast on wizard/hub pages and add Shared.showToast</name>
  <files>public/assets/js/core/shared.js, public/wizard.htmx.html, public/hub.htmx.html</files>
  <action>
Three changes to enable toast notifications on wizard and hub pages:

1. **shared.js — Add showToast to Shared exports (around line 507-529):**
   Add a `showToast` method to the `window.Shared` object that bridges to AgToast:
   ```javascript
   showToast: function(message, type) {
     if (window.AgToast && typeof AgToast.show === 'function') {
       AgToast.show(type || 'info', message);
     } else {
       console.warn('ag-toast not loaded, toast skipped:', type, message);
     }
   }
   ```
   Note: The caller convention in wizard.js/hub.js is `Shared.showToast(message, type)` (message first, type second).
   AgToast.show expects `(type, message)` — so the bridge must swap the argument order.

2. **wizard.htmx.html — Add ag-toast script (after line 387, before closing body):**
   Add this line after the wizard.js script tag:
   ```html
   <script type="module" src="/assets/js/components/ag-toast.js"></script>
   ```
   The `type="module"` is required because ag-toast.js uses ES module syntax (export default).
   Place it BEFORE the wizard.js script? No — wizard.js is not a module. Place it AFTER shared.js but the order doesn't matter since wizard.js checks `window.Shared && Shared.showToast` at call time (event handler), not at parse time. Place it after the last existing script tag.

3. **hub.htmx.html — Add ag-toast script (after line 225, before closing body):**
   Same as wizard: add `<script type="module" src="/assets/js/components/ag-toast.js"></script>` after the hub.js script tag.
  </action>
