# Phase 2 — UI Design Contract: Sidebar Navigation

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — custom CSS variables in `design-system.css` |
| Preset | not applicable |
| Component library | none — hand-rolled HTML + CSS components |
| Icon library | Custom SVG sprite at `/assets/icons.svg` (inline `<use href="...#icon-*">`) |
| Font | `Bricolage Grotesque`, system-ui, sans-serif (--font-sans) |

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`--gap-xs`) | Icon gaps, badge padding |
| sm | 8px (`--gap-sm`) | Nav item horizontal padding (right side) |
| md | 16px (`--gap-md`, `--pad-md`) | Nav item horizontal padding (left side) |
| lg | 24px (`--gap-lg`) | Sidebar internal section padding |
| xl | 32px (`--gap-xl`) | Layout gaps |
| 2xl | 48px (`--pad-2xl`) | Major section breaks |
| 3xl | 64px (`--space-16`) | Page-level (header height) |

Exceptions:
- Nav item gap between icon and label: 10px (`gap: 10px` on `.nav-item`) — kept as-is, established pattern
- Nav group header height: 20px content + 6px/2px padding = 28px total — will be raised to 44px (see Touch Targets)
- Sidebar width: 200px (replaces 58px rail and 252px expanded — source: CONTEXT.md decision)
- App-main padding-left: `calc(200px + 20px)` = 220px static (source: CONTEXT.md decision)

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px (`--text-base: 1rem`) | 400 (regular) | 1.5 |
| Label (nav items, group labels) | 16px (`--text-base`) | 500 (medium) | 1.2 |
| Group header label | 12px (`--text-xs`) | 700 (bold) | 1.2 |
| Version / metadata | 12px (`--text-xs`) | 400 (regular) | 1.2 |

> Source: Phase 1 token decision (`--text-base: 1rem`). Current `.nav-label` and `.nav-item` declare `font-size: 14px` — these must be updated to `var(--text-base)` (16px) as part of this phase. Nav group labels stay at 12px (section header role, not reading text).

Overrides required in this phase:
- `.nav-item`: change `font-size: 14px` → `font-size: var(--text-base)`
- `.nav-label`: change `font-size: 14px` → `font-size: var(--text-base)`

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--color-bg`: slate-200 (`#e2e8f0`) / dark `oklch(0.090 0.008 220)` | Page background, `.app-main` surface |
| Secondary (30%) | `--sidebar-bg`: `oklch(0.080 0.015 260)` light / `oklch(0.060 0.012 260)` dark | Sidebar background, nav container |
| Accent (10%) | `--color-primary`: `oklch(0.480 0.195 265)` (blue) | Active nav item indicator bar, active state background |
| Destructive | `--color-danger`: red-600 `oklch(0.510 0.175 25)` | Destructive actions only — not present in this phase |

Accent reserved for:
- `.nav-item.active::before` left accent bar (2.5px vertical stripe)
- `.nav-item.active` background (`--sidebar-active: oklch(0.520 0.195 265 / 0.30)`)
- Focus ring on keyboard-focused nav items (`--color-border-focus`)

Sidebar text colors (established, no change):
- Default: `--sidebar-text`: `oklch(1 0 0 / 0.85)` (white at 85% opacity)
- Hover: `--sidebar-text-hover`: `oklch(1 0 0)` (full white)
- Group header: `rgba(255, 255, 255, .5)` (50% white)

> Source: codebase scan. No color changes required in this phase — sidebar palette is already correct.

## Component Inventory

### `.app-sidebar` — Static Sidebar Container

Current state: `width: var(--sidebar-rail)` (58px), expands to `var(--sidebar-expanded)` (252px) on hover or `.pinned`.

Required change:
```css
--sidebar-width: 200px;   /* replaces both --sidebar-rail and --sidebar-expanded */
.app-sidebar {
  width: 200px;            /* static, never collapses */
  /* remove :hover and .pinned width overrides */
}
```

Remove entirely:
- `.app-sidebar:hover { width: var(--sidebar-expanded); }` block
- `.app-sidebar.pinned { ... }` width rules
- `.sidebar-pin` button element and styles
- `localStorage` pin persistence logic in `shell.js`
- `--sidebar-rail: 58px` and `--sidebar-expanded: 252px` tokens (or keep for backward compat, unused)

### `.nav-label` — Item Text Label

Current state: `opacity: 0; max-width: 0; overflow: hidden` — hidden in rail mode, revealed on hover/pin.

Required change:
```css
.nav-label {
  opacity: 1;              /* always visible */
  max-width: 180px;        /* fixed visible width */
  overflow: hidden;
  text-overflow: ellipsis;
  white-space: nowrap;
  font-size: var(--text-base);   /* 16px — replaces 14px */
  font-weight: 500;
  /* remove transition (no longer animated) */
}
```

### `.nav-group-label` — Section Header Label

Current state: `opacity: 0; max-width: 0` — hidden in rail mode.

Required change:
```css
.nav-group-label {
  opacity: 1;              /* always visible */
  max-width: 160px;
  font-size: 12px;         /* keep — section header role */
  font-weight: 700;
  letter-spacing: 1.4px;
  text-transform: uppercase;
  /* remove transition */
}
.nav-group-chevron {
  opacity: 1;              /* always visible */
  /* remove opacity transition */
}
```

### `.nav-item` — Navigation Link / Button

Current state: `height: 42px` — 2px below WCAG 2.5.8 minimum.

Required change:
```css
.nav-item {
  height: 44px;            /* was 42px — WCAG 2.5.8 minimum touch target */
  min-height: 44px;
  font-size: var(--text-base);   /* 16px — was 14px */
  /* all other properties unchanged */
}
```

### `.nav-group` — Section Header Button

Current state: `height: 20px; padding: 6px 10px 2px` — total ~28px, below 44px minimum.

Required change:
```css
.nav-group {
  height: 44px;            /* was 20px — raise to match touch target minimum */
  padding: 0 10px;         /* vertically centered */
  align-items: center;
}
```

### `.app-main` — Main Content Area

Current state: `padding-left: calc(var(--sidebar-rail) + 22px)` = 80px; toggled to `calc(var(--sidebar-expanded) + 22px)` = 274px when pinned.

Required change:
```css
.app-main {
  padding-left: calc(200px + 20px);   /* 220px static — no JS toggle needed */
}
/* Remove: .app-sidebar.pinned ~ .app-main { padding-left: ... } */
```

### `shell.js` — Sidebar Controller

Remove the following behaviors:
- Pin/unpin toggle (`#sidebarPin` click handler)
- `localStorage` read/write for pin state (`sidebar-pinned`)
- CSS class `.pinned` add/remove on `.app-sidebar`
- Dynamic `padding-left` recalculation on `.app-main`
- Keep: `--sidebar-top` calculation (header height offset)
- Keep: scroll fade indicators (`.sidebar-fade` scroll detection)
- Keep: nav group collapse/expand behavior

### `data-requires-role` Filtering

Current state: filtering already implemented in `auth-ui.js` via `/api/v1/whoami.php` response.

No changes to filtering logic. Verify voter role visibility:
- `voter` role sees: `/vote` (Voter) and `/settings` (Mon compte)
- All other items must have `data-requires-role` excluding `voter`

Current issue: `/vote` link appears in `.sidebar-device-section` labeled "Apercu votant" with role `admin,operator,president,voter` — this is the voter entry point. Confirm with auth-ui.js that a voter user gets exactly 2 visible items.

## States and Interactions

### Nav Item States

| State | Visual |
|-------|--------|
| Default | `--sidebar-text` (white 85%), transparent bg |
| Hover | `--sidebar-hover` bg (`oklch(1 0 0 / 0.10)`), `--sidebar-text-hover` (full white) |
| Active (current page) | `--sidebar-active` bg, full white text, 2.5px left accent bar in `--color-primary` |
| Focus (keyboard) | `outline: 2px solid var(--color-border-focus)` with 2px offset, visible focus ring |
| Disabled | `display: none` (role-filtered items) — no partially-visible disabled state |

### Nav Group Collapse/Expand

Behavior unchanged: click `.nav-group` toggles `.collapsed` class, rotates chevron -90deg. Groups default to expanded (`aria-expanded="true"`).

### Sidebar Version / Device Tags

`.sidebar-version` and `.device-tag` elements: set `opacity: 1` statically (currently only visible on hover/pin). The device preview section is visible to `admin,operator,president,voter` — keep as-is.
