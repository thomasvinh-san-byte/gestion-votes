Add these lines after line 324 (`e.detail.headers['X-CSRF-Token'] = token;` block closing brace):

```javascript
    // Auto idempotency key for mutating HTMX requests
    const method = (e.detail.verb || '').toUpperCase();
    if ((method === 'POST' || method === 'PATCH') && !e.detail.headers['X-Idempotency-Key']) {
      e.detail.headers['X-Idempotency-Key'] = crypto.randomUUID();
    }
```

Key details:
- Use `e.detail.verb` (HTMX's property for HTTP method), NOT `e.detail.method`
- Only POST and PATCH (matching backend IdempotencyGuard usage) -- NOT PUT (HTMX rarely uses PUT), NOT DELETE (no idempotency needed)
- Guard with `!e.detail.headers['X-Idempotency-Key']` so manually set keys (like in vote.js ballot cast) are not overwritten
- `crypto.randomUUID()` matches the existing pattern in the apiPost helper at line 642

Do NOT modify the apiPost/apiPut helper -- it already has its own idempotency key logic.
