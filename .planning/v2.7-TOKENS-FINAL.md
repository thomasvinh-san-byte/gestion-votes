> **Note importante sur la métrique spacing** : la commande grep planifiée par le critère V27-04
> `[0-9]+` capture aussi `margin: 0`, `padding: 0`, `margin: 0 auto` qui sont des exclusions audit
> (no-token-needed, mot-clé `auto`). La métrique réelle utilisée pour le statut est la commande

> du plan, le total descendant est 508 → 197 (les 175 résiduels sont tous `0`/`auto`).

## Méthode de comptage (reproductible à tout moment)

### Spacing — métrique raffinée (recommandée)

```bash
# Compte uniquement les valeurs px/rem hors exclusions no-token (0, auto, inherit)
total=0
for f in public/assets/css/*.css; do
  bn=$(basename "$f")
  [ "$bn" = "design-system.css" ] && continue
  c=$(grep -cE "(padding|margin)(-(top|right|bottom|left|inline|block|inline-start|inline-end|block-start|block-end))?:[^;]*[0-9]+(px|rem)" "$f")
  total=$((total + c))
done
echo "spacing real raw = $total"
# Résultat 2026-05-05 : 22
```

### Spacing — métrique brute originelle (V27-04 critère plan)

```bash
total=0
for f in public/assets/css/*.css; do
  bn=$(basename "$f")
  [ "$bn" = "design-system.css" ] && continue
  c=$(grep -cE "(padding|margin)(-(top|right|bottom|left|inline|block|inline-start|inline-end|block-start|block-end))?:[[:space:]]*[0-9]+" "$f")
  total=$((total + c))
done
echo "spacing broad = $total"
# Résultat 2026-05-05 : 197 (dont ~175 motifs 0/auto exclus par audit)
```

### Borders ratio (color tokenisée vs hex/rgb literal)

```bash
T=$(grep -rcE "border(-(top|right|bottom|left))?:[^;]*var\(--" public/assets/css/ --include="*.css" \
    | grep -v design-system.css | awk -F: '{s+=$2} END {print s}')
B=$(grep -rcE "border(-(top|right|bottom|left))?:[^;]*(#[0-9a-fA-F]{3}|rgba?\()" public/assets/css/ --include="*.css" \
    | grep -v design-system.css | awk -F: '{s+=$2} END {print s}')
echo "borders: tokenises=$T raw=$B ratio=$(echo "scale=3; $T/($T+$B)*100" | bc)%"
# Résultat 2026-05-05 : T=384 B=3 ratio=99.22%
```

### Shadows ratio

```bash
T=$(grep -rcE "box-shadow:[^;]*var\(--" public/assets/css/ --include="*.css" \
    | grep -v design-system.css | awk -F: '{s+=$2} END {print s}')
B=$(grep -rcE "box-shadow:[^;]*(rgba?\(|#[0-9a-f])" public/assets/css/ --include="*.css" \
    | grep -v design-system.css | awk -F: '{s+=$2} END {print s}')
echo "shadows: T=$T B=$B"
# Résultat 2026-05-05 : T=112 B=0 → 100%
```

### Hex (hors design-system + emails + print)

```bash
for f in public/assets/css/*.css; do
```

## Exceptions documentées

### Spacing (22 hits)

Toutes les valeurs ci-dessous ont été conservées en littéral parce que :
1. Aucun token de la palette `--space-*` n'est équivalent (off-grid), et
2. Le contexte visuel (alignement pixel-perfect, fine-tuning, calc dynamique) interdit
   l'arrondi à >2px du token le plus proche.

| Fichier:ligne | Code | Raison |
|---------------|------|--------|
| `app.css:384` | `padding: 1px var(--space-1-5);` | Bordure-tier 1px (échelle ne descend pas sous --space-1=4px) |
| `doc.css:162` | `padding: 0.3rem var(--space-2);` | Micro-fine-tuning (4.8px), entre --space-1 (4px) et --space-1-5 (6px) |
| `doc.css:347` | `padding: 0.15rem 0.4rem;` | Pill compact, fine-tuning (2.4px / 6.4px) hors grille |
| `help.css:141` | `padding-left: 2.75rem;` | Padding pour icône search (44px = icon 24 + gap 20) — calc explicite, pas un spacing logique |
| `hub.css:172` | `margin-top: 1px;` | Compensation 1px sub-pixel rendering, hors échelle |
| `meetings.css:128,482,486` | `padding: 1px ...;`, `margin-bottom: 1px;` | Bordure-tier 1px, idem app.css |
| `meetings.css:274` | `padding: 0.15rem 0.6rem;` | Pill badge compact, fine-tuning |
| `members.css:663` | `padding: 0.15rem var(--space-2);` | Pill badge compact, fine-tuning |
| `operator.css:210` | `padding: 0.3rem var(--space-4);` | Mode-indicator chip, fine-tuning (4.8px) |
| `operator.css:678` | `padding: 0.2rem 0.6rem;` | Indicateur état mini-pill |
| `operator.css:850` | `margin-bottom: -1px;` | Overlap 1px pour fusion border, intentionnel |
| `operator.css:1832` | `margin: 0 0 0.2rem;` (inline rule) | Spacing minuscule h4, fine-tuning |
| `public.css:801` | `padding: 0.2rem var(--space-2);` | Vote badge compact |
| `users.css:163` | `padding-left: 2.375rem;` | Padding pour icône search (38px = icon 22 + gap 16) — calc explicite |
| `users.css:177` | `padding: 0.3rem var(--space-3);` | Tab pill, fine-tuning 4.8px vertical |
| `users.css:334` | `margin-top: 0.1rem;` | Compensation baseline (1.6px), sub-token |
| `users.css:341,380` | `padding: 0.2rem 0.55rem;` | Role-badge compact (3.2px / 8.8px) |
| `vote.css:1077` | `padding-bottom: calc(60px + env(safe-area-inset-bottom, var(--space-4)));` | iOS safe-area dynamic calc — 60px est offset éditorial spécifique |
| `wizard.css:783` | `margin-top: -1px;` | Overlap 1px, idem operator.css:850 |

**Total : 22 exceptions** — toutes catégorisées en 3 familles :
1. **Bordures-tier 1px** (app/meetings) — `--space-*` ne descend pas sous 4px ;
2. **Fine-tuning sub-token** (badges, pills, mini-spacing 0.15-0.6rem) — l'arrondi vers
   `--space-1` (4px) ou `--space-2` (8px) déformerait visuellement les composants compacts ;
3. **Calc/icon-offset** (search inputs, safe-area) — pas un spacing logique mais une expression
   géométrique liée au design d'icône ou contraintes plateforme.
