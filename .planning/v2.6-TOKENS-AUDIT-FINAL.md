## Méthode de comptage

Token est *1-site* si son `var(--token)` apparaît **dans exactement 1 fichier** sous `public/assets/css/` hors `design-system.css` (occurrences multiples dans le même fichier — ex. `:hover`/`:focus` doublons — comptent comme un seul site). Détecté par grep itératif sur les 59 tokens `--border-*` / `--shadow-*` du design system, post-merge Phase 3 v2.6.

Méthode ratio : sur les fichiers CSS de `public/assets/css/` (hors design-system.css lui-même qui ne contient que les définitions), comparer les déclarations `border:` et `box-shadow:` qui contiennent un `var(--…)` vs celles qui n'en contiennent pas (hex, rgb, oklch, named colors).

## Ratio tokens (borders / shadows)

Mesure sur `public/assets/css/` hors `design-system.css` lui-même.

### Bordures
- **Total déclarations `border:` (hors `none`/`0`/`inherit`/`initial`/`unset`)** : 385
- Avec `var(--…)` : 376
- Sans `var(--…)` (hardcoded) : 9
- **Ratio : 376 / 385 × 100 = 97.7%** (cible ≥95% ✓)

Les 9 déclarations hardcoded restantes sont :
- `border: …px solid transparent;` (5 cas) — placeholder hover/active state pour préserver la largeur (technique standard, non régressif)
- `border-…: 1px solid #000;` (2 cas dans `editorial.css` `@media print`) — print stylesheet, hex justifié
- `border-top: 1px solid oklch(1 0 0 / 0.15);` (login.css:173) — divider sur fond gradient login, valeur unique justifiée
- `border-left: 2px solid transparent;` (doc.css:123) — placeholder hover

### Shadows
- **Total déclarations `box-shadow:` (hors `none`/etc.)** : 102
- Avec `var(--…)` : 102
- Sans `var(--…)` : **0**
- **Ratio : 102 / 102 × 100 = 100.0%** (cible ≥95% ✓)

Tous les `box-shadow` hors design-system.css utilisent un token `var(--shadow-…)` ou un inline `var(--color-…)` (cas Phase 7.4 emphasis intentionnels). Aucune fuite hardcoded.
